# 库存列表行显示方案

## 当前问题

当前库存列表是**平铺式**显示，每行一个独立记录，用户反馈与设计稿不符。

## 设计稿要求（分组折叠式）

设计稿中的库存列表采用 **"物料分组 + 批次展开"** 的两级结构：

### 1. 分组汇总行（可点击折叠/展开）

```
┌────┬──────────────────┬─────────┬────────┬────────────┬────────┬────────┬────────┐
│ ▶  │ 一次性手套       │ 3 批次  │ 403仓库│ 612.5 充足 │  -     │  -     │ 详情 出库 │
│    │ 100/盒           │         │        │            │        │        │        │
└────┴──────────────────┴─────────┴────────┴────────────┴────────┴────────┴────────┘
```

**分组行字段**：
- **耗材名称**：物料名称 + 规格（如"一次性手套" + "100/盒"）
- **批号**：显示批次数量（如"3 批次"）
- **库位**：该物料主要存放位置
- **库存数量**：该物料所有批次的库存总和（如"612.5 充足"）
- **状态**：整体状态标签
- **操作**：详情 + 出库按钮

### 2. 批次明细行（展开后显示）

点击分组行后展开，显示该物料下的各个批次：

```
┌────┬──────────────────┬──────────┬────────┬────────────┬──────────┬────────┬────────┐
│ ☐  │ └ 一次性手套     │ 2121212  │403仓库-│ 300 正常   │ 2026-12  │ 正常   │ 详情 出库│
│    │                  │          │库区1-  │            │          │        │        │
│    │                  │          │货架2   │            │          │        │        │
├────┼──────────────────┼──────────┼────────┼────────────┼──────────┼────────┼────────┤
│ ☐  │ └ 一次性手套     │ 202604303│403仓库-│ 200 正常   │ 2026-10  │ 正常   │ 详情 出库│
│    │                  │          │库区1-  │            │          │        │        │
│    │                  │          │货架2   │            │          │        │        │
├────┼──────────────────┼──────────┼────────┼────────────┼──────────┼────────┼────────┤
│ ☐  │ └ 一次性手套     │ 20260501 │403仓库-│ 112.5 正常 │ 2026-11  │ 正常   │ 详情 出库│
│    │                  │          │库区1-  │            │          │        │        │

└────┴──────────────────┴──────────┴────────┴────────────┴──────────┴────────┴────────┘
```

**批次行字段**：
- 缩进显示（带"└ "前缀），表示是子行
- **批号**：具体批号
- **库位**：完整层级路径
- **库存数量**：该批次的具体数量
- **有效期**：该批次的有效期
- **状态**：该批次的独立状态
- **操作**：详情 + 出库

### 3. 交互设计

```
点击分组行 ▶ 展开/收起该物料下的所有批次
点击详情    → 查看该物料/批次的详细信息
点击出库    → 打开出库弹窗
```

## 数据分组逻辑

```javascript
// 按 material_name 分组
const groups = {};
data.forEach(item => {
  if (!groups[item.name]) groups[item.name] = [];
  groups[item.name].push(item);
});

// 每组计算汇总
Object.keys(groups).forEach(name => {
  const batches = groups[name];
  const totalStock = batches.reduce((sum, b) => sum + (b.stock || 0), 0);
  const first = batches[0];
});
```

## 实施方案

### 1. 前端数据分组
在 `InventoryList.tsx` 的 `sortedData` useMemo 之后，添加分组逻辑：

```typescript
const groupedData = useMemo(() => {
  const groups: Record<string, InventoryRow[]> = {};
  sortedData.forEach(item => {
    if (!groups[item.name]) groups[item.name] = [];
    groups[item.name].push(item);
  });
  return groups;
}, [sortedData]);
```

### 2. 添加展开/收起状态
```typescript
const [expandedGroups, setExpandedGroups] = useState<Set<string>>(new Set());
```

### 3. 修改表格渲染
将当前平铺的 `<tr>` 改为：
- 先渲染分组汇总行（带 ▶/▼ 图标）
- 条件渲染批次明细行

### 4. 后端配合
当前后端已经返回了按物料+批次的明细数据（通过 `batches` 表 LEFT JOIN），但需要确保：
- 同一物料的不同批次都返回
- 按 `material_name` 排序，方便前端分组

### 5. 视觉样式
- 分组行：加粗字体、稍深背景色
- 批次行：缩进 48px、较浅背景色
- 展开/收起图标：右侧旋转的三角形

## 建议

由于这是一个较大的 UI 重构（涉及分组、折叠、汇总计算），建议作为一个独立的任务来处理，避免与当前入库修复任务混在一起。
