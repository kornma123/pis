# Wave A 库存与出库工作流规格

## 事实边界

- `/inventory` 当前返回的是按物料汇总的正库存缓存，不是逐批次、逐库位持仓表。
- `batch` / `expiry` 只代表当前最早可用的 FEFO 候选批次；页面不得把它写成“全部批次”或把一行写成一个批次。
- `locationId` 是物料级登记指针。P1-06 未批准前，只能显示“登记库位（未按批次验证）”；空指针显示“未登记”，有 ID 但名称解析失败显示“库位引用失效”。不展示容量承诺。
- 普通出库不指定批次：后端在全部 eligible 批次中按 FEFO 原子拆分。指定批次能力只有拿到可信批次证据后才能开放；指定批次不足应由后端 422 整单拒绝且零副作用。

## 信息层级与状态机

1. 页头：任务名称、证据边界、唯一主操作。
2. 状态摘要：只展示后端统计口径，并标明“物料条目”，不把条目数叫库存数量。
3. 筛选与分页：只提交活 API 支持的参数；筛选变化回第一页；当前页排序必须明确是本页排序。
4. 表格：物料 → 正库存缓存 → FEFO 起始批次/效期 → 登记库位证据 → 操作。
5. 请求状态：`loading → fresh | empty | error`；fresh 后刷新失败进入 `stale`，保留上次数据但禁止库存写；读权限不足进入 `permission`。
6. 出库提交：`editing → submitting → success`；400/409/422/5xx 回到 `editing + error`，保留全部输入且不显示成功。

## 表单、对话框与可达性

- 一张出库单使用一个项目、一个备注和多条物料；一次确认只调用一次 POST，重复点击合并为同一次提交。
- 数量逐字段校验，错误与字段关联；422 在对话框内说明“整单未出库，可调整数量后重试”。
- 对话框使用 `role="dialog"`、可读标题、关闭按钮名称与 Escape；打开后聚焦标题/首字段，关闭后归还焦点。
- 表格操作均为真实 `button`；排序使用 `aria-sort`；复选框有可读名称；状态不只依赖颜色。
- 桌面保持表格，窄屏允许横向滚动并保持主操作与错误说明可见，不裁切底部确认按钮。
