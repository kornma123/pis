# 客户退库工作流规格

- 业务效果：登记成功增加库存；撤销成功按相反方向扣回。仅 `returns:W` 可写，R-only 只读。
- 状态机：填写物料/数量/原因 → 明确库存方向 → 提交锁 → 仅含有效 `id` 的回执可判成功；拒绝与结果未知分开提示。同一 payload 在回执丢失或格式不可验证后重试复用后端幂等键。
- 列表：loading/error/empty 必须分离，筛选、排序、分页和导出沿用 Lane C 组件；撤销需二次确认，批量撤销逐条汇总成功/失败。
- 键盘/窄屏：共享 Modal/ConfirmDialog 和横向表格；主要控件可键盘操作。
- 已知 excluded 边界：共享 `_laneC/**` 的统计请求失败目前仍会显示初始化值，不能作为“真实为 0”的验收证据，本 wave 不越界修改。
