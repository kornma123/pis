# 合作方配置工作流规格

- 信息层级：先选医院，再按基本档案、业务分类、扣率、特殊结算、解析信息、变更记录分区编辑；版本、基线与未保存状态始终可见。
- 状态机：并行加载配置与历史 → 编辑草稿 → 校验 → 带 `expectedVersion` 保存 → 成功证据或冲突刷新；放弃、回滚、设基线均需确认并锁定重复提交。
- 权限：admin/finance 可进入；拆分/诊断口径整行（名称、启停、归属、费率、工作量来源、识别词、删除）仅 admin 可改，finance 只读。
- 恢复：列表/详情失败不渲染空配置；旧医院响应不得覆盖新选择；普通失败保留草稿，版本冲突刷新后明确要求重做。
- 键盘/窄屏：复用全局 Modal 的焦点圈、Escape、焦点恢复；tabs 支持左右键；表单网格在窄屏单列。
