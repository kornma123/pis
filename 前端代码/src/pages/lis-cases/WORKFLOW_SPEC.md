# LIS 病例工作流规格

- 信息层级：列表只展示病理号、医院、工作量、样本分类和流程状态；详情补充登记时间、批次与抗体，不展示患者身份、诊断或原始载荷。
- 状态机：文件白名单校验 → 病例行服务端预检（不落库）→ 人工确认 → 分块提交 → `全部完成 | 部分完成 | 处理结果未知`。仅有抗体行时明确标记病例预检不适用，合作方映射只以提交回执为证；病例拒收时抗体步骤停止，拒收/跳过/未匹配不得进入成功数。
- 交互：列表 loading/error/empty 分离；失败可重试，旧响应丢弃。未映射合作方的病例不得打开歧义详情。样本覆盖仅 `reconciliation:W` 可见，提交以同步锁防重复。
- 键盘/窄屏：病理号使用有名称的按钮；筛选有 label；表格横向滚动；卡片与数量网格在窄屏降列。
- 隐私：解析后仅在内存保存白名单列和文件名；不持久化原文件，不在提示、日志或结果中回显原始行或后端原始错误正文。
