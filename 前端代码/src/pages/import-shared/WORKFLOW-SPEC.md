# Wave B 工作流规格（冻结）

## 事实边界

- 采购单页面只展示采购事实，并把采购单号、物料和返回地址带到入库页作为只读上下文；不得调用采购单 `receive` 冒充真实收货。
- 直接入库 CSV 的每个有效行独立调用现有入库接口。它会立即影响库存，但不是整批事务；无效行不提交，已成功行不回滚。
- 财务对账单先预览、再提交；预览不等于落库。只有服务端返回 `NEEDS_CONFIRM` 才出现旁路确认，确认必须带非空理由。

## 状态机

### 直接入库 CSV

`未选择 → 读取/校验 → 可提交（有效行可为子集） → 人工确认 → 提交中 → 已结算`

- 文件级拒绝没有行级结果，不能显示为“0 条成功”。
- 已结算回执分别记录文件总行、校验拒绝、服务成功、服务失败；失败重试沿用原幂等键。
- 文件替换立即清掉旧预览与确认。若离开时仍在提交，返回后只显示“结果未知”，不自动重提。

### 财务对账单

`未选择 → 解析 → 待补医院/账期 → 预览中 → 待归类或可提交 → 提交中 → 需旁路确认 / 已结算 / 失败`

- 同一队列项只允许一个提交请求；页面在任一解析/提交期间显式锁住其他提交，内部忙碌计数保证并发返回时不会提前解锁。
- 403/医院列表失败是明确的不可用状态，不得伪装成空医院列表。
- 返回页面只恢复最小提交日志与成功回执，不恢复原始网格，也不自动提交。

## 信息与交互

- 首屏顺序：对象与影响范围 → 文件/上下文 → 状态摘要 → 行表/预览 → 明确提交动作 → 回执。
- 行表保留行号、关键匹配字段、状态与原因；长表启用 `content-visibility`，窄屏横向滚动。
- 文件按钮、错误摘要、确认理由和提交动作均可用键盘到达；错误摘要使用 `role="alert"`，异步状态使用 `aria-live` / `aria-busy`。
- 返回后的提交中日志只能表述为未知：先核对记录/回执，再决定是否重新选择文件。

## 浏览器持久化

- 仅使用当前标签页的 `sessionStorage` 保存文件名、对象标识、阶段、计数和服务端回执号。
- 不保存文件二进制、原始网格、行内容、金额明细、鉴权信息或权限判断。
