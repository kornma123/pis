# 供应商退货工作流规格

- 信息层级：列表显示退货单、物料、供应商、数量、原因、拟登记退款额和当前状态；详情仅展示可验证字段，不生成无时间证据的伪时间线。
- 状态机：`待发货 → 发货状态已登记 → 供应商已收货 → 退款状态已登记（未过账）`；允许状态可取消并恢复库存。删除仅限待发货。每一步都显示前置条件、确认、提交锁和回执。
- 写权限：所有创建、流转、删除只看 `supplier_returns:W` capability；R-only 用户完整只读。
- 恢复：列表/引用/详情错误与空态分离，独立请求并行；旧响应丢弃。创建用幂等键安全重试；状态/删除回执丢失则锁定写操作，先刷新详情核对。
- 财务真实性：`refunded` 只是工作流字段；当前无应付贷项/财务台账过账，不显示“已退款/已冲销/到账”。0 不显示为已收款，而显示未登记退款额。
- 键盘/窄屏：复用全局 Modal 的焦点管理；所有图标按钮有名称；表单窄屏单列、表格可横向滚动。
