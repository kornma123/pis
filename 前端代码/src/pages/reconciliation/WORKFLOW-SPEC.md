# Wave D 对账与报表工作流规格

## 范围与事实源

- 路由保持不变：`/account-reconcile`、`/reconciliation`、`/cost-analysis`。
- 功能事实只来自现有 API：账实核对、消耗对账、成本报表；不新增后端端点，不把本地状态当业务事件。
- 导出只使用当前已成功加载的前端事实集生成 CSV；筛选仍在加载、失败或陈旧时禁止导出。

## 信息层级与主任务

1. 选择月份 / 日期期间与对象；期间、页签、筛选和分页写入 URL，回退后保留上下文。
2. 加载来源事实；并行加载互不依赖的请求，旧请求不得覆盖新期间。
3. 定位差异；先给汇总，再给可展开的项目 / 医院 / 明细。
4. 展示证据与处理状态；LIS / 手工来源、未认账口径、待审核 / 已生效、只读权限均与数字或动作同视线。
5. 导出或复核；只开放真实 API 动作和真实 CSV，成功提示必须晚于服务端成功或文件生成。

## 状态机

- `loading`：隐藏旧数值和写操作，显示可感知的加载状态。
- `ready + data`：展示本次期间事实与处理动作。
- `ready + empty`：明确“本期无记录”，给出已有能力范围内的下一步。
- `error`：不伪装空态，保留重试入口；网络未连接与服务端失败使用不同文案。
- `stale`：仅允许回看上次成功结果，显式标注且禁用写入 / 导出。
- `unknown`：缺失、未连接或不可计算显示文字，不折成 0；真实 0 保持为 0。
- `permission`：无读权限显示无权限页；有读无写显示只读说明，不渲染写按钮。

## 组件、键盘与响应式

- 页签使用 `tablist/tab/tabpanel`，左右方向键可切换；展开项目使用原生按钮和 `aria-expanded`。
- 表格保留表格语义、列标题与说明文字；窄屏横向滚动，工具栏可换行，不把表格挤成卡片。
- 对话框具备名称、`aria-modal`、Escape 关闭与明确忙碌态；提交期间防重复。
- 长列表行启用 `content-visibility`；不在组件内部定义组件，不用 effect 保存可派生状态。

## 明确不做

- 不改变全局导航、视觉体系、共享组件、API、权限库或后端。
- 不实现当前 API 未提供的公共成本、病例成本明细、合作状态、同步时间或后端导出。
- 不把 PRD-1A 的待建月结子账本说成现有能力，不宣称上线或真实业务验收。
