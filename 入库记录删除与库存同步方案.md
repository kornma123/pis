# 入库记录删除与库存同步方案

## 问题背景

当前系统删除入库记录时，仅将 `inbound_records.is_deleted` 标记为 1，但 `inventory` 表的 `stock` 字段未同步扣减，导致库存列表数据与入库记录不一致。

## 核心矛盾

1. **数据一致性要求**：删除入库记录应同步扣减库存
2. **业务约束**：已使用的库存（出库、锁定、使用中）不能随意回退

---

## 推荐方案：限制删除条件 + 软删除 + 动态库存计算

### 1. 前端交互设计

**删除确认弹窗**（分两种情况）：

**情况A - 库存可删除：**
```
┌─────────────────────────────────────────┐
│  ⚠️ 删除确认                            │
│                                         │
│  删除入库记录 [IB-20260511-001]          │
│  将同步扣减库存：                        │
│    • 一次性手套 × 100 个                │
│    • 当前库存将变为：212.5 个            │
│                                         │
│  [取消]        [确认删除]               │
└─────────────────────────────────────────┘
```

**情况B - 库存不可删除：**
```
┌─────────────────────────────────────────┐
│  ❌ 无法删除                            │
│                                         │
│  该入库记录对应的库存已被使用/锁定，     │
│  无法直接删除。                          │
│                                         │
│  原因：当前有 50 个库存处于锁定状态      │
│                                         │
│  建议操作：                              │
│  • 先完成/取消关联的出库单              │
│  • 或使用「库存盘点」功能调整库存       │
│                                         │
│  [我知道了]                             │
└─────────────────────────────────────────┘
```

### 2. 后端校验逻辑（删除前）

```
删除入库记录流程：

1. 查询入库记录 → 获取 material_id、quantity
2. 查询 inventory 表 → 获取 stock、locked_stock
3. 查询 outbound_items → 检查该物料是否有出库记录
4. 查询 batch_usage_tracking → 检查是否有"使用中"记录
5. 判断条件：
   
   IF (locked_stock > 0) → 拒绝删除，提示"库存已被锁定"
   
   IF (存在出库记录) → 拒绝删除，提示"库存已有出库记录"
   
   IF (存在使用中记录) → 拒绝删除，提示"库存正在使用中"
   
   IF (stock < quantity) → 拒绝删除，提示"可用库存不足"
   
   ELSE → 允许删除：
     a. 扣减 inventory.stock
     b. 标记 inbound_records.is_deleted = 1
     c. 记录操作日志
```

### 3. 库存计算方式调整（已实施）

将 `inventory.stock` 从"物理存储字段"改为"实时计算字段"：

```sql
-- 库存数量 = 所有有效入库记录数量之和
SELECT COALESCE(SUM(quantity), 0) as stock
FROM inbound_records
WHERE material_id = ? AND status = 'completed' AND is_deleted = 0
```

**优势**：
- 数据强一致性，无需担心入库记录删除后库存不同步
- 保留完整的审计痕迹（所有入库记录都保留，仅标记删除）

### 4. 特殊情况处理

| 场景 | 处理方式 | 说明 |
|------|---------|------|
| 库存完全在库 | 直接删除 + 扣减库存 | 无业务关联，安全删除 |
| 库存部分锁定 | 拒绝删除 | 需先解锁或完成出库 |
| 库存已出库 | 拒绝删除 | 入库记录已成为成本依据，不可删除 |
| 库存使用中 | 拒绝删除 | 需先确认耗尽或退回 |
| 库存为负 | 拒绝删除 | 数据异常，需人工核查 |

### 5. 备选方案

如果业务上确实需要删除"已使用"的入库记录，提供以下替代方案：

**方案A：库存盘点冲销**
- 不删除入库记录
- 通过「库存盘点」功能创建一条负向调整记录
- 库存扣减，但保留原始入库记录作为审计痕迹

**方案B：退库入库**
- 创建一条退库类型的入库记录（数量为负）
- 对冲原始入库，保持账务平衡
