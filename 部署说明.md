# COREONE 部署指南

> 适用于公司 Linux 服务器的 Docker 部署

## 服务器要求

- **OS**: Linux (Ubuntu 22.04+ / CentOS 8+ / Debian 12+)
- **Docker**: 20.10+
- **Docker Compose**: 2.0+
- **端口**: 1 个可用端口（默认 8080，可自定义）
- **内存**: 最低 512MB，建议 1GB+
- **磁盘**: 1GB+（含 SQLite 数据库增长空间）

## 快速部署（3 步）

### 1. 上传代码到服务器

```bash
# 在本地打包项目（Windows 用 PowerShell / Git Bash；必须在 git 仓库根执行）
cd d:/Git/COREONE/最新代码
# 用 git archive 打包：只含**已跟踪**文件 → 自动排除未跟踪的 .env / token.json / 本地 .db
# 等敏感文件（tar -c . 会把它们一起打包，这是旧命令的泄露点）；仓库根 .gitattributes 的
# export-ignore 再排除测试/调试/种子脚本。
git archive --worktree-attributes -o coreone-deploy.tar.gz HEAD \
  前端代码 后端代码 docker-compose.yml 部署说明.md
# 校验包内不含敏感文件（下面这行应输出“✅ 无敏感文件”）：
tar tzf coreone-deploy.tar.gz | grep -Eq '(^|/)\.env$|token\.json|\.db$' \
  && echo '⚠️ 敏感文件仍在包内，勿上传！' || echo '✅ 无敏感文件'

# 上传到服务器（替换为你的服务器地址）
scp coreone-deploy.tar.gz root@your-server-ip:/opt/
```

### 2. 服务器端解压并启动

```bash
ssh root@your-server-ip

# 解压
cd /opt && mkdir -p coreone && tar -xzvf coreone-deploy.tar.gz -C coreone --strip-components=0
cd coreone

# ⚠️ 启动前必须注入强随机 JWT_SECRET（缺失则容器直接拒绝启动，这是有意的安全保险）：
export JWT_SECRET="$(openssl rand -base64 48)"
# 可选：受控创建初始管理员（不设则不创建任何默认账号）。口令须 ≥12 位、且非默认弱口令：
export ADMIN_INITIAL_PASSWORD="<设一个强口令>"

# 启动（首次会自动拉取 node:22-alpine 和 nginx:alpine 镜像）
docker compose up -d --build

# 查看日志
docker compose logs -f
```

### 3. 访问系统

打开浏览器访问 `http://your-server-ip:8080`

登录：**系统不再内置默认账号/默认口令**（旧默认 `admin/admin123` 曾泄露，已移除）。
- 若启动前设置了 `ADMIN_INITIAL_PASSWORD`：用用户名 `admin` + 该口令登录，并**立即改密**。
- 若未设置：暂时注入 `ADMIN_INITIAL_PASSWORD` 并重建 backend 一次来创建 admin；
  创建后删掉该变量并再次重建 backend，避免初始口令长期留在容器环境中。
  `reset-passwords` **只更新已存在账号**，不会创建 admin。

## 常用运维命令

```bash
# 查看运行状态
docker compose ps

# 查看日志
docker compose logs -f backend   # 后端日志
docker compose logs -f frontend  # 前端日志

# 重启服务
docker compose restart

# 停止服务
docker compose down

# 完全删除（含数据库！慎用）
docker compose down -v

# 更新代码后重新构建
docker compose up -d --build
```

## 配置说明

### 修改端口号

编辑 `docker-compose.yml` 或创建 `.env` 文件：

```bash
# 创建 .env 文件
echo "COREONE_PORT=3000" > .env

# 重新启动
docker compose up -d
```

### 修改 JWT Secret（生产环境必须改！）

```bash
# 创建 .env 文件
echo "JWT_SECRET=your-random-secret-key-here-$(openssl rand -hex 16)" > .env

# 重新启动
docker compose up -d
```

### 重置已存在账号口令

```bash
# 输入不回显，口令不进 shell 历史。
read -rsp "New admin password: " RESET_ADMIN_PASSWORD && echo
export RESET_ADMIN_PASSWORD

# 临时容器共享生产 volume；--rm 使它执行后立即删除。
docker compose run --rm --no-deps \
  -e RESET_ADMIN_PASSWORD \
  -e DATABASE_PATH=/app/data/coreone.db \
  backend npm run reset-passwords

unset RESET_ADMIN_PASSWORD
```

### 数据库备份

SQLite 数据存储在 Docker volume `coreone-data` 中：

```bash
# 备份数据库
docker cp coreone-backend:/app/data/coreone.db ./backup/coreone-$(date +%Y%m%d).db

# 恢复数据库（先停止服务）
docker compose down
docker cp ./backup/coreone-20260101.db coreone-backend:/app/data/coreone.db
docker compose up -d
```

## 部署架构

```
公司服务器 (Linux)
|
|-- docker-compose.yml
|
|-- coreone-frontend (Nginx 容器)
|   |-- 端口: 8080 -> 80
|   |-- 托管前端静态文件 (dist/)
|   |-- /api/v1/* 反向代理到后端
|
|-- coreone-backend (Node.js 22 容器)
|   |-- 端口: 3001 (仅内部网络)
|   |-- 运行后端 API (tsx src/app.ts)
|   |-- SQLite 数据库 (/app/data/coreone.db)
|
|-- Docker Volume: coreone-data
    |-- 持久化 SQLite 数据库文件
```

## 故障排查

| 现象 | 排查方法 |
|------|----------|
| 页面空白 404 | `docker compose logs frontend` 检查 Nginx 配置 |
| API 请求失败 | `docker compose logs backend` 检查后端是否启动 |
| 数据库权限错误 | `docker exec coreone-backend ls -la /app/data` |
| 端口被占用 | `netstat -tlnp \| grep 8080` 修改 `COREONE_PORT` |
| 容器无法启动 | `docker compose up`（不加 -d）查看实时错误 |

## 安全建议（生产环境）

1. **必须修改 JWT_SECRET** — 不要用默认值
2. **配置防火墙** — 只开放必要的端口
3. **启用 HTTPS** — 使用 Nginx + Let's Encrypt 证书
4. **定期备份数据库** — 建议每天自动备份
5. **限制 CORS** — 当前 `app.ts` 中 `cors()` 是开放配置，生产环境应限制为前端域名
