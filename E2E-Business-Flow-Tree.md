# COREONE 业务流程树与 E2E 场景

> **要求**: 至少 20 条主流程，每条流程含主路径 + 多条分支路径，每个分支对应一个完整 E2E 场景（从登录到结束）  
> **角色**: 以 warehouse_manager 为主视角，涉及权限越权时切换角色  
> **分支类型**: 中途取消、关闭弹窗、返回上一步、刷新页面状态保持、必填漏填、网络失败重试、业务冲突、权限不足

---

## 流程1: 采购入库流

```
采购入库流
├── 主路径: 登录 → 进入采购订单 → 创建采购订单 → 审批 → 收货入库 → 批次生成 → 库存增加 → 查看库存确认
├── 分支1: 创建采购订单后中途取消
├── 分支2: 收货入库时关闭弹窗不保存
├── 分支3: 收货入库必填漏填（未选物料）
├── 分支4: 收货数量超过订单数量
├── 分支5: 入库提交时网络中断，重试后成功
├── 分支6: 刷新页面后采购订单状态保持
├── 分支7: 已完成订单尝试取消（业务冲突）
├── 分支8: 有出库记录的入库单尝试删除（业务冲突）
└── 分支9: technician 角色尝试创建入库单（权限不足）
```

| 路径 | E2E 场景 (Given-When-Then) |
|:---|:---|
| 主路径 | Given 仓库管理员已登录系统，When 依次执行：1) 进入采购订单页 2) 点击"新增采购单" 3) 选择物料苏木素染液/数量100/单价180/供应商DAKO 4) 提交创建(status=pending) 5) 点击"收货"输入数量100 6) 系统自动创建入库单IB-xxx 7) 查看库存列表确认stock增加100，Then 采购订单状态变为completed，入库单出现在入库列表，库存实时更新，批次表中新增批次记录，stock_logs生成inbound流水 |
| 分支1-中途取消 | Given 仓库管理员已登录，创建了采购订单PO-20260513-0001(status=pending)，When 点击"取消订单"并确认，Then 订单状态变为cancelled，该订单不再出现在"待收货"筛选中，已取消订单可在全部列表查看 |
| 分支2-关闭弹窗不保存 | Given 仓库管理员在入库登记弹窗中已填写物料/数量/批次号，When 点击"关闭"按钮（或ESC/遮罩），Then 弹窗关闭，不创建入库记录，库存不变，页面停留在入库列表 |
| 分支3-必填漏填 | Given 仓库管理员打开入库登记弹窗，When 不填物料直接点击提交，Then 前端校验阻止提交，显示"物料为必填项"红色提示，提交按钮保持可用状态，无API请求发出 |
| 分支4-收货超量 | Given 采购订单orderedQty=100，已收货0，When 仓库管理员输入收货数量120，Then 系统提示"入库数量超过订单数量"，阻止提交，receivedQty保持0 |
| 分支5-网络中断重试 | Given 仓库管理员点击入库提交，When 模拟网络中断(API返回500)，Then 显示"提交失败，请重试"Toast，提交按钮恢复可用；When 网络恢复后再次点击提交，Then 入库成功，库存正确增加（避免重复） |
| 分支6-刷新页面状态保持 | Given 仓库管理员创建了采购订单(status=partial，已收30/100)，When 刷新浏览器页面，Then 页面重新加载后订单状态仍为partial，receivedQty=30，分页位置保持 |
| 分支7-已完成订单取消 | Given 采购订单状态为completed，When 仓库管理员尝试取消，Then 系统提示"已完成的订单不能取消"，返回400，订单状态不变 |
| 分支8-有出库记录删除入库单 | Given 入库单IB-001已有关联出库记录，When 仓库管理员尝试删除该入库单，Then 系统提示"该批次已有出库记录，不可删除"，返回400，入库单状态不变 |
| 分支9-权限不足 | Given 病理技术员已登录，When 尝试访问入库记录页或直接调用POST /api/v1/inbound，Then 前端路由守卫拦截/后端返回403，页面显示"无权访问"，不显示"新增入库"按钮 |

## 流程2: 项目领用出库流（BOM一键出库）

```
项目领用出库流
├── 主路径: 登录 → 进入出库记录 → 新增出库 → 选择项目 → 选择BOM → 输入样本数 → 系统自动计算物料需求 → FIFO分配批次 → 扣减库存 → 成本归集到项目
├── 分支1: 关闭出库弹窗不保存
├── 分支2: 未选项目直接提交
├── 分支3: 库存不足（某BOM物料库存不够）
├── 分支4: 网络中断后重试
├── 分支5: 刷新页面后出库列表状态保持
├── 分支6: 选择项目后切换BOM
├── 分支7: 输入样本数为0
├── 分支8: 出库后取消入库单（业务冲突）
└── 分支9: procurement角色尝试出库（权限不足）
```

| 路径 | E2E 场景 (Given-When-Then) |
|:---|:---|
| 主路径 | Given 病理技术员已登录，库存充足，When 依次执行：1) 进入出库记录页 2) 点击"新增出库" 3) 选择项目"HE常规制片" 4) 选择BOM"HE制片标准套装" 5) 输入样本数10 6) 系统显示物料需求：苏木素5ml、伊红3ml... 7) 确认提交，Then 出库单OB-xxx创建成功，库存按FIFO扣减，项目成本统计中totalCost增加，stock_logs生成outbound流水 |
| 分支1-关闭弹窗 | Given 技术员在出库弹窗中已选择项目和BOM，When 点击关闭按钮，Then 弹窗关闭，不创建出库记录，库存不变 |
| 分支2-未选项目 | Given 技术员打开出库弹窗，When 不选项目直接点击提交，Then 前端校验阻止，提示"请关联检测项目" |
| 分支3-库存不足 | Given 某BOM物料（如苏木素）库存仅3ml，需求5ml，When 技术员确认提交，Then 返回422 "Insufficient stock"，整单拒绝，所有物料均未出库，库存不变 |
| 分支4-网络中断重试 | Given 技术员点击出库提交，When 模拟网络中断，Then 显示提交失败Toast；恢复网络后重试，Then 出库成功，库存正确扣减 |
| 分支5-刷新页面状态保持 | Given 技术员刚创建出库单，When 刷新页面，Then 新出库单仍在列表顶部，状态为completed，分页保持 |
| 分支6-切换BOM | Given 技术员选择项目后选了BOM-A，When 重新选择BOM-B，Then 物料需求列表自动更新为BOM-B的物料清单，旧选择清空 |
| 分支7-样本数为0 | Given 技术员输入样本数0，When 提交，Then 系统提示"样本数必须大于0"，前端校验阻止 |
| 分支8-出库后取消入库 | Given 出库已成功（库存已扣减），When 仓库管理员尝试取消关联的入库单，Then 系统提示"已有出库记录，不可取消"，阻止操作 |
| 分支9-权限不足 | Given 采购专员已登录，When 尝试访问出库记录页，Then 403/无菜单入口，不显示"新增出库"按钮 |

## 流程3: 库存盘点流

```
库存盘点流
├── 主路径: 登录 → 进入库存盘点 → 新建盘点单 → 选择物料 → 输入实盘数量 → 系统计算差异 → 确认调整 → 库存更新 → 生成盘点差异报告
├── 分支1: 关闭盘点弹窗不保存
├── 分支2: 未选物料提交
├── 分支3: 实盘数量=系统数量（无差异）
├── 分支4: 实盘数量>系统数量（盘盈）
├── 分支5: 实盘数量<系统数量（盘亏）
├── 分支6: 网络中断后重试
├── 分支7: 刷新页面后盘点状态保持
├── 分支8: 确认调整时取消
└── 分支9: technician角色尝试盘点（权限不足）
```

| 路径 | E2E 场景 (Given-When-Then) |
|:---|:---|
| 主路径 | Given 仓库管理员已登录，苏木素系统库存10，When 依次执行：1) 进入库存盘点 2) 新建盘点单 3) 选择苏木素 4) 输入实盘8 5) 系统显示差异-2 6) 确认调整，Then 库存更新为8，stock_logs生成adjust流水，盘点单状态completed，盘点差异报告显示盘亏2瓶 |
| 分支1-关闭弹窗 | Given 仓库管理员在盘点弹窗中已选物料和实盘数量，When 点击关闭，Then 不创建盘点单，库存不变 |
| 分支2-未选物料 | Given 打开盘点弹窗，When 直接点击提交，Then 前端校验阻止，提示"请选择物料" |
| 分支3-无差异 | Given 苏木素系统库存10，When 输入实盘10，Then 差异=0，库存不变，不生成stock_logs adjust记录，盘点单状态completed |
| 分支4-盘盈 | Given 苏木素系统库存10，When 输入实盘12，Then 差异=+2，库存更新为12，stock_logs显示adjust +2，before_stock=10 after_stock=12 |
| 分支5-盘亏 | Given 苏木素系统库存10，When 输入实盘5，Then 差异=-5，库存更新为5，stock_logs显示adjust -5，触发低库存预警 |
| 分支6-网络中断重试 | Given 仓库管理员确认盘点调整，When 网络中断，Then 提交失败；恢复后重试，Then 库存正确调整，避免重复 |
| 分支7-刷新状态保持 | Given 盘点单刚创建(status=completed)，When 刷新页面，Then 盘点单仍在列表，状态不变 |
| 分支8-确认时取消 | Given 盘点弹窗显示差异-2，When 点击"取消"不确认调整，Then 弹窗关闭，库存不变，不创建盘点记录 |
| 分支9-权限不足 | Given 技术员已登录，When 尝试访问库存盘点页，Then 403/无菜单入口 |

## 流程4: 物料新增流

```
物料新增流
├── 主路径: 登录 → 进入耗材管理 → 新增物料 → 填写编码/名称/规格/分类/单位/安全库存 → 提交 → 创建inventory记录(stock=0) → 物料列表刷新
├── 分支1: 关闭弹窗不保存
├── 分支2: 编码已存在
├── 分支3: 必填字段漏填（名称/单位/分类）
├── 分支4: 价格输入负数
├── 分支5: 网络中断后重试
├── 分支6: 刷新页面后新物料仍在列表
├── 分支7: 删除有库存的物料
├── 分支8: 批量禁用物料
└── 分支9: technician角色尝试新增物料（权限不足）
```

| 路径 | E2E 场景 (Given-When-Then) |
|:---|:---|
| 主路径 | Given 管理员已登录，When 依次执行：1) 进入耗材管理 2) 点击"新增物料" 3) 填写code=REA-NEW-001/name=测试染液/unit=瓶/categoryId=试剂类 4) 提交，Then 创建成功，物料列表新增记录，inventory表自动创建stock=0记录，分类下拉同步更新 |
| 分支1-关闭弹窗 | Given 弹窗中已填编码和名称，When 点击关闭，Then 不创建物料，列表不变 |
| 分支2-编码已存在 | Given 编码REA-00001已存在，When 输入相同编码提交，Then 返回409 "Code already exists"，前端显示红色提示 |
| 分支3-必填漏填 | Given 弹窗中未填名称，When 提交，Then 前端校验阻止，提示"物料名称为必填项" |
| 分支4-价格负数 | Given 输入price=-10，When 提交，Then 后端未显式拦截可能创建成功（前端应限制），或返回400 |
| 分支5-网络中断重试 | Given 点击提交时网络中断，When 恢复后重试，Then 创建成功，检查编码唯一性避免重复 |
| 分支6-刷新后存在 | Given 新物料刚创建，When 刷新页面，Then 新物料仍在列表，分页正确 |
| 分支7-删除有库存 | Given 物料stock=10，When 尝试删除，Then 返回409 "Stock exists"，阻止删除 |
| 分支8-批量禁用 | Given 选择3个物料，When 批量禁用，Then 3个物料status变为inactive，列表状态标签更新 |
| 分支9-权限不足 | Given 技术员已登录，When 尝试访问耗材管理页，Then 403/无菜单，不显示"新增物料"按钮 |

## 流程5: 预警处理流

```
预警处理流
├── 主路径: 登录 → 进入预警中心 → 查看待处理预警 → 点击处理 → 选择处理方式 → 填写备注 → 提交 → 状态变为handled → 预警历史同步更新
├── 分支1: 关闭处理弹窗不保存
├── 分支2: 未填写备注提交
├── 分支3: 处理不存在的预警
├── 分支4: 重复处理同一预警
├── 分支5: 批量处理多条预警
├── 分支6: 网络中断后重试
├── 分支7: 刷新页面后预警状态保持
├── 分支8: 禁用规则后不再生成该类型预警
└── 分支9: 低库存预警触发采购入库（跨流程联动）
```

| 路径 | E2E 场景 (Given-When-Then) |
|:---|:---|
| 主路径 | Given 仓库管理员已登录，系统存在低库存预警（苏木素stock=3, safety=5），When 依次执行：1) 进入预警中心 2) 查看pending预警 3) 点击处理 4) 选择action=handled 5) 填写remark="已安排采购" 6) 提交，Then 预警状态变为handled，handled_at有值，预警历史列表出现新记录 |
| 分支1-关闭弹窗 | Given 处理弹窗已打开，When 点击关闭，Then 预警状态不变，仍为pending |
| 分支2-无备注 | Given 处理弹窗中action已选，When 不填备注直接提交，Then 后端允许，remark为空字符串，状态变为handled |
| 分支3-不存在预警 | Given 输入不存在的预警ID，When 调用POST /alerts/xxx/handle，Then 返回404 "Not found" |
| 分支4-重复处理 | Given 预警已handled，When 再次提交处理，Then 返回200，状态更新为新的action |
| 分支5-批量处理 | Given 选择3条pending预警，When 批量处理，Then 3条预警状态均变为handled，预警历史同步更新 |
| 分支6-网络中断重试 | Given 点击处理提交，When 网络中断，Then 提交失败；恢复后重试，Then 处理成功，状态变更 |
| 分支7-刷新状态保持 | Given 预警刚被处理，When 刷新页面，Then 预警中心该预警显示handled，预警历史出现记录 |
| 分支8-禁用规则 | Given 禁用低库存预警规则，When 系统扫描生成预警，Then 不生成低库存预警，仅生成其他类型 |
| 分支9-跨流程联动 | Given 低库存预警已处理（已安排采购），When 采购入库完成后stock>safety，Then 再次扫描时不重复生成该物料的低库存预警 |

## 流程6: 退货入库流

```
退货入库流
├── 主路径: 登录 → 进入退货管理 → 新增退货 → 选择原出库单 → 选择物料 → 输入数量 → 选择原因 → 提交 → 库存扣减
├── 分支1: 关闭弹窗不保存
├── 分支2: 未关联原出库单
├── 分支3: 退货数量超过原出库数量
├── 分支4: 网络中断后重试
├── 分支5: 刷新页面后退货单状态保持
├── 分支6: 退货后取消（不支持的反向操作）
├── 分支7: 退货数量为0
└── 分支8: technician角色尝试退货（权限不足）
```

| 路径 | E2E 场景 (Given-When-Then) |
|:---|:---|
| 主路径 | Given 仓库管理员已登录，原出库单OB-001出库苏木素5瓶，When 依次执行：1) 进入退货管理 2) 新增退货 3) 关联原出库单 4) 选择苏木素 5) 输入数量3 6) 原因"质量不合格" 7) 提交，Then 退货单创建成功，库存扣减3，原出库单显示退货关联 |
| 分支1-关闭弹窗 | Given 弹窗中已选原出库单和物料，When 点击关闭，Then 不创建退货单 |
| 分支2-未关联出库单 | Given 未选择原出库单，When 提交，Then 前端允许（后端不强制关联），创建退货单，库存扣减 |
| 分支3-超量退货 | Given 原出库数量5，When 输入退货数量8，Then 系统提示"退货数量超过原出库数量"，阻止提交 |
| 分支4-网络中断 | Given 提交时网络中断，When 恢复后重试，Then 退货成功，库存正确扣减 |
| 分支5-刷新状态 | Given 退货单刚创建，When 刷新页面，Then 退货单仍在列表，状态正常 |
| 分支6-取消退货 | Given 退货单已创建，When 尝试取消/删除，Then 系统可能不支持退货取消，或软删除退货单但库存不回退（取决于实现） |
| 分支7-数量为0 | Given 输入退货数量0，When 提交，Then 后端可能未拦截，创建无意义记录 |
| 分支8-权限不足 | Given 技术员已登录，When 尝试访问退货管理页，Then 403/无菜单入口 |

## 流程7: 调拨出库流

```
调拨出库流
├── 主路径: 登录 → 进入出库记录 → 新增出库 → 选择调拨类型 → 选择物料 → 选择来源库位 → 选择目标库位 → 输入数量 → 提交 → 来源库位减/目标库位增
├── 分支1: 关闭弹窗不保存
├── 分支2: 未选目标库位
├── 分支3: 来源库位库存不足
├── 分支4: 来源库位=目标库位
├── 分支5: 网络中断后重试
├── 分支6: 刷新页面后调拨单状态保持
├── 分支7: 调拨数量为0
```
