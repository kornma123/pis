# 入库记录删除校验与数据同步优化方案

## 问题背景

当前系统删除入库记录时存在以下问题：

1. **缺少前置校验**：删除入库记录前未检查该记录对应的库存是否已被使用（出库、锁定、关联使用中），导致删除后下游数据（出库记录、成本归集、项目消耗跟踪）出现悬空引用
2. **inventory.stock 已弃用**：库存列表已改为从 `inbound_records` 实时计算，`inventory.stock` 字段不再作为数据源，当前 `DELETE /inbound/:id` 中扣减 `inventory.stock` 的逻辑已无实际意义
3. **采购订单未回退**：关联采购订单的 `received_qty` 未回退，导致采购订单显示已收货但实际入库记录已被删除
4. **批次记录未处理**：`batches` 表中对应的批次数量未同步扣减

---

## 优化方案

### 一、删除前置校验规则

```
删除入库记录流程：

1. 查询入库记录 → 获取 material_id, batch_no, quantity, purchase_order_id
2. 校验条件链（按优先级）：

   a. 该入库记录是否已被取消（status = 'cancelled'）
      → 是：可直接删除，无库存影响

   b. 是否存在关联的出库记录？
      → SELECT * FROM outbound_items WHERE material_id = ? AND batch_no = ?
      → 存在：拒绝删除，提示"该批次已有出库记录，不可删除"

   c. 是否存在锁定/使用中记录？
      → SELECT * FROM depletion_tracking WHERE material_id = ? AND batch = ? AND status = 'in_use'
      → 存在：拒绝删除，提示"该批次库存正在使用中，请先确认耗尽"

   d. 删除后该物料+批次库存是否为负？
      → 计算：SUM(inbound_records.quantity WHERE is_deleted=0 AND status='completed') - SUM(outbound_items.quantity)
      → 为负：拒绝删除，提示"删除后库存将变为负数，请先处理下游出库"

3. 全部通过 → 允许删除
```

### 二、删除后的数据同步

```typescript
router.delete('/:id', (req, res) => {
  const db = getDatabase()
  const record = db.prepare('SELECT * FROM inbound_records WHERE id = ?').get(req.params.id) as any
  if (!record) { error(res, '记录不存在'); return }
  
  // 前置校验
  if (record.status === 'completed') {
    // 1. 检查是否有出库记录
    const outboundExists = db.prepare(
      'SELECT 1 FROM outbound_items WHERE material_id = ? AND batch_no = ? LIMIT 1'
    ).get(record.material_id, record.batch_no)
    if (outboundExists) {
      error(res, '该入库记录对应的批次已有出库记录，不可删除。请先作废关联的出库单', 'BUSINESS_RULE', 400)
      return
    }
    
    // 2. 检查是否有使用中的消耗跟踪
    const inUseExists = db.prepare(
      "SELECT 1 FROM depletion_tracking WHERE material_id = ? AND batch = ? AND status = 'in_use' LIMIT 1"
    ).get(record.material_id, record.batch_no)
    if (inUseExists) {
      error(res, '该批次库存正在使用中，请先确认耗尽后再删除', 'BUSINESS_RULE', 400)
      return
    }
    
    // 3. 检查删除后库存是否为负
    const totalInbound = (db.prepare(
      "SELECT COALESCE(SUM(quantity),0) as total FROM inbound_records WHERE material_id = ? AND batch_no = ? AND status = 'completed' AND is_deleted = 0 AND id != ?"
    ).get(record.material_id, record.batch_no, record.id) as any)?.total || 0
    
    const totalOutbound = (db.prepare(
      'SELECT COALESCE(SUM(quantity),0) as total FROM outbound_items WHERE material_id = ? AND batch_no = ?'
    ).get(record.material_id, record.batch_no) as any)?.total || 0
    
    if (totalInbound < totalOutbound) {
      error(res, `删除后该批次库存将变为负数（当前有效入库 ${totalInbound}，已出库 ${totalOutbound}），不可删除`, 'BUSINESS_RULE', 400)
      return
    }
    
    // 4. 回退采购订单收货数量
    if (record.purchase_order_id) {
      const orderItem = db.prepare(
        'SELECT * FROM purchase_order_items WHERE order_id = ? AND material_id = ?'
      ).get(record.purchase_order_id, record.material_id)
      if (orderItem) {
        const newReceived = Math.max(0, Number(orderItem.received_qty) - record.quantity)
        const itemStatus = newReceived === 0 ? 'pending' : 'partial'
        db.prepare('UPDATE purchase_order_items SET received_qty = ?, status = ? WHERE id = ?')
          .run(newReceived, itemStatus, orderItem.id)
        
        // 更新订单头状态
        const allItems = db.prepare('SELECT status FROM purchase_order_items WHERE order_id = ?')
          .all(record.purchase_order_id) as any[]
        const orderStatus = allItems.every((i: any) => i.status === 'pending') ? 'pending' : 'partial'
        db.prepare("UPDATE purchase_orders SET status = ? WHERE id = ?")
          .run(orderStatus, record.purchase_order_id)
      }
    }
    
    // 5. 扣减批次数量
    if (record.batch_no) {
      db.prepare('UPDATE batches SET quantity = quantity - ?, remaining = remaining - ? WHERE material_id = ? AND batch_no = ?')
        .run(record.quantity, record.quantity, record.material_id, record.batch_no)
      
      // 如果批次剩余为0，标记为无效
      const batch = db.prepare('SELECT remaining FROM batches WHERE material_id = ? AND batch_no = ?')
        .get(record.material_id, record.batch_no) as any
      if (batch && batch.remaining <= 0) {
        db.prepare('UPDATE batches SET status = 0 WHERE material_id = ? AND batch_no = ?')
          .run(record.material_id, record.batch_no)
      }
    }
  }
  
  // 6. 软删除入库记录
  db.prepare('UPDATE inbound_records SET is_deleted = 1, updated_at = CURRENT_TIMESTAMP WHERE id = ?')
    .run(req.params.id)
  
  // 7. 记录操作日志
  const logId = uuidv4()
  db.prepare(`
    INSERT INTO stock_logs (id, type, material_id, quantity, before_stock, after_stock, related_id, related_type, operator, remark)
    VALUES (?, 'delete', ?, ?, ?, ?, ?, 'inbound_delete', ?, '删除入库记录')
  `).run(logId, record.material_id, record.quantity, totalInbound + record.quantity, totalInbound, req.params.id, req.body.operator || 'system')
  
  success(res, null, '删除成功，库存已同步扣减')
})
```

### 三、前端交互优化

**删除确认弹窗（分两种情况）：**

**情况A - 可直接删除：**
```
┌─────────────────────────────────────────┐
│  ⚠️ 删除确认                            │
│                                         │
│  删除入库记录 [IB-20260511-001]          │
│                                         │
│  物料：一次性手套 × 100 个              │
│  批次：23132                            │
│                                         │
│  删除后将同步：                          │
│    • 扣减库存 100 个                    │
│    • 回退采购订单收货数量               │
│    • 作废关联批次记录                   │
│                                         │
│  [取消]        [确认删除]               │
└─────────────────────────────────────────┘
```

**情况B - 无法删除：**
```
┌─────────────────────────────────────────┐
│  ❌ 无法删除                            │
│                                         │
│  该入库记录不可删除                      │
│                                         │
│  原因：该批次已有出库记录（50个）        │
│                                         │
│  建议操作：                              │
│  • 先作废关联的出库单                   │
│  • 或使用库存盘点功能调整               │
│                                         │
│  [我知道了]                             │
└─────────────────────────────────────────┘
```

**前端删除按钮点击流程：**
```typescript
const handleDelete = async (record: InboundRecord) => {
  // 先调用校验接口（或本地预校验）
  const { canDelete, reason, details } = await inboundApi.checkDeletable(record.id)
  
  if (!canDelete) {
    // 显示情况B弹窗
    setDeleteBlockedReason(reason)
    setDeleteBlockedDetails(details)
    setDeleteBlockedModalOpen(true)
    return
  }
  
  // 显示情况A确认弹窗
  setDeleteRecord(record)
  setDeleteConfirmModalOpen(true)
}
```

### 四、新增 API 接口

```typescript
// GET /inbound/:id/check-deletable
router.get('/:id/check-deletable', (req, res) => {
  const db = getDatabase()
  const record = db.prepare('SELECT * FROM inbound_records WHERE id = ?').get(req.params.id) as any
  if (!record) { error(res, '记录不存在', 'NOT_FOUND', 404); return }
  
  const reasons: string[] = []
  let canDelete = true
  
  if (record.status === 'completed') {
    const outboundExists = db.prepare(
      'SELECT COALESCE(SUM(quantity),0) as total FROM outbound_items WHERE material_id = ? AND batch_no = ?'
    ).get(record.material_id, record.batch_no) as any
    if (outboundExists && outboundExists.total > 0) {
      canDelete = false
      reasons.push(`该批次已有出库记录 ${outboundExists.total} ${record.unit}`)
    }
    
    const inUseExists = db.prepare(
      "SELECT 1 FROM depletion_tracking WHERE material_id = ? AND batch = ? AND status = 'in_use' LIMIT 1"
    ).get(record.material_id, record.batch_no)
    if (inUseExists) {
      canDelete = false
      reasons.push('该批次库存正在使用中')
    }
    
    const totalInbound = (db.prepare(
      "SELECT COALESCE(SUM(quantity),0) as total FROM inbound_records WHERE material_id = ? AND batch_no = ? AND status = 'completed' AND is_deleted = 0 AND id != ?"
    ).get(record.material_id, record.batch_no, record.id) as any)?.total || 0
    const totalOutbound = outboundExists?.total || 0
    if (totalInbound < totalOutbound) {
      canDelete = false
      reasons.push(`删除后库存将变为负数（剩余 ${totalInbound}，已出库 ${totalOutbound}）`)
    }
  }
  
  success(res, { canDelete, reasons, record: { name: record.material_name, quantity: record.quantity, batch: record.batch_no } })
})
```

### 五、数据一致性保障

| 操作 | 同步内容 | 说明 |
|------|---------|------|
| 删除入库记录 | `inbound_records.is_deleted = 1` | 软删除，保留审计痕迹 |
| 删除入库记录 | 回退 `purchase_order_items.received_qty` | 采购订单数据同步 |
| 删除入库记录 | 扣减 `batches.remaining` | 批次库存同步 |
| 删除入库记录 | 写入 `stock_logs` | 操作审计 |
| 库存列表查询 | 实时计算 `SUM(inbound_records.quantity WHERE is_deleted=0)` | 数据强一致 |

### 六、实施优先级

| 优先级 | 内容 | 工作量 |
|--------|------|--------|
| P0 | 删除接口增加前置校验（出库/使用中存在性检查） | 中 |
| P0 | 删除接口回退采购订单收货数量 | 中 |
| P0 | 删除接口扣减批次数量并标记无效批次 | 小 |
| P1 | 新增 `GET /inbound/:id/check-deletable` 接口 | 小 |
| P1 | 前端删除弹窗分情况展示（可删除/不可删除） | 中 |
| P2 | 库存列表删除后自动刷新（已天然支持实时计算） | 无 |
