## 1. 项目背景与目标

### 1.1 产品愿景

COREONE 是一款专为病理实验室设计的高精度耗材管理系统，通过数字化手段解决实验室耗材管理中的库存混乱、成本不清、追溯困难等核心痛点。系统以"精准追溯、成本透明、高效协同"为核心理念，将传统的纸质台账升级为智能化管理平台。

### 1.2 问题陈述

**当前痛点**:
- **库存管理混乱**: 手工记录易出错，无法实时掌握库存状态，经常出现缺货或积压
- **成本归集困难**: 耗材成本无法准确分摊到具体检测项目，难以核算真实成本
- **有效期预警缺失**: 试剂过期后才发现，造成浪费甚至影响检测结果准确性
- **追溯链条断裂**: 出库后无法追溯耗材使用去向，质量问题难以定位
- **供应商管理粗放**: 缺乏对供应商的统一管理和采购价格分析

**目标用户**:
- **主要用户**: 病理科仓库管理员、实验室技术人员（25-45岁，专业医学背景）
- **次要用户**: 科室主任/管理者（关注报表和成本分析）、采购人员
- **使用场景**: 日常入库/出库操作、定期库存盘点、月度成本核算、采购决策

### 1.3 成功指标 (KPI)

| 指标类别 | 指标名称 | 目标值 |
|---------|---------|-------|
| **效率指标** | 库存盘点时间 | 从4小时缩短至30分钟 |
| **效率指标** | 入库/出库操作时间 | 单条记录≤2分钟 |
| **准确性指标** | 库存准确率 | ≥99.5% |
| **成本指标** | 成本归集准确度 | ≥98% |
| **预警指标** | 过期预警覆盖率 | 100% |
| **追溯指标** | 出库追溯完整度 | 100% |

### 1.4 业务目标与约束

**业务目标**:
- **短期目标** (v1.1, 3个月): 完成8个核心模块开发，覆盖日常库存管理、BOM关联、基础报表
- **中期目标** (v1.2, 6个月): 增加预警自动化、移动端支持、供应商评级
- **长期目标** (v2.0, 12个月): 智能采购建议、AI辅助成本优化、多院区协同

**已知约束**:
- 必须符合医疗器械管理相关法规要求
- 数据需保留至少5年
- 支持断网情况下的本地缓存操作
- 与现有HIS/LIS系统的集成能力

### 3.3 功能详细说明

#### 3.3.1 入库管理

**功能描述**: 管理所有物料入库操作，支持扫码快速入库、批次管理、入库审核流程。

**入库流程**:
```
选择入库方式（直接入库/采购入库/退货入库）
    ↓
扫描/搜索物料 → 自动填充物料信息
    ↓
录入批次信息（批次号、生产日期、有效期）
    ↓
录入入库数量和存放库位
    ↓
提交入库 → 更新库存 → 生成入库单
```

**界面要素**:
- 入库单号（自动生成，格式：IB-YYYYMMDD-XXX）
- 入库类型（直接入库/采购入库/退货入库）
- 物料信息（扫码/搜索选择）
- 批次信息（批次号、生产日期、有效期至）
- 入库数量、单位
- 供应商（关联采购订单时自动填充）
- 存放库位
- 经办人、入库时间
- 备注

**数据字段**:

| 字段名 | 类型 | 必填 | 验证规则 | 说明 |
|--------|------|------|----------|------|
| inboundNo | String | 系统自动 | IB-YYYYMMDD-XXX | 入库单号 |
| inboundType | Enum | 是 | direct/purchase/return | 入库类型 |
| materialId | String | 是 | 关联物料ID | 物料ID |
| batchNo | String | 条件必填 | 批次号格式 | 采购入库必填 |
| productionDate | Date | 条件必填 | 不能晚于今天 | 试剂类必填 |
| expiryDate | Date | 条件必填 | 必须晚于生产日期 | 有效期至 |
| quantity | Number | 是 | > 0 | 入库数量 |
| unit | String | 是 | 关联物料单位 | 单位 |
| supplierId | String | 条件必填 | 关联供应商ID | 采购入库必填 |
| locationId | String | 是 | 关联库位ID | 存放库位 |
| operator | String | 是 | 当前登录用户 | 经办人 |
| remark | String | 否 | ≤500字符 | 备注 |

**业务规则**:
- 入库单号全局唯一，按日期自动生成序号
- 试剂类物料必须录入有效期，系统根据有效期自动预警
- 批次号在同一物料下唯一
- 入库完成后库存实时增加，可即时查询
- 已审核入库单不可删除，有误需走退货流程

**异常处理**:
- 物料不存在：提示"物料不存在，请先维护物料信息"
- 批次号重复：提示"该物料下已存在此批次号"
- 有效期异常：生产日期晚于今天或有效期早于生产日期时提示
- 库存溢出：超出库位容量时提示并建议其他库位

#### 3.3.2 出库管理

**功能描述**: 管理物料出库领用，强制关联检测项目实现成本归集，支持BOM一键出库。

**出库流程**:
```
选择出库类型（项目领用/调拨/报废出库）
    ↓
选择/搜索检测项目（项目领用时）
    ↓
选择出库物料方式（单个选择/BOM一键选择）
    ↓
录入出库数量 → 系统自动检查库存
    ↓
提交出库 → 扣减库存 → 记录成本归属
```

**界面要素**:
- 出库单号（自动生成，格式：OB-YYYYMMDD-XXX）
- 出库类型（项目领用/调拨/报废）
- 关联项目（搜索/下拉选择）
- 物料列表（物料、批次、数量、成本）
- 出库时间、经办人
- 审批状态（如启用审批流程）

**数据字段**:

| 字段名 | 类型 | 必填 | 验证规则 | 说明 |
|--------|------|------|----------|------|
| outboundNo | String | 系统自动 | OB-YYYYMMDD-XXX | 出库单号 |
| outboundType | Enum | 是 | project/transfer/scrap | 出库类型 |
| projectId | String | 条件必填 | 关联项目ID | 项目领用必填 |
| items | Array | 是 | 至少一条 | 出库明细 |
| items.materialId | String | 是 | 关联物料 | 物料ID |
| items.batchNo | String | 否 | 批次号 | 批次（先进先出） |
| items.quantity | Number | 是 | > 0 | 数量 |
| items.unitCost | Number | 系统自动 | 该批次入库成本 | 单位成本 |
| items.totalCost | Number | 系统自动 | quantity × unitCost | 总成本 |
| operator | String | 是 | 当前用户 | 经办人 |
| status | Enum | 系统自动 | pending/completed/cancelled | 状态 |

**业务规则**:
- 项目领用必须关联检测项目，否则无法出库
- 出库时按"先进先出"原则自动选择批次（可手动调整）
- 库存不足时阻止出库并提示
- 出库成本 = 该批次入库时的实际采购成本
- 出库完成后自动累加到项目成本

**异常处理**:
- 库存不足：提示具体物料的缺口数量
- 项目已停用：提示"该项目已停用，无法领用"
- 物料已停用：提示"该物料已停用，如需使用请先启用"

#### 3.3.3 BOM管理

**功能描述**: 管理检测项目的标准物料清单（BOM），支持多版本管理和成本预估。

**界面要素**:
- BOM编码、名称、版本号
- 关联检测项目
- 物料清单（物料、规格、用量/样本、成本占比）
- 支持样本数预估总成本

**数据字段**:

| 字段名 | 类型 | 必填 | 验证规则 | 说明 |
|--------|------|------|----------|------|
| bomId | String | 是 | BOM-XXX | BOM编码 |
| name | String | 是 | 2-50字符 | BOM名称 |
| version | String | 系统自动 | vX.Y | 版本号 |
| projectId | String | 是 | 关联项目ID | 所属项目 |
| materials | Array | 是 | 至少一条 | 物料清单 |
| materials.materialId | String | 是 | 关联物料 | 物料 |
| materials.usagePerSample | Number | 是 | > 0 | 单样本用量 |
| materials.unit | String | 是 | - | 用量单位 |

**业务规则**:
- BOM修改后自动生成新版本，保留历史版本
- BOM成本 = ∑(物料单价 × 用量)
- 支持设置替代物料
- 项目关联BOM后，出库时可一键选择该BOM的全部物料

#### 3.3.4 成本分析报表

**功能描述**: 多维度成本统计和分析，支持项目成本、物料成本、供应商成本三个视角。

**成本归集逻辑**:
```
项目成本 = ∑(该项目关联的所有出库记录成本)
物料成本 = ∑(该物料的所有出库数量 × 对应批次入库成本)
供应商成本 = ∑(该供应商的所有采购入库金额)
```

**报表类型**:
1. **项目成本报表**
   - 按时间段筛选
   - 显示：项目、样本数、总成本、单样本成本、成本占比
   - 下钻：查看明细出库记录

2. **物料成本报表**
   - 按物料分类/具体物料筛选
   - 显示：物料、消耗量、总成本、成本占比、环比变化
   - 趋势：显示用量/成本变化曲线

3. **供应商成本报表**
   - 按供应商/时间段筛选
   - 显示：供应商、采购金额、采购次数、占比
   - 分析：价格趋势、采购频率

### 4.2 关键业务规则

1. **库存变动规则**:
   - 入库：库存增加
   - 出库：库存减少
   - 盘点差异：按实盘调整
   - 报废：库存减少，成本计入损耗

2. **成本计算规则**:
   - 物料成本 = ∑(出库数量 × 对应批次入库单价)
   - 项目成本 = ∑(该项目关联的所有出库物料成本)
   - 批次成本单价固定，采用先进先出法

3. **预警规则**:
   - 低库存预警：库存 ≤ 安全库存
   - 有效期预警：距离过期 ≤ 预警天数（默认30天）
   - 呆滞预警：超过X天无出入库记录
