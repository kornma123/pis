批次消耗跟踪 · 设计讨论稿

目标：追踪同一物料从出库到实际消耗完的时间节点

核心矛盾

一瓶 500ml 苏木精染液出库后，系统知道"何时出库"，但不知道"何时真正用完"。

方案对比

方案 原理 优点 缺点
1. 库存归零触发 库存从>0变为0时认为用完 自动触发 多批次并行时永远不会归零
2. 批次FIFO推算 按先进先出累积出库量，达容量时自动标记 全自动 挥发/浪费/报废导致不准确
3. 人工标记空瓶 技术员用完一瓶后点击"已空" 最准确 依赖人工习惯，易漏
4. 定期盘点校准 每周盘点时更新各瓶剩余量 校准理论值 操作成本高

建议方案：批次消耗跟踪 + 人工确认

  1. 入库时：记录批次号和初始数量（如 LOT-2024-001，500ml）
  2. 出库时：系统自动按FIFO分配批次，记录"从批次X消耗了Y ml"
  3. 系统自动推算：
    • 累积计算每个批次的剩余量
    • 剩余量 ≤ 0 时，标记"预计已用完"，给出预计时间点
    • 库存列表显示：批次 LOT-2024-001 · 剩余约 23ml · 预计 3 天后用完
  4. 人工确认：
    • 提供"标记为空瓶"按钮（1秒操作）
    • 与系统推算对比，逐步优化BOM准确度

需要新增的数据结构

CREATE TABLE batch_consumption (
    batch_id TEXT PRIMARY KEY,      -- 批次号
    material_id TEXT,               -- 物料ID
    initial_qty DECIMAL(18,4),      -- 初始数量
    remaining_qty DECIMAL(18,4),    -- 剩余数量（系统推算）
    estimated_empty_date DATE,      -- 预计用完日期
    actual_empty_date DATE,         -- 实际确认用完日期
    empty_confirmed_by TEXT,        -- 确认人
    status TEXT                     -- in_use / estimated_empty / confirmed_empty
);
