# 采购入库流程优化方案

## 现状问题

1. **入库记录与采购订单无关联**：`inbound_records` 表缺少 `purchase_order_id` 字段，入库后无法追溯对应采购订单
2. **采购订单仅支持单物料**：一条采购订单只能对应一种物料，实际采购中常有多物料订单
3. **自动填充不完整**：前端选择采购订单后只填充了物料信息，未填充供应商、单价等关键信息
4. **收货数量更新与入库创建非原子操作**：`purchaseOrderApi.receive()` 和入库创建是两次独立请求，可能数据不一致
5. **部分入库后采购订单列表未刷新**：前端入库后下拉框中已部分入库的订单状态未实时更新

---

## 优化方案

### 一、数据层改造

#### 1.1 inbound_records 增加采购订单关联

```sql
ALTER TABLE inbound_records ADD COLUMN purchase_order_id TEXT;
ALTER TABLE inbound_records ADD COLUMN purchase_order_no TEXT;
CREATE INDEX idx_inbound_po_id ON inbound_records(purchase_order_id);
```

#### 1.2 采购订单支持多物料

```sql
-- 改造 purchase_orders 为订单头表
ALTER TABLE purchase_orders ADD COLUMN title TEXT;

-- 新增采购订单明细表
CREATE TABLE purchase_order_items (
  id TEXT PRIMARY KEY,
  order_id TEXT NOT NULL,
  material_id TEXT NOT NULL,
  material_name TEXT,
  spec TEXT,
  ordered_qty DECIMAL(18,4) NOT NULL,
  received_qty DECIMAL(18,4) NOT NULL DEFAULT 0,
  unit TEXT NOT NULL,
  unit_price DECIMAL(18,4) DEFAULT 0,
  total_amount DECIMAL(18,4) DEFAULT 0,
  expected_date TEXT,
  remark TEXT,
  status TEXT DEFAULT 'pending', -- pending / partial / completed
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (order_id) REFERENCES purchase_orders(id)
);
```

#### 1.3 数据库迁移脚本

```javascript
// 兼容旧数据：将现有单物料采购订单迁移到明细表
db.prepare(`
  INSERT INTO purchase_order_items (id, order_id, material_id, material_name, ordered_qty, received_qty, unit, unit_price, total_amount, status)
  SELECT 
    lower(hex(randomblob(16))), id, material_id, material_name, ordered_qty, received_qty, unit, unit_price, total_amount, status
  FROM purchase_orders
`).run()
```

### 二、后端 API 改造

#### 2.1 入库创建时自动关联采购订单

```typescript
// POST /inbound 增加 purchaseOrderId 参数
router.post('/', (req, res) => {
  const { type, materialId, batchNo, quantity, price, supplierId, locationId, purchaseOrderId, productionDate, expiryDate, remark } = req.body
  
  // ... 创建入库记录 ...
  
  // 如果关联了采购订单
  if (purchaseOrderId) {
    // 1. 更新采购订单明细收货数量
    const orderItem = db.prepare('SELECT * FROM purchase_order_items WHERE order_id = ? AND material_id = ?').get(purchaseOrderId, materialId)
    if (orderItem) {
      const newReceived = Number(orderItem.received_qty) + quantity
      const itemStatus = newReceived >= Number(orderItem.ordered_qty) ? 'completed' : 'partial'
      db.prepare('UPDATE purchase_order_items SET received_qty = ?, status = ? WHERE id = ?').run(newReceived, itemStatus, orderItem.id)
      
      // 2. 更新订单头状态
      const allItems = db.prepare('SELECT status FROM purchase_order_items WHERE order_id = ?').all(purchaseOrderId) as any[]
      const allCompleted = allItems.every((i: any) => i.status === 'completed')
      const anyPartial = allItems.some((i: any) => i.status === 'partial')
      const orderStatus = allCompleted ? 'completed' : (anyPartial ? 'partial' : 'pending')
      db.prepare("UPDATE purchase_orders SET status = ? WHERE id = ?").run(orderStatus, purchaseOrderId)
    }
    
    // 3. 入库记录关联采购订单
    db.prepare('UPDATE inbound_records SET purchase_order_id = ?, purchase_order_no = ? WHERE id = ?')
      .run(purchaseOrderId, orderNo, id)
  }
  
  // ... 更新库存 ...
})
```

#### 2.2 采购订单列表 API 改造

```typescript
// GET /purchase-orders 返回时包含 items 数组
router.get('/', (req, res) => {
  // ... 查询订单头 ...
  const orders = db.prepare('SELECT * FROM purchase_orders ...').all() as any[]
  
  const result = orders.map(o => {
    const items = db.prepare('SELECT * FROM purchase_order_items WHERE order_id = ?').all(o.id) as any[]
    return {
      ...o,
      items: items.map(item => ({
        ...item,
        remainingQty: Number(item.ordered_qty) - Number(item.received_qty),
      })),
      totalItems: items.length,
      totalReceived: items.reduce((sum, i) => sum + Number(i.received_qty), 0),
    }
  })
  
  successList(res, result, page, pageSize, total)
})
```

### 三、前端交互改造

#### 3.1 采购订单选择器优化

```tsx
{form.type === 'purchase' && (
  <div>
    <label>采购订单 <span className="text-gray-400">(可选)</span></label>
    <select
      value={selectedOrderId}
      onChange={e => {
        const orderId = e.target.value
        setSelectedOrderId(orderId)
        if (orderId) {
          const order = purchaseOrders.find(o => o.id === orderId)
          if (order) {
            // 自动填充第一个待入库的物料
            const pendingItem = order.items?.find((item: any) => item.status !== 'completed')
            if (pendingItem) {
              setForm(prev => ({
                ...prev,
                materialId: pendingItem.material_id,
                quantity: pendingItem.remainingQty, // 默认填入待入库数量
                price: pendingItem.unit_price,
                supplierId: order.supplier_id,
              }))
            }
          }
        }
      }}
    >
      <option value="">不关联采购订单</option>
      {purchaseOrders.filter(o => o.status !== 'completed').map(o => (
        <option key={o.id} value={o.id}>
          {o.order_no} · {o.title || '采购订单'} · 已入 {o.totalReceived}/{o.totalItems} 项
        </option>
      ))}
    </select>
    
    {/* 显示订单明细待入库列表 */}
    {selectedOrderId && selectedOrder && (
      <div className="mt-2 p-3 bg-gray-50 rounded-md text-sm">
        <div className="font-medium mb-1">待入库物料：</div>
        {selectedOrder.items?.filter((item: any) => item.status !== 'completed').map((item: any) => (
          <div key={item.id} className="flex justify-between text-gray-600">
            <span>{item.material_name} · 待入 {item.remainingQty}{item.unit}</span>
            <button
              onClick={() => setForm(prev => ({
                ...prev, materialId: item.material_id, quantity: item.remainingQty, price: item.unit_price
              }))}
              className="text-blue-500 hover:text-blue-600"
            >
              选择
            </button>
          </div>
        ))}
      </div>
    )}
  </div>
)}
```

### 四、业务规则

| 场景 | 规则 |
|------|------|
| 选择采购订单入库 | 入库数量不能超过该物料的待入库数量 |
| 部分入库 | 采购订单状态变为 `partial`，明细显示已入/待入数量 |
| 全部入库 | 采购订单状态变为 `completed`，不再显示在选择器中 |
| 入库记录关联订单 | 入库列表显示采购订单号，点击可跳转 |
| 删除入库记录 | 如果关联了采购订单，同步扣减订单收货数量（见删除方案） |

### 五、实施优先级

| 优先级 | 内容 | 工作量 |
|--------|------|--------|
| P0 | inbound_records 增加 purchase_order_id 字段 | 小 |
| P0 | 入库创建时自动关联并更新采购订单收货数量 | 中 |
| P0 | 前端选择采购订单后自动填充完整信息（数量、单价、供应商） | 小 |
| P1 | 采购订单支持多物料（新增明细表） | 大 |
| P1 | 入库列表显示采购订单号 | 小 |
| P2 | 采购订单详情页显示入库记录列表 | 中 |
