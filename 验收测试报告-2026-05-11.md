## 二、角色场景测试

### 2.1 管理员场景

#### 场景A：创建采购订单 → 部分入库 → 查看库存
1. 进入入库管理页面
2. 选择「采购入库」类型
3. 选择采购订单（自动填充物料/数量/单价/供应商）
4. 输入批次号、选择库位
5. 确认入库
6. **预期结果**：库存列表显示新增批次，采购订单状态变为 partial

#### 场景B：删除入库记录（带校验）
1. 选择一条无出库关联的入库记录
2. 点击删除
3. **预期结果**：弹出确认弹窗，显示同步影响
4. 确认删除
5. **预期结果**：库存扣减，采购订单回退，列表刷新

#### 场景C：删除有出库关联的入库记录
1. 选择一条已有出库记录的入库记录
2. 点击删除
3. **预期结果**：弹出「无法删除」弹窗，显示原因和建议操作

### 2.2 操作员场景

#### 场景D：入库登记
1. 点击「新增入库」
2. 选择采购订单 → 系统自动填充物料信息
3. 输入批次号、数量
4. 选择库位
5. 确认入库
6. **预期结果**：入库成功，库存更新

#### 场景E：出库登记
1. 进入库存列表
2. 点击「出库」按钮
3. 添加物料、输入数量、选择领用人
4. 确认出库
5. **预期结果**：库存扣减，出库记录生成

### 5.2 统计卡片负数与过期数据修复

| 问题 | 原因 | 修复方案 | 状态 |
|------|------|----------|------|
| 「正常」卡片显示负数 | SQL 未过滤 stock <= 0 的批次，normal 计算逻辑覆盖不全 | `stock === 0` 改为 `stock <= 0`；normal 定义为「非低库存 ∧ 非即将过期 ∧ 非已过期 ∧ 有库存」 | ✅ 已修复 |
| 无有效期物料被计入「即将过期」 | SQL 未判断 `expiry IS NULL` | 所有日期比较前增加 `expiry IS NOT NULL AND expiry != ''` 条件 | ✅ 已修复 |
| 无有效期物料被计入「已过期」 | SQL 未判断 `expiry IS NULL` | 同上 | ✅ 已修复 |

**涉及文件**：
- [`后端代码/server/src/routes/inventory-v1.1.ts`](后端代码/server/src/routes/inventory-v1.1.ts)
- [`前端代码/src/pages/inventory/InventoryList.tsx`](前端代码/src/pages/inventory/InventoryList.tsx)
