# ADR-007 · P0 验收闸门 = 脱敏成本侧贡献毛利 golden（真医院月端到端手核）

> **日期**：2026-07-04 | **状态**：已采纳（PM 拷问 Q1 拍板）| **性质**：hard-to-reverse（决定"P0 完成"的定义/验收闸）。
> **编号说明**：本决策是 PM 拷问的**第一条（Q1）**、时间上最早；因当时先落成 `docs/adr/0001-...` 孤儿文件、编号与 ADR-001（月轴）撞号，现**并入权威序列为 ADR-007**（ADR 按formalize顺序编号，孤儿文件已删、内容无损迁入本文）。
> **关联**：P0 spec §7 硬前置 / §10.C 验收前置 / §10.F；ADR-001（关账·golden 冻结）、ADR-002/004（同源·被测对象）；方法论铁律"真跑端到端 + 真数据手核"。

## 背景（Q1·已核实）

签入的 `coreone.db` 里 `case_revenue / lis_case_markers / antibodies / ihc_cost_params / special_stain_kits` 五张表**都不存在**（只有 `lis_cases` 6 行种子）→ P0 引擎写完后**没有任何真数据能跑出真值**。本项目方法论铁律是「真跑端到端 + 真数据手核」，因此**只交付计算引擎 + fixture 单测（造）不算 P0 完成**。

## 决策

**P0 完成的硬闸门 = 导入至少一家真实医院月的三件套**（对账单 → `case_revenue`、工作量表 → `lis_cases`、抗体清单 → `lis_case_markers`），**跑出该院贡献毛利并人工手核**。验收物 = **一条脱敏的成本侧贡献毛利 golden**，与现有 ¥13,152 / ¥27,870 并列成**第三条 CI 回归锚**。

- **选院**：全流程院 ×1 + 代送加做院 ×1，专门行使**同源不变量**（代送院不得减本院未执行的工序，ADR-002/004 的被测对象）。**并须含一个代阅片 case（`lab_revenue=0`+有 marker）作 Q4 回归守卫、一个跨月复用 case_no 作 Q8 守卫。**
- **golden 冻结**：golden 选院选月**必须已关账 + 快照冻结**（ADR-001）——未结算月会被补收/纠错重算 → golden 会抖、CI 抖。补收/反关账不改冻结基线。
- **手核边界**：只验引擎 / join / 不变量的正确性，**不验约定价的绝对值**（约定价准不准归"换价源"线，不在 P0 验收范围）。

## 为什么不只做"引擎 + fixture 单测（造）"

fixture 单测能证"引擎逻辑自洽"，但**证不了"数据接得通"和"真实数据下不双计"**——这两个风险恰恰是碰钱口径最容易翻车处，必须用真数据手核堵住。

## 后果

- ✅ "P0 完成"锚在真数据手核，不是"引擎写完 + 单测绿"（防"逻辑对但数据接不上/双计"上线）。
- ✅ golden 第三锚进 CI → 成本侧口径此后有回归保护。
- ⚠️ **依赖先导入一家真院月三件套**——这也是 CM_TARGET 校准（待拍清单 B）、率分布观测的前置：**同一条"第一条真数据"同时解锁验收 + CM_TARGET 校准**。
- ⚠️ 选院要凑齐"全流程 + 代送 + 代阅片 case + 跨月复用 case"的守卫组合，选样要设计（非随手取一家）。

## 备选（未采纳）
- 只交引擎 + fixture 单测：否（证不了数据接通与不双计，碰钱高危）。
- 用签入 seed 库验收：否（五表空，跑不出真值）。
