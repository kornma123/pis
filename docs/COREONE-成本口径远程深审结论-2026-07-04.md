# COREONE 成本口径远程深审结论

> 日期：2026-07-04
> 分支：`claude/cost-methodology-deepreview`
> 详细证据：`docs/codex-handoff/findings/11-cost-methodology-deepreview-codex-2026-07-04.md`、`docs/codex-handoff/findings/12-cost-methodology-business-logic-deepdive-2026-07-04.md`

## 最终裁决

这套方法论的大方向成立：COREONE 当前不应追“单张切片真实耗用了哪瓶哪批试剂”，也不应把固定成本摊到医院后用来砍院。正确方向是：

```text
标准成本 = 按 SOP 和已确认工序集“该花多少”
贡献毛利 = 已确认实收 - 可避免标准变动成本
固定成本/间接成本 = 只在实验室整体或长期经营层看，不下沉做短期去留裁决
```

但当前文档如果直接交实现，会有三处高风险误用：

1. **贡献毛利、标准全成本、ABC 毛利、财务实际成本必须分 lane**。一个“利润”数不能同时服务接单、谈价、关账、年度投资。
2. **合作形态/工序集是第一前置**。没有 case 级 `service_step_scope`，代送加做染色、全流程、外购转销、现场服务会被混算，医院去留结论可能反转。
3. **抗体换价源不是第一优先级**。`MAT-IHC-*` seed 价确实要换，但多条生产路径显式传 `materialCost` 或读历史出库快照，回填 materials 价不会自动修正所有报表。

## PM 可采纳结论

### 可以拍板

- 成本口径用“标准成本（该花多少）”，不是单片实际成本。
- 医院去留/谈价优先看贡献毛利，不看固定成本摊完后的全成本盈亏。
- 缺价、工序集未知、LIS/对账覆盖不足时，不允许输出“精确毛利”。
- 分子/FISH 院内成本暂标未建模；NGS 外购转销若有已核外包成本，可单独算毛利。
- 期末实际差异可以回摊做总账勾稽和运营监管，但不能冒充逐 case 真实耗用。

### 暂不应拍板为实现方案

- 不能只做“把 `antibodies.per_test_price` 回填到 `MAT-IHC-*`”就认为成本侧闭环。
- 不能让 B4 工时/设备/房租校准后的每片成本默认进入贡献毛利。
- 不能把 `service_scope = technical_only / with_diagnosis` 当成工序集，它只是收入范围概念。
- 不能用 `cm_sim.py` 的单次百分比作为真实量级依据；它只证明结构性风险。

## 推荐下一步

优先顺序：

1. **写成本指标 ADR**：定义 `contributionMargin`、`standardFullCostMargin`、`abcMargin`、`actualFinancialCost` 的字段、用途、禁止用途。
2. **做 case 工序集 resolver**：合同默认 + 账单/LIS 反推 + 人工覆盖；低置信阻断精确毛利。
3. **贡献毛利/标准全成本双 lane 输出**：B4 校准默认进标准全成本，只有被确认可避免的成本进贡献毛利。
4. **价源契约测试**：证明抗体台账价进入哪些路径、哪些历史快照不会变、是否需要重算和版本化。
5. **缺价三桶展示**：精算成本、含估值成本、缺价暴露分开。
