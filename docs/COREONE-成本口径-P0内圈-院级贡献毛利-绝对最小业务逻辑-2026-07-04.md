# COREONE 成本口径 · P0 内圈 —— 院级贡献毛利「绝对最小业务逻辑」（框死版 v2）

> **日期**：2026-07-04 | **性质**：把"内圈"（PM 认可的产品主战场）下钻成一条**框死**的 P0 业务逻辑。
> **"框死"= 硬约束**：下面的数据源、公式、白名单、不变量、状态真值表都是硬的，违反即 bug。**本 v2 经三视角红队（framed-tight/purity/producible·`wf_4ed1c102`）砸出 6 破框 + 多收紧后重写**，逐条已封口。
> **上游**：`…方法论固化-2026-07-03.md`（①②③/贡献毛利/合作形态）；三圈范围见讨论图。
> **诚实措辞（红线）**：本 P0 产出的是**「贡献毛利（标准成本口径）」——实收与片数是真值(①)，可避免成本是约定标准成本(①/①*，非实测消耗)**。全文不得称其为"真实成本/真数/利润"。

## 2 · P0 计算（框死的公式，①/①* 分桶）

```
# 每 case，两个桶分开算、分开留标记（不合并成单一"成本"后丢失可追溯）
case 桶A·纯①材料   = Σ_row( 二抗显色 secondary_per_slide )              # row = advice_type∈真抗体码 的 marker 行
case 桶B·①*估材料  = Σ_row( 一抗 per_test_price[resolve(marker名)] )     # 一抗价内嵌换算率=①*
                   + special_stain_count × 特染每片约定价(kit_price/nominal_tests)   # 按盒摊=①*
                   + 组织处理耗材(每蜡块约定料价 × 蜡块数)   # 仅"本院执行该工序"为真时；均摊料价=①*
case 可避免材料成本 = 桶A + 桶B
case 贡献毛利(标准成本口径) = 实收 − case 可避免材料成本

# 缺价的 marker 行：per_test_price 查无 → 该行一抗价【不进桶B合计】，只累加到"缺价暴露片数"
# 院级上卷（同院·同 collected_month）
院级贡献毛利 = Σ_case( 贡献毛利 )          # case 归集口径见 §4 不变量⑥
每例贡献毛利 = 院级贡献毛利 / 有效 case 数   # 院际对比只用它（不变量④）
①*估值占比   = 桶B / (桶A+桶B)             # 必须披露（不变量③）
```

**算例（框死"多蜡块/联检/重复抗体=逐行累加"，消歧义）**：某 case 2 蜡块，marker 明细 3 行真抗体（蜡块1: CK7、Ki-67；蜡块2: CK7）→ **染色片数 = 3**（CK7 计 2 片，不因同抗体去重）→ 桶A = 3×¥15；桶B = per_test_price[CK7]×2 + per_test_price[Ki-67]×1。

## 3 · 减项白名单（**封闭**，只减这些）

**✅ 进贡献毛利（可避免变动成本）**：
- 一抗（`per_test_price` × 明细片数）——`①*`
- 二抗+显色（`secondary_per_slide` × 明细片数）——①
- 特染（`special_stain_count` × 每片约定价）——`①*`
- 组织处理耗材（每蜡块约定料价 × 蜡块数，**仅本院执行该工序时**）——`①*`

**❌ 永不进 P0 贡献毛利**（穷举，白名单外一律不计）：
- 工时 / 设备折旧 / 房租 / 在编固定人力（②，属中圈 ABC）——**含 `computeFullSlideCost` 里的 `laborPerSlide`/`equipmentPerSlide`(G2)，见 §8**
- **可变人力**（外包/加班/计件）——**P0 不纳入**（无三件套内数据源、需工时数据 → 与不变量⑤冲突；列为中圈候选）
- 对照片 / 质控片（②，批次固定、不下沉单片）
- 单片实际消耗 / 浪费 / 深切连切额外耗 / 白片玻片（③ 或已知低估）
- 任何 ABC 作业成本、`outbound_abc_details.costTotal`

## 4 · 硬不变量（**违反 = bug**，回归门禁锁死）

1. **同源**：某 case 成本步骤集 = 该 case 实收覆盖的服务步骤集。代送加做的 case **不减**未由本院执行的工序（尤其组织处理）。
2. **只减可避免**：工时/设备/房租/在编人力/对照片分摊**永不进**贡献毛利。P0 是**独立新 lane**，**不复用** `partner-pnl-service` 的 `grossMargin`（= 收入 − ABC 全成本）。
3. **不冒充精确·三诚实字段并列**：缺价 marker 行**不进合计**、只计**缺价暴露率（片数占比，不折金额）**；`①*估值占比` 必须披露；覆盖率必须披露。**`缺价暴露率` 或 `①*估值占比` 或 覆盖率 任一超阈值 → 禁止输出去留/砍院结论，只能 `需补数据`/`需校准约定价`**。
4. **院际比每例·同口径**：跨院比较**只用每例贡献毛利**，且**只比同口径院**（"含前处理"与"仅染色/需补工序集"的院**不得直接排名**；看板对每院显式标口径）。
5. **零第四类源依赖**：P0 计算不读 inventory/outbound/batches/消耗对账/实际消耗/工时台账。
6. **case 归集口径**：① 月轴 = `collected_month`（实收确认月，对齐补收口径 #33），成本侧经 case_no 对齐同月（复用 `partner-pnl-service.ts:182`）；② 退单/作废 case（实收冲红/置 0）→ **分子分母双剔**，单列"作废 case 数"；③ 补收致实收变化 → 院级为**派生值、按当前 `collected_month` 快照重算**（增量纠错架构）；④ 同院同 `case_no` 撞号 → 按 `(院, case_no, 接收号/蜡块)` 联合键区分，**不静默合并成本**。
7. **降级须先判染色归属**：优雅降级（§5）**仅当已知"本院至少执行了染色步骤"时**才输出"染色贡献毛利（不含前处理）"；若连"是否本院染色"都未知（纯代送阅片/代诊场景）→ 该 case 标 `needs_step_scope`、**不输出任何贡献毛利，只上卷实收**。
8. **分子/NGS 剔除**：院内 FISH/NGS case（成本侧不在白名单）→ 从 P0 院级贡献毛利**分子分母双剔**，单列"分子线实收（成本未建模）"；外购转销 NGS 若有已核外包成本，另桶 `售价−外包成本`，**不并入 P0 贡献毛利率**。

## 5 · P0 输出（看板最小集 + 状态真值表，框死）

**每院每月字段**：院级实收 / 贡献毛利(标准成本口径) / 贡献毛利率 / **每例贡献毛利** / **缺价暴露率** / **①*估值占比** / **覆盖率** / `needs_step_scope` 占比 / 口径(完整|仅染色) / 作废 case 数 / 分子线实收(未建模) / **状态**。

**优雅降级**：`染色贡献毛利(不含前处理)` 与 `贡献毛利(完整)` **分列两字段**、不合并（不变量④禁跨口径排名）。

**状态真值表（框死判定顺序：数据 gate 先于经营判定）**：

| 优先级 | 条件 | 状态 |
|---|---|---|
| 1（gate） | `缺价暴露率 > MAX_MISSING_PRICE_RATE` 或 `①*估值占比 > MAX_STAR_RATE` 或 `覆盖率 < MIN_COVERAGE` 或 `needs_step_scope占比 > MAX_UNSCOPED` | **需补数据**（先于一切经营判定，禁出去留结论） |
| 2 | 标准/实际差异异常（P0 无实际侧 → 本状态 P0 暂不自动判，留人工置位） | 需运营排查 |
| 3 | 每例贡献毛利 < 0 | **停止候选** |
| 4 | 0 ≤ 每例贡献毛利 < CM_TARGET | **需谈价** |
| 5 | 外部标记 `capacity_constrained` 且毛利为正且 CM/瓶颈分钟低 | 需限量（产能数据不在三件套→**仅人工置位**） |
| 6 | 以上均不触发、毛利健康 | **可留** |

> **阈值为必须先定的具名常量**（值待 PM 拍板，先给保守默认）：`MAX_MISSING_PRICE_RATE`(默认 10%)、`MAX_STAR_RATE`(默认 60%)、`MIN_COVERAGE`(默认 85%)、`MAX_UNSCOPED`(默认 15%)、`CM_TARGET`(待定)。**禁用词**：利润/盈利/亏损（贡献毛利 ≠ 利润）。

## 6 · P0 明确不做（边界，框死）

- ❌ 单片真实成本 / 逐 case 实际消耗（③）。
- ❌ 工时/设备/房租/间接分摊（②，中圈 ABC）；不复用 `outbound_abc_details` 的 `grossMargin`。
- ❌ 对照片下沉单片；白片/玻片耗材单列（已知低估，标注）。
- ❌ 依赖库存/出库/消耗对账。
- ❌ "利润"数、单片精确排名、全成本砍院。
- ❌ 院内分子线（FISH/NGS）建模（看板标"成本未建模"）。
- ❌ 可变人力（无数据源）。

## 8 · 与现有代码的关系（P0 不碰什么·codex/红队已核）

- P0 是**新的直算 lane**：`case_revenue` + `lis_case_markers`/`lis_cases` + 约定价 → 贡献毛利。**不经** `cost-calculator.ts` / `outbound_abc_details` / `calculateMaterialCost`（那条读出库、混全成本、数据是 seed）。
- **可复用**：`resolveAntibodyName`、`per_test_price`、`secondary_per_slide`、`special_stain_kits`。**不得复用** `computeFullSlideCost` **整包**（它含 `laborPerSlide`/`equipmentPerSlide` G2 = §3 明令排除的工时/设备）——P0 只取一抗价 + 二抗 + 特染，显式排除 labor/equipment。
- 现有 `partner-pnl-service.ts` 的 `grossMargin`（收入 − ABC 全成本）是**中圈 lane**，P0 与它并存、不改它。

## 9 · 一句话框死

> **P0 = 一条只吃「实收(真) + LIS 抗体明细片数(真) + 约定价(①/①*)」、只减「白名单内可避免材料」、①/①* 分桶并披露估值占比、按合作形态裁工序集、缺价/未知诚实降级不冒充、只输出「贡献毛利(标准成本口径) + 三诚实字段 + 状态词」的新 lane；工时设备实际消耗库存全不碰。它在"真实数据已导入"后就能出一个口径明确的贡献毛利估——不碰任何已有成本管道。**
