## 2 · P0 计算（框死的公式，①/①* 分桶）

```
# ⚠️ 同源闸（Q4/ADR-002·最关键）：染色成本(桶A/桶B)只在 lab_revenue > 0 时扣。
#    lab_revenue = Σ(IN 结算) = 实验室技术收入；=0 ⟺ 代阅片/纯诊断(医院自己染、只送片来我们出报告)。
#    lab_revenue=0 的 case → 即便有 marker 行也【不扣任何染色成本】，移出染色贡献毛利、计入"诊断桶 case 数"(P0 不做诊断 P&L)。
#    ——marker 行证明"这批片染了什么"，不证明"这色是我们染的"；"我们染的"由 lab_revenue>0(收入含技术)判定 = 同源①落地。
# 仅对 lab_revenue > 0 的 case，两个桶分开算、分开留标记（不合并成单一"成本"后丢失可追溯）
case 桶A·纯①材料   = Σ_row( 二抗显色 secondary_per_slide )              # row = advice_type∈真抗体码 的 marker 行
case 桶B·①*估材料  = Σ_row( 一抗 per_test_price[resolve(marker名)] )     # 一抗价内嵌换算率=①*
                   + special_stain_count × 特染每片约定价(kit_price/nominal_tests)   # 按盒摊=①*
                   + 组织处理耗材(P0_TISSUE_PROCESSING_MATERIAL_PER_BLOCK × block_count)
                     # ①*·仅当 (a)该封闭参数已定 且 (b)工序集 tissue_processing=true 才计入；缺任一 → 不计、走染色口径
case 可避免材料成本 = 桶A + 桶B
case 贡献毛利(标准成本口径) = lab_revenue − case 可避免材料成本
# ⚠️ 组织处理料价源(P0_TISSUE_PROCESSING_MATERIAL_PER_BLOCK)现无表/字段(ihc_cost_params 只有二抗/工时/设备)
#    → 该参数补齐前，P0 今天可产的核心 = 「染色贡献毛利(不含前处理)」；组织处理是 P0 的第一扩展。

# 缺价的 marker 行：per_test_price 查无 → 该行一抗价【不进桶B合计】，只累加到"缺价暴露片数"
# 院级上卷（同院·同 service_month）
院级贡献毛利 = Σ_case( 贡献毛利 )          # case 归集口径见 §4 不变量⑥
每例贡献毛利 = 院级贡献毛利 / 有效 case 数   # 院际对比只用它（不变量④）
①*估值占比   = 桶B / (桶A+桶B)             # 必须披露（不变量③）
```

**算例（框死"多蜡块/联检/重复抗体=逐行累加"，消歧义）**：某 case 2 蜡块，marker 明细 3 行真抗体（蜡块1: CK7、Ki-67；蜡块2: CK7）→ **染色片数 = 3**（CK7 计 2 片，不因同抗体去重）→ 桶A = 3×¥15；桶B = per_test_price[CK7]×2 + per_test_price[Ki-67]×1。

## 3 · 减项白名单（**封闭**，只减这些）

**✅ 进贡献毛利（可避免变动成本）**：
- 一抗（`per_test_price` × 明细片数）——`①*`
- 二抗+显色（`secondary_per_slide` × 明细片数）——①
- 特染（`special_stain_count` × 每片约定价）——`①*`
- 组织处理耗材（每蜡块约定料价 × 蜡块数，**仅本院执行该工序时**）——`①*`

**❌ 永不进 P0 贡献毛利**（穷举，白名单外一律不计）：
- 工时 / 设备折旧 / 房租 / 在编固定人力（②，属中圈 ABC）——**含 `computeFullSlideCost` 里的 `laborPerSlide`/`equipmentPerSlide`(G2)，见 §8**
- **可变人力**（外包/加班/计件）——**P0 不纳入**（无三件套内数据源、需工时数据 → 与不变量⑤冲突；列为中圈候选）
- 对照片 / 质控片（②，批次固定、不下沉单片）
- 单片实际消耗 / 浪费 / 深切连切额外耗 / 白片玻片（③ 或已知低估）
- 任何 ABC 作业成本、`outbound_abc_details.costTotal`

## 4 · 硬不变量（**违反 = bug**，回归门禁锁死）

1. **同源**：某 case 成本步骤集 = 该 case 实收覆盖的服务步骤集。代送加做的 case **不减**未由本院执行的工序（尤其组织处理）。
2. **只减可避免**：工时/设备/房租/在编人力/对照片分摊**永不进**贡献毛利。P0 是**独立新 lane**，**不复用** `partner-pnl-service` 的 `grossMargin`（= 收入 − ABC 全成本）。
3. **不冒充精确·三诚实字段并列**：缺价 marker 行**不进合计**、只计**缺价暴露率（片数占比，不折金额）**；`①*估值占比` 必须披露；覆盖率必须披露。**`缺价暴露率` 或 `①*估值占比` 或 覆盖率 任一超阈值 → 禁止输出去留/砍院结论，只能 `需补数据`/`需校准约定价`**。
4. **院际比每例·同口径**：跨院比较**只用每例贡献毛利**，且**只比同口径院**（"含前处理"与"仅染色/需补工序集"的院**不得直接排名**；看板对每院显式标口径）。
5. **零第四类源依赖**：P0 计算不读 inventory/outbound/batches/消耗对账/实际消耗/工时台账。
6. **case 归集口径**（月轴见 ADR-001·权责发生制）：① **月轴 = `case_revenue.service_month`**（权责/配比——成本物理发生在服务月、无"到账月"；`collected_month` 列不存在、且只对现金回收报表有意义）。成本经 case_no join 到该 case 的 service_month，`operate_time` **仅异常提示**（`operate_time≠service_month` 标数据异常）。`service_month` 为 NULL 的 case → 归不了月，走 `needs_data`；② 退单/作废见 §10.E（无 status 列，暂不自动剔）；③ **补收致实收变化 → 关账感知重算**：原月**未关账** → 回填原 case 的 service_month、重算原月（增量纠错）；原月**已关账**（`account-reconcile` 定版·`:147` 409 不可改）→ **不改历史定版**，补收作原月**追溯调整单列**（贡献毛利标"含追溯补收¥X"），或走反关账（理由+经手人）才改——**保护 golden 稳定 + 关账冻结**；④ 撞号见 §10.E（数据异常禁输出）。**与 #33 显式分家**：#33 的 `collected_month` = 现金/补收追踪视图；P0 的 `service_month` = 毛利配比视图，两表答不同问题、各自成立、不强并。
7. **染色归属：数据派生 + 同源双条件（§10.C·Q3+Q4）**：staining = (有真抗体 marker 或 `special_stain_count>0`) **AND `lab_revenue>0`**——marker 证"染了什么"、`lab_revenue>0`(=Σ IN 结算) 证"是我们染的"（收入含技术=同源）。**不由 scope 字段 gate**（破 Q3 全院 0-输出死循环）；**有 marker 但 `lab_revenue=0`（代阅片/纯诊断，医院自己染）→ 不减染色、移"诊断桶"**（防 Q4 误减本院没做的染色，守同源①）。`tissue_processing` gate 只管前处理：未知 → 退染色口径、标 `needs_tissue_scope`，不阻断输出。
8. **分子/NGS 剔除**：院内 FISH/NGS case（成本侧不在白名单）→ 从 P0 院级贡献毛利**分子分母双剔**，单列"分子线实收（成本未建模）"；外购转销 NGS 若有已核外包成本，另桶 `售价−外包成本`，**不并入 P0 贡献毛利率**。

## 5 · P0 输出（看板最小集 + 状态真值表，框死）

**每院每月字段**：院级实收 / 贡献毛利(标准成本口径) / 贡献毛利率 / **每例贡献毛利** / **缺价暴露率** / **①*估值占比** / **覆盖率** / `needs_tissue_scope` 占比 / 口径(完整|仅染色) / 作废 case 数 / 分子线实收(未建模) / **状态**。

**优雅降级**：`染色贡献毛利(不含前处理)` 与 `贡献毛利(完整)` **分列两字段**、不合并（不变量④禁跨口径排名）。

**状态真值表（框死判定顺序：数据 gate 先于经营判定）**：

| 优先级 | 条件 | 状态 |
|---|---|---|
| 1（gate） | `缺价暴露率 > MAX_MISSING_PRICE_RATE` 或 `①*估值占比 > MAX_STAR_RATE` 或 `覆盖率 < MIN_COVERAGE` 或 `needs_tissue_scope占比 > MAX_UNSCOPED`(仅判"完整口径"可靠性·不阻断"染色口径"输出) | **需补数据**（先于一切经营判定，禁出去留结论） |
| 2 | 标准/实际差异异常（P0 无实际侧 → 本状态 P0 暂不自动判，留人工置位） | 需运营排查 |
| 3 | 每例贡献毛利 < 0 | **停止候选** |
| 4 | 0 ≤ 每例贡献毛利 < CM_TARGET | **需谈价** |
| 5 | 外部标记 `capacity_constrained` 且毛利为正且 CM/瓶颈分钟低 | 需限量（产能数据不在三件套→**仅人工置位**） |
| 6 | 以上均不触发、毛利健康 | **可留** |

> **阈值为必须先定的具名常量**（值待 PM 拍板，先给保守默认）：`MAX_MISSING_PRICE_RATE`(默认 10%)、`MAX_STAR_RATE`(默认 60%)、`MIN_COVERAGE`(默认 85%)、`MAX_UNSCOPED`(默认 15%)、`CM_TARGET`(待定)。**禁用词**：利润/盈利/亏损（贡献毛利 ≠ 利润）。

## 6 · P0 明确不做（边界，框死）

## 8 · 与现有代码的关系（P0 不碰什么·codex/红队已核）

- P0 是**新的直算 lane**：`case_revenue` + `lis_case_markers`/`lis_cases` + 约定价 → 贡献毛利。**不经** `cost-calculator.ts` / `outbound_abc_details` / `calculateMaterialCost`（那条读出库、混全成本、数据是 seed）。
- **可复用**：`resolveAntibodyName`、`per_test_price`、`secondary_per_slide`、`special_stain_kits`。**不得复用** `computeFullSlideCost` **整包**（它含 `laborPerSlide`/`equipmentPerSlide` G2 = §3 明令排除的工时/设备）——P0 只取一抗价 + 二抗 + 特染，显式排除 labor/equipment。
- 现有 `partner-pnl-service.ts` 的 `grossMargin`（收入 − ABC 全成本）是**中圈 lane**，P0 与它并存、不改它。

## 9 · 一句话框死

> **P0 = 一条只吃「实收(真) + LIS 抗体明细片数(真) + 约定价(①/①*)」、只减「白名单内可避免材料」、①/①* 分桶并披露估值占比、按合作形态裁工序集、缺价/未知诚实降级不冒充、只输出「贡献毛利(标准成本口径) + 三诚实字段 + 状态词」的新 lane；工时设备实际消耗库存全不碰。它在"真实数据已导入"后就能出一个口径明确的贡献毛利估——不碰任何已有成本管道。**

## 10 · P0 工程契约（codex 复核后锁定·把"框死"落到字段级）

> codex 远程复核（`findings/14`）逐条经本地核实为真后，把 v2 还留的实现歧义锁到这里。**实现必须遵此契约。**

**A · P0 SQL 契约（收入分子 + 月轴 + join + 禁止）**
```
输入 case 集 = case_revenue cr
  WHERE cr.partner_id=:p AND cr.service_month=:m
    AND cr.lab_revenue IS NOT NULL AND cr.revenue_source IN ('statement','corrected')
    AND cr.lab_revenue > 0     # 同源闸(ADR-002)：仅有技术收入的 case 进染色贡献毛利；
                               # lab_revenue=0(代阅片/纯诊断) → 移出、单列"诊断桶 case 数"，不扣染色成本(防破同源①)
P0 收入 = cr.lab_revenue                       # 绝不用 net_amount（含诊断/外送/未分配）
P0 月轴 = cr.service_month                      # case_revenue 无 collected_month
成本 join = LEFT JOIN lis_cases / lis_case_markers ON (partner_id, case_no)
           lis_cases.operate_time 只做异常提示，不做主月轴过滤
禁止：net_amount 当收入 · outbound_abc_details · computeFullSlideCost · specialStainPerTestCost 的含 labor total · operate_time 当主月轴
```

**B · 五个具名常量（值待 PM 拍板，先给保守默认；缺则 P0 不出对应口径）**
```
P0_ANTIBODY_ADVICE_TYPES = {'Y000001','Y000003'}   # 有 advice_type 只认白名单；无码→不进合计、计入"marker 口径缺失暴露"；主算不用名字兜底(名字兜底仅审计提示)
P0_STAIN_MATERIAL_DENOM  = COALESCE(NULLIF(actual_yield,0), nominal_tests)  # 特染每片=kit_price/denom，禁读 labor_per_test
P0_TISSUE_PROCESSING_MATERIAL_PER_BLOCK = <无表/字段，须新建封闭参数>  # 缺→P0 只出「染色贡献毛利(不含前处理)」
阈值：MAX_MISSING_PRICE_RATE=10% · MAX_STAR_RATE(①*占比)=60% · MIN_COVERAGE=85% · MAX_UNSCOPED=15% · CM_TARGET=待定
```

**C · 工序集 = 只解决"前处理做没做"；staining 从数据派生（Q3 破 0-输出死循环）**

死循环（已核）：`service_step_scope` 全 NULL（仅 `DatabaseManager.ts:1468` 预埋、无 resolver/写入）。若按"staining!==true 就不出" → **全院贡献毛利 = 0**，掐死 Q1 的 B 验收。
根因：**staining 不该由一个从没被写的字段来 gate——染没染，LIS 数据自证。**
```
"我们染的"判定（同源双条件·Q3 破死循环 + Q4 防误减）：
  条件甲(染了什么) = 有真抗体 marker(advice_type∈P0_ANTIBODY_ADVICE_TYPES) 或 special_stain_count>0   # marker 证"这批片染了什么"
  条件乙(是我们染) = lab_revenue > 0 (=Σ IN 结算>0，收入含技术份额)                                # 证"这色是我们染的"=同源
  staining = 甲 AND 乙
  甲真乙假(有 marker 但 lab_revenue=0) = 代阅片/纯诊断(医院自己染、只送片来我们出报告) → 不扣染色、移出染色贡献毛利、计"诊断桶 case 数"
  甲假(无 marker) → 无染色成本可减(天然 0)

service_step_scope 收窄到只承载 tissue_processing（前处理·marker 推不出，代送加做/全流程都有 marker）：
  { "tissue_processing": true|false|null, "source": "case_override|bill|partner_default|unknown" }
  优先级 = case 人工覆盖 > 账单/LIS 推断 > partner 协议默认(每院一个标志：代送院=false/全流程院=true) > unknown

输出规则（不再 block-all；染色由 甲 AND 乙 gate）：
  staining=true(甲 AND 乙)                          → 出「染色贡献毛利」   ← P0 day-1 的真数
    ├ tissue_processing===true                      → 追加前处理 = 「完整贡献毛利」
    └ tissue_processing 未知/false                  → 「染色贡献毛利(不含前处理)」，标 needs_tissue_scope
  甲真乙假(代阅片)                                  → 移出染色贡献毛利、计"诊断桶 case 数"（不误减·守同源①）
  乙真甲假(lab_revenue>0 但无 marker)               → coverage gap(§10.D)，标 needs_marker（非 block-all）
```
**验收前置（Q1 的 B·同源不变量的被测对象）**：golden 样本**必须同时含两类 case，缺一则 Q4/同源漏洞会静默上线**：① **代送加做 case**（lab_revenue>0 + 有 marker，partner `tissue_processing=false`）→ 测"减染色、不减前处理"；② **代阅片 case**（lab_revenue=0 + 有 marker 行）→ 测"有 marker 也不减染色"（Q4 回归守卫）。填 partner 默认标志 + 备一个代阅片 case 是**被测对象本身**，非抄近路。**partner 标志一个/院、非逐 case。**

**D · 分母定义（MEDIUM 3·状态 gate 可回归的前提）**
```
revenue_case_count      = P0 case 集中 lab_revenue 非空的 case 数
coverage                = (有真抗体 marker 或 ihc_count=0 的 case 数) / revenue_case_count
missing_price_rate      = 缺价的真抗体 marker 行数 / 真抗体 marker 总行数     # 只计片数占比，不折金额
valid_case_count(每例CM分母) = 通过数据 gate 且未双剔(分子/NGS/作废)的 case 数
```

**E · 撞号 & 作废（HIGH 4 / MEDIUM 4·现 schema 兜不住，收窄承诺）**
- **撞号**：`lis_cases` 唯一键=(partner_id,case_no)、`case_revenue`=(partner_id,case_no,service_month)，**无接收号维度**→ 联合键区分**做不到**。故 P0 改口径：**同院 `case_no` 复用 = 数据异常，禁输出该 case 贡献毛利**（不再承诺联合键）。若业务无法保证院内 case_no 唯一 → 先加 `case_instance_key/accession_id` 让三表同键，**属 P0 外前置**。
- **作废双剔**：`case_revenue` **无 `status` 列**→ 作废识别信号**待补**（需 `status∈(voided,refunded)` 或冲红原单引用）。在此之前：**`lab_revenue=0` 且无 void 证据者保留在分母**（可能是免费/移出/未分配，非作废），不自动剔——防止抬高每例贡献毛利。

**F · 措辞（MEDIUM 5）**：把"表已建/今天能出真数"改为"**schema initializer 具备这些表；导入三件套(对账单+工作量表+抗体清单)后，才能对指定院月产 P0 结果**"。签入 `data/coreone.db` 快照里这些表是空的（codex 实测：case_revenue/lis_case_markers/antibodies 均 MISSING）——不能直接 join 出真数。
