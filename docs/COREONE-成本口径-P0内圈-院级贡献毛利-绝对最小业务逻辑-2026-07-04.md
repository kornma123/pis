## 2 · P0 计算（框死的公式，①/①* 分桶）

```
# ⚠️ 同源闸（Q4/ADR-002·最关键）：染色成本(桶A/桶B)只在 lab_revenue > 0 时扣。
#    lab_revenue = Σ(IN 结算) = 实验室技术收入；=0 ⟺ 代阅片/纯诊断(医院自己染、只送片来我们出报告)。
#    lab_revenue=0 的 case → 即便有 marker 行也【不扣任何染色成本】，移出染色贡献毛利、计入"诊断桶 case 数"(P0 不做诊断 P&L)。
#    ——marker 行证明"这批片染了什么"，不证明"这色是我们染的"；"我们染的"由 lab_revenue>0(收入含技术)判定 = 同源①落地。
# 仅对 lab_revenue > 0 的 case，两个桶分开算、分开留标记（不合并成单一"成本"后丢失可追溯）
case 桶A·纯①材料   = Σ_row( 二抗显色 secondary_per_slide )              # row = advice_type∈真抗体码 的 marker 行
case 桶B·①*估材料  = Σ_row( 一抗 per_test_price[resolve(marker名)] )     # 一抗价内嵌换算率=①*
                   + special_stain_count × 特染每片约定价(kit_price/nominal_tests)   # 按盒摊=①*
                   + 组织处理耗材(P0_TISSUE_PROCESSING_MATERIAL_PER_BLOCK × block_count)
                     # ①*·仅当 (a)该封闭参数已定 且 (b)工序集 tissue_processing=true 才计入；缺任一 → 不计、走染色口径
case 可避免材料成本 = 桶A + 桶B
case 贡献毛利(标准成本口径) = lab_revenue − case 可避免材料成本
# ⚠️ 组织处理料价源(P0_TISSUE_PROCESSING_MATERIAL_PER_BLOCK)现无表/字段(ihc_cost_params 只有二抗/工时/设备)
#    → 该参数补齐前，P0 今天可产的核心 = 「染色贡献毛利(不含前处理)」；组织处理是 P0 的第一扩展。

# 缺价的 marker 行：per_test_price 查无 → 该行一抗价【不进桶B合计】，只累加到"缺价暴露片数"
# 院级上卷（同院·同 service_month）
院级贡献毛利 = Σ_case( 贡献毛利 )          # case 归集口径见 §4 不变量⑥
每例贡献毛利 = 院级贡献毛利 / 有效 case 数   # ⚠️仅同院趋势·院际【禁用】它（每例CM case-mix 敏感，不变量④）；跨院主指标=贡献毛利率(CM÷lab_revenue)、辅=每片贡献毛利
①*估值占比   = 桶B / (桶A+桶B)             # 必须披露（不变量③）
```

**算例（框死"多蜡块/联检/重复抗体=逐行累加"，消歧义）**：某 case 2 蜡块，marker 明细 3 行真抗体（蜡块1: CK7、Ki-67；蜡块2: CK7）→ **染色片数 = 3**（CK7 计 2 片，不因同抗体去重）→ 桶A = 3×¥15；桶B = per_test_price[CK7]×2 + per_test_price[Ki-67]×1。

## 3 · 减项白名单（**封闭**，只减这些）

**✅ 进贡献毛利（可避免变动成本）**：
- 一抗（`per_test_price` × 明细片数）——`①*`
- 二抗+显色（`secondary_per_slide` × 明细片数）——①
- 特染（`special_stain_count` × 每片约定价）——`①*`
- 组织处理耗材（每蜡块约定料价 × 蜡块数，**仅本院执行该工序时**）——`①*`

**❌ 永不进 P0 贡献毛利**（穷举，白名单外一律不计）：
- 工时 / 设备折旧 / 房租 / 在编固定人力（②，属中圈 ABC）——**含 `computeFullSlideCost` 里的 `laborPerSlide`/`equipmentPerSlide`(G2)，见 §8**
- **可变人力**（外包/加班/计件）——**P0 不纳入**（无三件套内数据源、需工时数据 → 与不变量⑤冲突；列为中圈候选）
- 对照片 / 质控片（②，批次固定、不下沉单片）
- 单片实际消耗 / 浪费 / 深切连切额外耗 / 白片玻片（③ 或已知低估）
- 任何 ABC 作业成本、`outbound_abc_details.costTotal`

## 4 · 硬不变量（**违反 = bug**，回归门禁锁死）

1. **同源**：某 case 成本步骤集 = 该 case 实收覆盖的服务步骤集。代送加做的 case **不减**未由本院执行的工序（尤其组织处理）。
2. **只减可避免**：工时/设备/房租/在编人力/对照片分摊**永不进**贡献毛利。P0 是**独立新 lane**，**不复用** `partner-pnl-service` 的 `grossMargin`（= 收入 − ABC 全成本）。
3. **不冒充精确·三诚实字段并列**：缺价 marker 行**不进合计**、只计**缺价暴露率（片数占比，不折金额）**；`①*估值占比` 必须披露；覆盖率必须披露。**`缺价暴露率` 或 `①*估值占比` 或 覆盖率 任一超阈值 → 禁止输出去留/砍院结论，只能 `需补数据`/`需校准约定价`**。
4. **院际比复杂度中性指标·同口径（Q9#2·每例 CM 会被 case-mix 扭曲）**：跨院比较**主用 贡献毛利率（CM÷lab_revenue）**、辅以 **每片贡献毛利（CM÷片数）**——二者 case-mix 中性。**每例贡献毛利 case-mix 敏感**（做 10 抗大 case 的院每例 CM 虚高、误伤做简单便宜 case 但单位效率高的院）→ **只用于同院趋势 / 绝对贡献看总额，禁用于跨院排名**。绝对贡献力度看 **院级总贡献毛利**。仍**只比同口径院**（"含前处理"与"仅染色"不得直接排名，看板标口径）。
5. **零第四类源依赖**：P0 计算不读 inventory/outbound/batches/消耗对账/实际消耗/工时台账。
6. **case 归集口径**（月轴见 ADR-001·权责发生制）：① **月轴 = `case_revenue.service_month`**（权责/配比——成本物理发生在服务月、无"到账月"；`collected_month` 列不存在、且只对现金回收报表有意义）。成本经 case_no join 到该 case 的 service_month，`operate_time` **仅异常提示**（`operate_time≠service_month` 标数据异常）。`service_month` 为 NULL 的 case → 归不了月，走 `needs_data`；② 退单/作废见 §10.E（走负数行→冲红白捡：全额冲红 lab_revenue≤0 已被准入闸剔、部分冲红净额已扣无双计）；③ **补收致实收变化 → 关账感知重算**：原月**未关账** → 回填原 case 的 service_month、重算原月（增量纠错）；原月**已关账**（`account-reconcile` 定版·`:147` 409 不可改）→ **不改历史定版**，补收作原月**追溯调整单列**（贡献毛利标"含追溯补收¥X"），或走反关账（理由+经手人）才改——**保护 golden 稳定 + 关账冻结**；④ 撞号见 §10.E（数据异常禁输出）。**与 #33 显式分家**：#33 的 `collected_month` = 现金/补收追踪视图；P0 的 `service_month` = 毛利配比视图，两表答不同问题、各自成立、不强并。
7. **染色归属：数据派生 + 同源双条件（§10.C·Q3+Q4）**：staining = (有真抗体 marker 或 `special_stain_count>0`) **AND `lab_revenue>0`**——marker 证"染了什么"、`lab_revenue>0`(=Σ IN 结算) 证"是我们染的"（收入含技术=同源）。**不由 scope 字段 gate**（破 Q3 全院 0-输出死循环）；**有 marker 但 `lab_revenue=0`（代阅片/纯诊断，医院自己染）→ 不减染色、移"诊断桶"**（防 Q4 误减本院没做的染色，守同源①）。`tissue_processing` gate 只管前处理：未知 → 退染色口径、标 `needs_tissue_scope`，不阻断输出。
8. **成本模型不覆盖的 IN 线一律出率（Q6/ADR-004·分子/NGS 只是其一）**：P0 成本模型**只覆盖 IHC/组织学线**（一抗/二抗/特染/组织处理料）。**准入闸 = 有 IHC 材料信号（真抗体 marker 或 `special_stain_count>0`）AND `lab_revenue>0`**；不满足的 case 从**精确贡献毛利率分子分母双剔**：
   - `lab_revenue=0`（有 marker）→ 代阅片，诊断桶（Q4/ADR-002）；
   - `lab_revenue>0` 但**无 IHC 材料信号** → **"成本模型不适用/待判"桶**——含**纯细胞病理/冰冻/宫颈液基技术线**（有技术实收、但不做 IHC → P0 算不出成本 → 否则减≈0、率虚高到≈100%）**或**缺 marker 的 IHC 案；二者用 `classifier` 再判（`case_revenue_lines.charge_code` 走逐院 config 分线）分成 `非IHC线·成本未建模` / `needs_marker`。
   院内 FISH/NGS 同此；外购转销 NGS 若有已核外包成本另桶 `售价−外包成本`。**→ 保证"贡献毛利率"是 IHC/组织学线的率，不被非 IHC 技术线（细胞/冰冻/宫颈/分子）拉虚高。**

## 5 · P0 输出（当前两层摘要 + 历史状态机制，待 #180 改版）

**当前账户对照表字段**：院级实收 / **IHC/组织学线贡献毛利(标准成本口径)绝对额** / **贡献毛利率** / 每片贡献毛利 / 每例贡献毛利(仅同院趋势) / **同账户历史趋势** / **P0率覆盖技术收入占比** / **缺价暴露率** / **①*估值占比** / **覆盖率** / `needs_tissue_scope` 占比 / 口径(完整|仅染色) / 作废 case 数 / **非IHC线实收(成本未建模)** / **数据就绪与置信标签** / **数据截止月（data-as-of）**。绝对额、率、同账户趋势必须并列；系统不自动排名、打分或生成具名账户砍院/谈价清单。组合层另展示 ∑贡献毛利、覆盖倍数与产能利用率，且不点名账户。

**优雅降级**：`染色贡献毛利(不含前处理)` 与 `贡献毛利(完整)` **分列两字段**、不合并（不变量④禁跨口径排名）。

> **以下“状态真值表 + 行动层”为历史机制，当前不得实现自动判档/自动清单**；其中数据质量、样本量、结算、持续性、置信度与乐观上界继续作为“数字能否展示、如何提示可信度”的就绪约束。#180 负责把正文完整重写为两层框架，未改完前以本节开头的当前摘要为准。

**历史状态真值表（非当前实现命令）**：原判定顺序 = 经营线闸 → 数据质量闸 → 样本/结算/持续性闸 → 经营判定。

| 序 | 闸 / 条件 | 结果 |
|---|---|---|
| **G-1 经营线未定** | `CM_TARGET` 或 `CM_MARGIN_FOR_VARLABOR` **未经 PM 正式拍板**（仍是占位默认） | **经营线未定·仅供观察**（数字[贡献毛利/率/每片CM]照出+可排序，但**不出**可留/停止候选/需谈价等强去留判定；先于 G0） |

| G2 结算 | 判定所依据的月**未关账**（暂态·补收/纠错会重算，ADR-001） | **暂态·待结算**（出临时贡献毛利、**不出强判定**；防在会翻正的月上喊砍） |
| —— 经营判定：仅当 G0–G2 全通过，且看**滚动已结算月窗口**（非单月） —— | | |
| 停止候选 | **连续/多数（≥`PERSIST_MONTHS`）已结算月**，每例贡献毛利 `< −CM_MARGIN_FOR_VARLABOR` | **停止候选**（趋势性负·非单月快照） |
| 需谈价/需关注 | 每例贡献毛利 ∈ `[−CM_MARGIN_FOR_VARLABOR, CM_TARGET)`，或单月转负**未达持续性** | **需谈价 / 需关注** |
| 需限量 | 外部 `capacity_constrained` 且 CM 正且 CM/瓶颈分钟低 | 需限量（人工置位） |
| 可留 | 每例贡献毛利 `≥ CM_TARGET`（**已留可变人力余量**）且 G0–G2 通过 | **可留**（置信度作**正交标签**必随行显示=可留·高置信 / 可留·低置信；低置信**不进**自动"继续合作"动作、只作参考——不再把"高置信"写进本闸合取，与下方 line 109"可留·低置信"消歧） |

> **每个经营判定必须挂置信度**（不再只 gate"需补数据"）：置信 = f(覆盖率, 缺价暴露率, ①\*估值占比, P0率覆盖技术收入占比, 样本量)。看板显式区分"**可留·高置信**"(①\*5%/覆盖99%) vs "**可留·低置信**"(①\*60%/覆盖85% 勉强过闸)——同为可留、可信度不同**必须可见**。
>
> **P0 贡献毛利 = 双向乐观上界（框死认知·Q7+Q9#3）**：真实贡献毛利 ≤ P0 数，两轴都只减不加——
> **轴一·可变人力**（外包/加班/计件·可避免但 P0 未建模，无三件套内数据源）；
> **轴二·标准 vs 实际浪费**（P0 用**约定价=该花多少**，看不见**实际耗**——过量领用/试剂过期/深切连切额外耗/损耗；实际耗 ≥ 标准 → 一家院可能**标准 CM 正、实际 CM 负**，P0 结构上看不见，因那是 ③实际消耗·明令不碰）。
> 故 ①"可留"只发给 CM **明确 >0 且留出两轴余量**（`≥ CM_MARGIN_FOR_VARLABOR`）的院——**近零正 = 符号不确定 → 需谈价，不判可留**；②所有 CM 标注"**乐观上界·未含可变人力·未含实际浪费(约定价口径)**"；③ 收得率校准（瓶数÷片数）只吸收**系统性**损耗的一部分，**深切/过量/过期的超额浪费仍不可见**。
>
> **具名常量**（值待 PM 拍板，先给保守默认）：`MAX_MISSING_PRICE_RATE`(10%)、`MAX_STAR_RATE`(60%)、`MIN_COVERAGE`(85%)、`MIN_LINE_COVERAGE`(P0率覆盖技术收入占比下限·70%)、`MAX_UNSCOPED`(15%)、`MIN_CASES_FOR_VERDICT`(20)、`PERSIST_MONTHS`(3·连续/多数已结算月)、`CM_MARGIN_FOR_VARLABOR`(可变人力乐观余量·按行业可变人力占比保守取·待校准)、**`CM_TARGET[合作形态/口径]`（Q10#1·按档非单值：全流程院 vs 代送加做院成本结构不同，单一全局 target 会误判——是设计位，须按合作形态分档定，与不变量④"只比同口径院"同源）**。**禁用词**：利润/盈利/亏损（贡献毛利 ≠ 利润）。
>
> **⚠️阈值未拍板降级规矩（复核补·落地为上方 G-1 闸）**：`CM_TARGET`/`CM_MARGIN_FOR_VARLABOR` 是**经营红线**，PM 正式拍板前**只用占位默认做排序/观察、绝不驱动强判定**——命中 G-1 的院月状态列输出 `经营线未定·仅供观察`，数字照出、去留结论（可留/停止候选/需谈价）**不出**。**解锁路径（不凭空定数）**：首个真实已关账 golden 院月落地后，看该月各院 `每例CM/贡献毛利率` 的**实际分布**，取"让明显【持续】亏的院落进需谈价"的位置定**初始档**（可粗、按合作形态分档），PM 签字即解锁强判定。——这把"数据未就绪→诚实降级"(G0)的同一原则，补到"判定线本身未就绪"这条此前的空白。

**行动层（Q9#1/ADR-006·P0 是行动清单不是记分牌·产品重心）**：裁决词没有"谁看/多久/做什么"就只是个数字。每个状态必须绑定行动；**重心 = 需谈价（月度主菜），非停止候选（罕见·砍院有临床/关系代价，一年可能零次）**。

| 状态 | 行动主人 | 复核节奏 | 建议动作（携带数字） |
|---|---|---|---|
| **需谈价/需关注** | 财务 / BD | **月度（主菜）** | 拿"本院哪条线/哪些抗体 CM 为负、缺口 ¥X、当前扣率 vs 目标率差"去**谈扣率/涨价/调服务范围** |
| 停止候选 | 管理层 | 季度/触发（罕见） | 连续 N 月负 → **上会评估**（计临床/关系代价），**非自动砍** |
| 需限量 | 运营 | 产能紧时 | 按单位瓶颈资源排序 |
| 需补数据 | 数据/导入 | 即时 | 补缺价/工序集/覆盖/映射 |
| 观察中·样本不足 / 暂态·待结算 | —— | 待量足/待关账 | **不行动**，等数据够/关账 |

> **默认视图 = "本月滑入需谈价的院清单 + 每家谈判弹药"，不是砍院记分牌**——财务每月打开就能拿着数字去谈，这才是 P0 的日常价值。
>
> **P0 是后视镜·非实时（Q10#3）**：靠对账单/三件套导入，天然滞后 1–3 月，叠加 G2 结算闸 + 持续性闸再往后推 → 裁决基于 **4–6 月前**的数据。对"战略级去留/谈价"可接受，但看板**必须显式标 `数据截止月`**、注"后视镜·非实时"，别让人以为反映当下。
>
> **与 `/hospital-pnl` 现有 ABC 全成本毛利的共存 = ADR-003（PARTIAL）**：保留一条瀑布（贡献毛利主、ABC 全成本为长期趋势参考）与数据就绪门控；ABC 不驱动具名账户红色、排名、砍院或谈价。原“把 KPI/排序/颜色/状态全部从 ABC 切给贡献毛利”的命令已退役——贡献毛利同样不自动发具名账户去留/谈价/限量状态。现行经营呈现只认两层框架：组合层不点名；账户层绝对额+率+同账户趋势并列，由人判断。

**A · P0 SQL 契约（收入分子 + 月轴 + join + 禁止）**

**C · 工序集 = 只解决"前处理做没做"；staining 从数据派生（Q3 破 0-输出死循环）**

死循环（已核）：`service_step_scope` 全 NULL（仅 `DatabaseManager.ts:1468` 预埋、无 resolver/写入）。若按"staining!==true 就不出" → **全院贡献毛利 = 0**，掐死 Q1 的 B 验收。
根因：**staining 不该由一个从没被写的字段来 gate——染没染，LIS 数据自证。**
```
"我们染的"判定（同源双条件·Q3 破死循环 + Q4 防误减）：
  条件甲(染了什么) = 有真抗体 marker(advice_type∈P0_ANTIBODY_ADVICE_TYPES) 或 special_stain_count>0   # marker 证"这批片染了什么"

  甲假(无 marker) → 无染色成本可减(天然 0)

service_step_scope 收窄到只承载 tissue_processing（前处理·marker 推不出，代送加做/全流程都有 marker）：

                                                     → 不进精确率、单列收入+case数，classifier 再判 非IHC线·成本未建模 / needs_marker（非 block-all）
```
**验收前置（Q1 的 B·同源不变量的被测对象）**：golden 样本**必须同时含两类 case，缺一则 Q4/同源漏洞会静默上线**：① **代送加做 case**（lab_revenue>0 + 有 marker，partner `tissue_processing=false`）→ 测"减染色、不减前处理"；② **代阅片 case**（lab_revenue=0 + 有 marker 行）→ 测"有 marker 也不减染色"（Q4 回归守卫）。填 partner 默认标志 + 备一个代阅片 case 是**被测对象本身**，非抄近路。**partner 标志一个/院、非逐 case。**
> **golden 必须钉在已关账月的冻结快照（Q10#2）**：ADR-001 定未关账月是暂态（补收/纠错会重算）→ golden 若建在未结算月，数据一重算 golden 就变、CI 抖。故 **golden 选院选月必须 `已关账` + 快照冻结**（存该月定版的 case 明细 + 期望贡献毛利），补收/反关账**不改冻结的 golden 基线**（要更新须显式换 golden 版本+留痕）。

**E · 撞号 & 作废（HIGH 4 / MEDIUM 4·现 schema 兜不住，收窄承诺）**
- **撞号（真问题=两表键不对称·跨月复用·PM Q8 拍板认）**：`case_revenue` 键含月 (partner_id,case_no,service_month)、允许跨月复用 case_no；`lis_cases` 键无月 (partner_id,case_no)、**不允许**。病理科按年/月重置 case_no 很常见 → 同院 001 在 A、B 两月各有收入行、但 `lis_cases` 只存一行，且导入是 **`ON CONFLICT(partner_id,case_no) DO UPDATE`（`lis-cases-v1.1.ts:51`）→ 后导入覆盖先导入，早月片数/marker 物理丢失**。P0 join (partner_id,case_no) → 两月收入挂到同一行成本 = **跨月成本错配 + 数据丢失**（单月看不出，最阴）。
  - **P0 现在（前置未做前）**：检测 `SELECT partner_id,case_no FROM case_revenue GROUP BY partner_id,case_no HAVING COUNT(DISTINCT service_month)>1` → 命中的 case **禁输出贡献毛利**，标 `跨月复用·需实例键`（P0 修不了覆盖丢失，只能诚实挡）。
  - **跨月硬前置（PM 认）**：① golden 手核**必须含一个跨月复用的 case_no**（否则此静默错配上线不可察）；② 要么确认院内 case_no **不跨月复用**，要么先建 `case_instance_key/accession_id`（三表同键 + 改 `lis_cases` 唯一键含月/接收号 + 改 ON CONFLICT），**属 P0 外前置**。
- **作废双剔（PM Q8 拍板：走负数行 → 冲红白捡·落死·免加列免双计）**：`case_revenue` 无 `status` 列，但冲红=负 net_amount 行归 IN，`lab_revenue=Σ(IN结算)` **已含负行** → 作废信号天然可得：
  - **全额冲红** → `lab_revenue≤0` → **已被 `lab_revenue>0` 准入闸剔除**（Q4/ADR-004，自动双剔）；计入"**作废 case 数**"，凭"该 case 有负 net_amount 行"与无负行的**诊断桶**（代阅片·ADR-002）区分。
  - **部分冲红** → `lab_revenue>0` 但含负行 → `lab_revenue` **已是净额（冲红已扣）** → 正常进率、标"含冲红调整"，**无双计**（负行已在 Σ 里扣过一次，不再另扣）。
  - → 作废识别 = 现有 `lab_revenue>0` 闸 + "有无负 net_amount 行"标记，**无需加 `status` 列**。
  - **⚠️已披露边界（复核补·M2·两引擎均点名）**：负 net_amount 行**只存在于 `case_revenue_lines`**（`case_revenue` 落库时已按 (partner_id,case_no,service_month) 净额聚合成单行、无负行、无 `status` 列），而 `case_revenue_lines` **不在 §1"只用这些"白名单**（且该表**无 partner_id 列**，匹配键=case_no+service_month）→ "凭有无负行区分 全额冲红 vs 代阅片(诊断桶)"**当前查不到数据源**、两类会混桶。**二选一处置**：(a) 把 `case_revenue_lines` 显式纳入 §1 白名单 + 判据 `EXISTS(SELECT 1 FROM case_revenue_lines l WHERE l.case_no=cr.case_no AND l.service_month=cr.service_month AND l.net_amount<0)`；(b) 未纳入前把"作废 case 数"与"诊断桶 case 数"**合并为单一 `lab_revenue≤0·移出染色桶`**、不单列作废数。**不碰钱**（两类都被 `lab_revenue>0` 准入闸挡在毛利计算外，仅两个统计字段互相污染）。

**F · 措辞（MEDIUM 5）**：把"表已建/今天能出真数"改为"**schema initializer 具备这些表；导入三件套(对账单+工作量表+抗体清单)后，才能对指定院月产 P0 结果**"。签入 `data/coreone.db` 快照里这些表是空的（codex 实测：case_revenue/lis_case_markers/antibodies 均 MISSING）——不能直接 join 出真数。
