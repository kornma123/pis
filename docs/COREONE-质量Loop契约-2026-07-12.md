# COREONE 质量 Loop 契约

> **状态**：v1.0（2026-07-12，经一轮质疑关五问收敛）· 待 PM 定稿拍板
> **从属**：本契约从属于 ① `docs/工作模型-通用版-…` 与 `…-COREONE项目版-…`（方法论主线）、② `docs/agent-operating-contract.md`（跨工具协作契约）。冲突时一律以那两者为准。本契约只做一件事：把质量 Loop 家族的**共享不变量**集中到这一处。
> **单一事实源红线**：五件套、六步骨架、假设两级、异步节奏、证据分级、复核堆栈、护栏——这些**共享规则只此一份**。5 个阶段入口（PRD/Mockup/写码/真跑验收/报告结论）与总览**不得复制或改写本契约的任何规则**，只能指回。凡在别处重述本契约规则的文本，视为漂移，按护栏 §7 的 drift-guard 处理。

## 0. 家族地图（读这一段就知道去哪）

```
docs/COREONE-质量Loop总览-…      ← 纯路由：哪个阶段走哪个入口
  └─ 本契约（共享规则，唯一源）
  └─ 5 个薄阶段入口（只放该阶段的增量 + 会话注入块，指回本契约）：
       PRD质量Loop / 前端Mockup质量Loop / 写码质量Loop / 真跑验收质量Loop / 报告结论质量Loop
```

**主链**：想法 → PRD → Mockup → 写码 → 真跑验收 → PM 验收拍板。**横切**：报告/结论 Loop（任何调研/审计/评估/复盘）。

## 1. 六步骨架（每一轮都走，多数由 AI 独自走）

```
① 产出 / 修订
② 自检（该阶段的清单，指回权威源，不在此复制）
③ 摊假设（按 §3 两级标记）
④ 自我质疑（先攻击自己的稿子，能修当轮修）
⑤ 写台账（本轮五件套按 §2 累积进台账，不逐轮投递 PM）
⑥ 到闸点才清算（按 §4，只在 5 个闸点找 PM）
```

「一轮」是 AI 的内部单位，不等于「找 PM 一次」。二者的解耦是 §4。

## 2. 五件套（定义 + 适用边界）

每轮固定产出**五件套**：① 本体　② 本轮改动清单　③ 假设清单（含 §3 分级）　④ 讨论项（修不掉/拿不准）　⑤ 结尾大白话（非技术 PM 看得懂：做了什么→结果→对业务/用户意味什么）。

**适用边界（第 2 问补充证据逮到的止血 —— 五件套撞机器闸）**：

- **无机器闸的阶段交接**（PRD/mockup 的轮次交付）：五件套是净增益，照产、累积进台账。
- **有机器闸的交付面（PR body）**：**唯一字段源 = `docs/agent-handoffs/TEMPLATE.md` + `scripts/issue-handoff/check-pr-body.cjs`**（后者对每个 PR 打 `issue-handoff-contract` 状态、硬性强制字段）。写码阶段**不得自造「五件套」字段名**，一律用机器闸/模板的真实名字（**字节精确**，将来 drift-guard 会按字节 grep 本契约）：「改动清单」→ required 标题 `变更摘要`；「测试与复核证据」→ 既有字段 **`测试与真数据 / golden 证据`**（归在 required 标题 `验证` 下·**是斜杠不是中点**）；「大白话」→ `PM 大白话`（**模板/文化字段·非 `check-pr-body` required 字段**，机器闸不查它、但契约文化仍要求留）。前两者都是既有字段被改名重造，删。Loop 真正增量的两颗牙——**偏离清单**、**变异证有牙**——作为**内容要求**并入模板既有字段；若要升为独立机器字段，正确路径是「改模板 + 改 `check-pr-body.cjs` 一起动」（治理即代码·单一源），那是独立拍板项。

## 3. 摊假设 · 两级（级别决定能不能带着走）

- **细节级**（默认值、排序、文案措辞、入口位置）：AI 按最优解读**带标记前进**——台账记「此处基于未拍板假设 X」，产物上留痕（PRD 段落标 ⚠️ / mockup 贴角标 / 代码走 PR 偏离清单）。走错代价 = 草稿级返工。
- **方向级**（做不做、给谁做、核心口径选型）：**禁止带标记前进**。方向错则整段作废；等不起就切并行线，不硬闯。方向级即「3 轮不收敛 → 并排出候选」的对象——出候选本身异步友好：PM 回来面对的是选择题不是论述题。
- 分级判断本身由 AI 做、会判错 → **缓解**：台账里每条假设必须自标级别，闸点清算时 PM 看到的不只是假设、还有**分级本身**（分级也在被审）。此层有残余风险，不声称为零。

## 4. 异步节奏条款：轮次自主，闸点等人

**病根**：原草稿把「沉默不算同意 / 每轮 PM 三分类」摊到每一轮，假设 PM 每轮在场。真实工作流是 Codex + Claude 在各自 worktree 异步跑、PM 只在决策点出现。等不起的地方立神圣规矩 = 规矩必死。

**规则**：
- **轮次自主**：①–⑤ 由 AI 独自连跑 N 轮，不需要 PM 出现；产出累积进台账。
- **闸点等人**：「沉默不算同意」只在这 **5 个闸点**神圣——
  1. **PRD 定稿**　2. **mockup 定稿**　3. **合并**（PR 进 master）　4. **验收通过**　5. **接受 grade C 口径 / 碰钱口径拍板**
- **现实对齐**：5 个闸点里**只有「合并」今天有机器门**（required checks + PM 合并权，见 §6 与 §8）。散文要和这个现实对齐，不假装每轮有一道人肉门。其余 4 个闸点暂靠台账纪律 + 对抗复核撑（「定稿状态可机器查」是登记在案的后续增强，见 §9，现在不 overclaim）。
- **拍板包**：闸点到达 / 台账出现方向级待拍项 / PR 开出时，AI 把台账整理成一张拍板包，按「阻塞闸点的 / 影响方向的 / 细节可后补的」排序，每项带推荐 + 候选——接项目已有的 PM 待拍板模式。闸点清算时 PM 的反馈仍按**拍板 / 要改 / 新讨论**三分类归位（这是原六步 ⑥ 的下落）。
- **卡点切线**：被闸点卡住而 PM 不在 → 切到不依赖该拍板的并行线，或挂起并留自包含 handoff（都是项目已有纪律，Loop 不与之打架）。
- **台账 = 异步接口**：台账是 AI 轮次与 PM 决策点之间的接口。字段见 §5，运行时角色见本节。

**为什么这不是把「自信地一起错」从后门放回来**：旧病要害 = 错得**不可见** + 错误**穿透到最贵处**才爆炸。本设计里：解读全部显式挂台账带标记走（可见）；闸点清算时「未拍板却当已拍用了」会被台账对出来（定稿不许有未拍残留 = 清算规则）；最贵的穿透点（合并）有机器门（不穿透）。**最坏情况从「错到 master」降级为「一段草稿白做」。**

**通用定稿退出模板（所有阶段共用 · 入口只准追加、不得重写这四条）**：任何阶段把产物标「定稿 / 通过」前，四条全绿——
1. **PM 在对应闸点明确说「定稿 / 通过 / 验收通过」**（沉默不算）；
2. **台账零未拍残留**（含 §3 假设分级、§5 未签的 grade-C）；
3. **该阶段自检清单全绿**（清单住在各入口、指回权威源，不在此复制）；
4. **最近一整轮 PM 零「要改」零「新讨论」**。

各阶段只在此模板上**追加**本阶段特有项（如 PRD +「每功能点有真数据验收例」、验收 +「口径台账等级/源未被偷改」），**不得重写这四条**。退出条件是共享不变量、只此一处——这正是防「5 个入口各自重写退出条件、镜像病原样长回」的锁。

## 5. 证据分级 A/B/C（碰钱/口径档的退出条件）

**这是把 PRD 背景知识审计逮到的 #1 病灶——「约定常量顶替真值、纸面 done 冒充 real」——变成每道门必须回答的 yes/no。**

- **每个关键数字**（进决策、进计算引擎的参数）在口径台账里必须带三个字段：**值、等级、记名验证源**。写成 `A（源：和睦家2026-05对账单）` / `B（源：国标文件）` / `C（借用ICL占比·推测）`。**记名源是反通胀的钉子**：等级旁挂着源，`A（仅和睦家）` 就骗不了 n≈1，不必另发明样本量分级。
- **A 的反通胀定义（写死，不许软化）**：`A = 对外部真实数据核过`。**算术自洽不算 A、测试绿不算 A、黄金锚绿不算 A**（黄金锚锁「结果没变」不是「结果正确」）。审计里 ¥27,870 那一类正是靠自洽混进「已验证」的。
- **退出条件**：碰钱/口径的 PRD 里，任何**支撑决策**的数字仍是 B/C 时，**不许标「口径定稿」**——除非 PM 在闸点明写「接受 grade C（未验证）」并留痕。
- **接受 C 自动入队**：每一次「PM 接受 grade C」的留痕，自动走 surfaced-items 规矩（`docs/github-issue-pr-management-loop.md` / `.claude/workflows/surface-to-issues.js`）去重、起草成**校准 Issue 候选**（draft-then-confirm，PM 过目后开）。每个被放行的 C 都有编号、有队列、有认领路径。
- **「口径校准任务」= 报告 Loop 的具名标准靶型**：定靶 = 指定口径升 A；退出 = 升到 A，**或**明确「升不动 + 为什么 + 还差什么数据」。给显式性一个零成本替身，不新增文档。
- **验收侧只复核「等级/源没被偷改」**（没人在实现中拿测试自洽把 C 悄悄改标 A）；**真值验证本身不塞进验收快门**——慢工作离开门、进队列（焊进快门只会门被焊死或人造假过门）。
- **诚实边界（写死，防下个会话吹牛）**：**本契约不制造数据**。PM 可以对每个 C 都签「接受」，系统会退化回今天的状态——但两点不同：每次拿约定值凑数都是 PM **睁眼签的字**，且每个 C 都在队列挂着号。审计逮到的病是「暗替」，本条把它变成「明签 + 挂号」。这是流程的上限，剩下那截是拿数据的业务动作，流程装不下、也不假装装得下。
- **范围**：**存量 B/C 口径**（审计已逮出那批）走队列消化，新规**不追溯翻旧账**；新规管往后每一次定稿。

## 6. 复核堆栈 L0 / L1（替换旧草稿 5 处散写的「强制异构复核」）

> 事实核过（本会话开卷 · file:line）：项目已有一道机器 AI 复核门，但**今天不在拦合并的 required 集合里**（`branch protection required = ["vitest","gate"]`）。契约照实写，不吹成已 required。

**L0 = `ai-review-gate`（DeepSeek，每个非 draft 的 master PR 自动跑，机器）**
- **边界（照实）**：patch-only、**不 checkout PR 码、不跑测试、无文件系统/网络**（`.github/codex/ai-review-config.toml`、`ai-review-prompt.md:10-12`）；二进制/submodule 变更不审；**超 200 文件 / 2 万行 / 2MB 直接拒审**（`ai-review-gate.yml:190`）；P0/P1 发现 → verdict FAIL → 门红（`ai-review-prompt.md:33-34`）。fail-closed、SHA 钉死、只 COMMENT 不 APPROVE。
- **消费义务（现在缺失，补上）**：① 写码交付必须**逐条处置 L0 的 review 评论**（修 / 讨论项 / 误报附反证——三分类）；② **L0 红 = 机器关未过，与 vitest 同级对待**。
- **它不是碰钱档的对抗面板，永远不许折抵 L1。** 现状约束力靠「不合红 PR」的习惯，不是机器——「L0 进 required 集合」是登记的 PM 拍板项（见 §9）。

**L1 = 碰钱/口径/权限档的定向对抗复核（人工第二视角）**
- **触发**：仅碰钱/口径/权限档 PR。**多镜头**：口径对黄金锚、数值边界、权限对运行库、SoD。
- **引擎须与实现方异构**：codex（`codex exec -s read-only`）可用时首选；**Claude Workflow 对抗面板是文档在案的等价兜底**（codex 长请求易断流，见 `codex-cli-usage.md`——散文别再假装只有 codex 一条路）。
- **两态制（反造假焊接）**：L1 只有两个合法终态——**完成**（PR 内留定式产物：镜头清单、每镜头结论、CONFIRMED/REFUTED 计数、run 指针）/ **未做**（写明原因，闸不过）。「断流→跳过→已复核」这个第三态**按定义取缔**：无产物 = 未做；谁写「已复核」而 PR 里没有产物，台账上直接可证伪。

**双向不折抵（一句写死）**：L0 绿 **不构成** L1 已做；L1 做过 **不豁免** L0 findings 的逐条处置。

## 7. 共享护栏（防跑偏 / 空转 / 打转 / 偷改 / 漂移）

| 护栏 | 规则 |
|---|---|
| 防空转 | 每轮必有可见差异；连续 2 轮无实质变化 = loop 卡住，升级 |
| 防打转 | 同一议题第 3 轮未收敛 → 停止再猜，并排出候选（方向级尤其） |
| 防偷改 | 已拍板决定不许在后续轮悄悄改回 |
| 防无效轮 | 缺假设清单 / 缺自检 = 无效轮，重交 |
| **drift-guard（漂移锁）** | 本家族配最小**负控判别器**：Loop 文档里 grep 到黑名单词条枚举、或出现自造 PR 字段名 = **FAIL**，挂进已有 preflight / 纪律闸。「契约 + 闸」才是 PR#121 的完整教训，光有契约不够——见 §9 机器焊队列。 |

## 8. 落地映射（哪些焊现有闸，哪些纯纪律）

| Loop 产出 | 落地方式 |
|---|---|
| PR 交付字段（写码交付面） | 焊 `check-pr-body.cjs`（已 required：`issue-handoff-contract`） |
| 开工核实 / 越界脏文件 / dev-DB 复原 | 焊 `scripts/agent-preflight.cjs` + guardrails |
| 幽灵 404 / 无消费者端点 | 焊 build-discipline 闸（已 required：`gate`） |
| 机器 AI 复核 | 焊 `ai-review-gate`（L0，**未 required**·PM 拍板项） |
| 黑名单 / 设计令牌一致性 | 指针化到权威源（前端标准 §4.2 术语表 / `V1.1设计稿/v1.1/DESIGN.md`）+ drift-guard 负控 |
| 摊假设 / 自我质疑 / 大白话 / 真跑验收人视角 / L1 对抗结论质量 | **纯纪律**（机器测不了，保持精简散文；质量靠 L0×L1 双引擎对表 + PM 抽查 + 报告 Loop 防断言通胀） |

## 9. 诚实边界（写死，防吹牛）

1. 本契约**不制造数据**（§5）；不验**复核质量**，只能验「产物存在」——敷衍的面板也能留漂亮产物，质量靠 L0×L1 对表 + PM 抽查 + 报告 Loop 防通胀。
2. **L0 今天不在 required 集合**；「定稿状态可机器查」（文档标定稿但台账有未拍项 = FAIL）暂无机器门。二者都是登记的**后续增强**，不 overclaim。
3. **登记的机器焊队列**（surfaced-items 起草成 Issue 候选 → PM 过目 → 独立小 PR → RED→GREEN 真探针）：(a) drift-guard 负控；(b) L1 产物 presence-check（碰钱路径变更 → 缺 L1 产物节 = FAIL）；(c) 偏离清单是否升 `check-pr-body` 独立字段（改模板 + 闸脚本一起动）；(d) 总览挂进 preflight `AUTHORITY_FILES`。
4. **登记的 PM 授权项**：`ai-review-gate` + `ai-review-integrity` 进 required 集合（代价：DeepSeek 抖动会卡合并 → 走 pr-governance §5「平台故障需 PM 授权例外」）。

## PM 大白话

这份契约是那套质量循环的「共用规则总闸」，把原来抄在 6 份文档里、已经开始各改各的那批规矩（每轮交什么、假设怎么摊、涉及钱的口径要不要真数据核过、谁来第二遍挑刺），收敛成**只写一遍**，别的文档只指过来。三件最要紧的事焊死了：① 涉及钱的每个数字都得标明「是真数据验过的、还是听来的、还是猜的」，还得写出处，猜的想过关你必须**亲手签字接受**、而且自动排进待办队列——审计逮到的「拿约定数悄悄当真数」从此藏不住；② AI 不再假装每一轮都等你拍板（现实里你不在），改成它自己连着干、只在**五个真正的岔路口**才来找你，最坏也只是「一段草稿白做」，不会错到正式代码里；③ 涉及钱的第二遍挑刺不许用「已复核」三个字糊过去，没留下挑刺记录就当没做、卡住不放行。剩下机器能自动查的都接到你**已经在跑**的检查上，机器查不了的（比如挑刺认不认真）老实标成「靠人盯」，不吹成自动门。
