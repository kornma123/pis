# COREONE 质量 Loop 总览（唯一入口）

> **状态**：v1.0 草稿（2026-07-12）· 待 PM 质疑/拍板后生效
> **用途**：质量 Loop 家族的**唯一路由入口**。要找「这个阶段该走哪个循环、注入块在哪」，看这一页就够；规则细节以各 Loop 文档为准，**本页不复制任何规则细节**（防两处文本各改各的）。

## 主链（做一个功能的完整路径）

```
想法 / 痛点
  → PRD 质量 Loop           docs/COREONE-PRD质量Loop-2026-07-12.md
  → Mockup 质量 Loop        docs/COREONE-前端Mockup质量Loop-2026-07-12.md
  → 写码质量 Loop           docs/COREONE-写码质量Loop-2026-07-12.md
  → 真跑验收质量 Loop       docs/COREONE-真跑验收质量Loop-2026-07-12.md
  → PM 验收拍板 = 完成（完成 = 消费者被服务）
```

**横切**（不在主链，随时插入）：报告/结论质量 Loop——任何调研、审计、评估、复盘类产出。`docs/COREONE-报告结论质量Loop-2026-07-12.md`

每一段的出口都是「PM 拍板的定稿」，它同时是下一段的**唯一输入基准**；下游发现上游定稿有问题，回上游重拍，不许自由发挥。

## 路由表

| 你要做的事 | 走哪个 Loop |
|---|---|
| 有个想法/痛点，要变成需求 | PRD Loop |
| 需求定了，要画界面 | Mockup Loop |
| 界面定了，要写代码（含 bug 修复） | 写码 Loop |
| 代码合了，要确认真的能用 | 真跑验收 Loop |
| 要一份调研/审计/评估/复盘结论 | 报告/结论 Loop |
| 纯执行小事（改错别字、调颜色） | 各 Loop 内的轻量道（见各文档分档） |

## 全家族共同规矩（速记，细则以各文档为准）

- 每轮交付**五件套**（本体 + 改动清单 + 假设/偏离清单 + 讨论项 + 大白话）
- **摊假设**：AI 替 PM 做的每个决定都标「待拍板」
- **沉默不算同意**：PM 明确说「定稿 / 通过」才出循环
- **3 轮不收敛就并排出候选**让 PM 选，不许一直猜
- **已拍板不偷改**；每轮必须有可见差异；缺假设清单或自检 = 无效轮
- 碰钱 / 口径 / 权限的东西，各 Loop 都有专属加固节，强制走

## 有意不建的 Loop（摊出来，防止被当成遗漏）

| 候选 | 不建理由 |
|---|---|
| 讨论循环 | 已内嵌在每个 Loop 的「新讨论 → 继续循环」里；单列会造成两处规则镜像 |
| DB 迁移 | 已有 /db-migration 技能 + CLAUDE.md 数据库规范；缺的不是循环是照做 |
| E2E 编写 | 已有 /e2e-testing 技能；e2e 回归网可信度重建是已登记的 PM 待拍项，不归 Loop 管 |
| 事故/操作安全 | 属操作铁律层（契约 / guardrails），是红线不是循环形态 |

## 给 PM 的大白话

这一页是全家的地图：从一个想法到功能真正能用，中间要过四道循环（写需求 → 画草稿 → 写代码 → 真跑验收），每道都是「AI 自查 → 摊假设 → 你拍板」转圈，你不点头不放行；另外有一道管所有「给结论的报告」。这页只管指路，具体规矩在各自文档里。哪些环节故意没建循环、为什么，也写在上面了，免得以后误以为是漏掉。六份文件现在都是草稿，等你逐份质疑、拍板后才算生效。
