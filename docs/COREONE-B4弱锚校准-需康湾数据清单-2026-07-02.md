# COREONE B4 · 弱锚校准 —— 需要康湾哪些数据（交 PM）

> **日期**：2026-07-02 | **线**：F（逐抗体成本弱锚校准）| **性质**：数据需求清单 + 摊算方法 + 写回留痕说明
> **配套**：设计基线 §1.3；未决清单 B4；实现见 `后端代码/server/src/utils/antibody-cost.ts` + `routes/antibody-cost-v1.1.ts`。
> **一句话**：逐抗体"算全"里，**工时 + 设备折旧**这半是 **G2 估**（占逐片成本约 52%，设备最弱），是成本 band 里**最不牢的承重点**。机制已就绪——只要康湾给下面几项真实数据，系统就能**摊算写回**并把状态从"G2 估·待校准"翻成"已校准"，且**全程留痕**。**真值待 PM 补，本文只列要什么、怎么算、写到哪。**

---

## 1. 现状（诚实边界）

- 一抗、二抗/显色 = **台账真价**（牢）。
- **工时 `labor_per_slide` / 设备 `equipment_per_slide` = G2 估占位**（默认 ¥8 / ¥3，`ihc_cost_params` 表 `source='G2估' confidence='粗估'`）。
- 系统**不冒充精确**：`/cost-preview` 与 `/cost-params` 一律透出 `laborEquipmentSource='G2估'` + 备注"待校准"，前端如实标注。**校准前绝不显示为"精确成本"**。

## 2. 需要康湾提供的数据（校准输入）

| # | 数据项 | 用途（对应字段） | 取数来源建议 | 必填 |
|---|---|---|---|---|
| 1 | **月技师人力总成本**（工资 + 社保 + 福利，免疫组化组口径） | `monthlyTechnicianCost` | HR 工资表 / 财务人力成本表，按免疫组化组人头分摊 | ✅ |
| 2 | **月设备折旧**（染色机 / 切片机 / 脱水机 / 烤片机等） | `monthlyEquipmentDepreciation` | 财务固定资产折旧表（原值 × 年折旧率 ÷ 12），或设备清单自算 | ✅ |
| 3 | **月免疫组化产片量**（分母） | `monthlySlideVolume` | LIS / 账单免疫组化片数（就是账实核对里的物理片数口径） | ✅ |
| 4 | **月房租 / 水电等固定摊**（可选） | `monthlyFacilityCost` | 财务房租水电，按面积或人数分摊到免疫组化 | ⬜（缺省 0） |
| 5 | **房租等固定摊归工时 vs 设备的分配比**（可选） | `facilityToLaborRatio` | 经验/协商，默认 0.5（一半归人、一半归机） | ⬜（缺省 0.5） |

> **进阶（可选，更精确）**：若做**工时测量**（time-and-motion：一片从上机到出片的实际技师分钟数 × 单位人力成本），可替代第 1 项的"月总成本 ÷ 片量"均值法，得到逐工序工时。目前机制先用月度摊算均值，够用即可，不阻塞。

## 3. 摊算方法（"回归"= 成本摊算，不是统计回归）

```
每片工时  labor_per_slide     = (月技师人力成本 + 房租摊 × 分配比)       ÷ 月产片量
每片设备  equipment_per_slide = (月设备折旧     + 房租摊 × (1−分配比))   ÷ 月产片量
```

- 分母守卫：月产片量必须 > 0，否则接口 **400**（不产 NaN/Infinity）。
- **示例**：月人力 ¥40,000、月折旧 ¥6,000、房租 0、月产 2,000 片 → 工时 **¥20/片**、设备 **¥3/片**。
- 结果仍是**月度摊算均值**，非逐片实测；比 G2 估牢，但如需更细走上面的工时测量进阶。

## 4. 写回路径 + 留痕

**接口**：`POST /api/v1/antibody-cost/cost-params/calibrate`（权限 `antibody_cost:W`，财务/管理员）

请求体：
```json
{
  "monthlyTechnicianCost": 40000,
  "monthlyEquipmentDepreciation": 6000,
  "monthlyFacilityCost": 0,
  "monthlySlideVolume": 2000,
  "facilityToLaborRatio": 0.5
}
```

写回效果：
1. `labor_per_slide` / `equipment_per_slide` 值 = 摊算结果；
2. `source` → `康湾实测校准`，`confidence` → `已校准`，`remark` 记本次输入；
3. **留痕**：`abc_audit_logs` 落一条 `module='antibody_cost' action='calibrate'`，`detail` 含 **before/after + 输入 + 方法**，`operator=` 操作人（碰口径的写，与关账/成本核算同口径）；全站 `operation_logs` 另由 `auditWrite` 中间件自动落"谁在何时改了什么"。
4. 之后 `/cost-preview` 的 `laborEquipmentSource` **自动翻成"已校准"**——不再显示 G2 估。

**手工微调**（非整套校准）：`PUT /cost-params/:key`，可带 `{ value, source, confidence, remark }`，同样落 `param_update` 留痕。（此前该接口丢弃 confidence/remark，导致手工改后置信永远卡在初值——本线已修。）
