# ADR-001 · P0 院级贡献毛利的归集月轴 = 权责发生制（service_month）

> **日期**：2026-07-04 | **状态**：已采纳（PM 拷问 Q2 拍板）| **性质**：hard-to-reverse（决定整条 P0 lane 的归集轴 + golden 怎么切月）
> **关联**：P0 spec §4⑥ / §10.A；补收线 #33；关账状态机 `account-reconcile-v1.1.ts`。

## 背景

P0 逐 case 贡献毛利 = 实收（`lab_revenue`） − 可避免材料成本。「一个 case 算进哪个月」有两种会计基础：
- **权责发生制**（accrual）：算进**服务发生月** = `case_revenue.service_month`。
- **收付实现制**（cash）：算进**钱到账月** = `collected_month`（补收线 #33 用它）。

初版 P0 spec 自相矛盾：不变量⑥① 写 `collected_month`，§10.A/§1 写 `service_month`。核实 schema：`case_revenue`（`DatabaseManager.ts:427/1394-1434`）**只有 `service_month`、无 `collected_month` 列、无 `status` 列**。

## 决策

**P0 归集月轴 = `case_revenue.service_month`（权责发生制 / 配比原则）。单轴，不用 `collected_month`。**

理由（决定性的一条）：**成本没有"到账月"这个概念**——材料在切片染色那一刻消耗，物理上属服务月。贡献毛利（实收 − 成本）要在同一个月自洽，**分子分母必须同轴**；成本天然在 service_month，故收入也必须落 service_month。`collected_month` 只对"现金/补收回收报表"有意义，对"毛利配比"无意义。且 `case_revenue` 物理上只能按 service_month。

**同轴机制**：成本经 case_no join 到该 case 的 `service_month`；`lis_cases.operate_time` **仅异常提示**（`operate_time≠service_month` → 标数据异常，不改归月）。`service_month` 为 NULL 的 case → `needs_data`，不进月度贡献毛利。

**补收（关账感知，防撞关账冻结）**：
- 原月**未关账** → 补收回填原 case 的 service_month、重算原月（增量纠错架构·派生值可重算）。
- 原月**已关账**（`account-reconcile` 定版不可逆，`:147` 返回 409 PERIOD_CLOSED）→ **不改历史定版**；补收作原月**追溯调整单列**（贡献毛利标"含追溯补收¥X"），或走反关账（必填理由+经手人）才改。

**与 #33 显式分家**：#33 的 `collected_month` 视图 = 现金/补收追踪报表；P0 的 `service_month` 视图 = 毛利配比报表。**两张表回答不同问题，各自成立，不强行统一。**

## 后果

- ✅ 贡献毛利分子分母同轴、配比自洽；成本无需伪造"到账月"。
- ✅ P0 golden 按 service_month 切月，**稳定**（不会被后到补收改写已关账月）——保护 Q1 验收锚。
- ✅ 不违反已落地的关账冻结（避开 409）。
- ⚠️ 已关账月的补收 = 追溯调整单列，不进原月定版 → 现金视图（#33）与配比视图（P0）在"补收归月"上**故意不一致**，需在看板/文档说明二者答不同问题。
- ⚠️ 需要 `service_month` NULL 的兜底（needs_data）+ `operate_time≠service_month` 的异常提示，二者都要落实现。

## 备选（未采纳）

- **collected_month（收付实现制）**：否——成本无到账月，分子分母不同轴，毛利不配比；且 `case_revenue` 无此列。
- **operate_time 定成本月 + service_month 定收入月（双轴）**：否——切片月与开单月可能不同，分子分母跨月，毛利错期。
- **强行统一 P0 与 #33 到一个月轴**：否——两者答不同问题（配比 vs 现金），统一必然牺牲其一。
