> **状态**：v1.0（2026-07-12）· 待 PM 定稿拍板
> **从属**：本文是《`docs/COREONE-质量Loop契约-2026-07-12.md`》在 **mockup 阶段**的薄入口 + 《COREONE-前端标准-流程质量设计文案UX-2026-06-27》「mockup 先行·未定稿不写真代码」红线的执行细化。**六步骨架/五件套/假设两级/异步节奏/退出模板/复核堆栈全在契约、本文不复制；视觉令牌以 `V1.1设计稿/v1.1/DESIGN.md` 为唯一源、文案以前端标准 §4.2 术语表为唯一源，本文一律指针不抄值。** 冲突以前端标准与契约为准。
> **适用**：新建/大改前端页面/流程。已定稿页面小修、纯文案微调不走本 loop。

## 0. 位置

需求定稿 →【Mockup（本文）】→ PM 定稿 → 写真代码 → 真跑验收。
使命：写任何真代码前，把「AI 理解的」和「PM 想要的」偏差磨到零——返工发生在改草稿（几分钟）不是改代码（几小时）。

## 1. 本阶段增量（契约之外，Mockup 特有）

**① 产出**：用**真实业务样例数据**（真项目名/真金额量级/真病理号格式），禁 lorem、禁「测试1/abc」（假数据掩盖布局和口径问题）。一页至少覆盖**三态：正常有数据 / 空态（第一次进来） / 出错**；涉角色权限差异补「低权限用户看到什么」。HTML/组件式 mockup 的输入框**显式白底 + 深色文字 + 浅灰 placeholder**（防宿主深色模式刷黑、输入不可见）。修订轮开头一句「这轮改了什么」。

**② 自检清单（过完贴勾选结果，指回权威源不抄值）**：
- 视觉：**过 `V1.1设计稿/v1.1/DESIGN.md` 令牌**（字体/圆角分级/主色/文字色/边框/阴影/按钮高/focus 光圈）——**本文不再罗列具体数值**（数值唯一源在 DESIGN.md，抄 = 镜像漂移）。无 0 圆角、无纯黑、无大面积刺眼饱和色。
- 文案：**过前端标准 §4.2 术语表（用左/弃右）**——本文不枚举黑话词表。按钮是动词短语（「保存修改」不是满屏「确定」）、错误提示告诉用户**下一步怎么办**、空态有引导不是一片空白。
- 流程：从哪进这页（导航/别页入口）/ 每个操作完成后去哪看到什么反馈 / 危险操作有确认或可撤销 / 不同角色分别看到什么（以**运行库真实权限**为准，别按源码种子矩阵想当然）/ 同类交互与现有页面一致。

## 2. Mockup 特有退出追加项（叠加契约「通用定稿退出模板」，不重写那四条）

- 必备三态（正常/空态/出错，含权限视角如适用）都有稿。

**定稿动作**：归档最终稿（HTML/截图 + 拍板记录，写日期版本）。**这份定稿 = 写码阶段唯一验收基准。**

## 3. 碰钱/口径页面加固（叠加契约 §6）

对账/成本/毛利/预算类页面，定稿前加一道 **L1 定向对抗复核**（`codex exec -i <定稿截图>` 视觉审 / Claude 对抗面板，二者等价·留定式产物，见契约 §6）。

## 5. 给 PM 的大白话

做前端先用草稿图循环打磨：AI 每轮自己过三张清单、把替你做的决定摊出来让你拍、自己先挑毛病，才交你看。你没说「定稿」前不写正式代码。颜色圆角这些具体数字**不再抄在这份文档里**（只指向唯一的设计令牌表，免得两处对不上），文案统一过术语表。返工都发生在改草稿上，最后写出来的代码长成你拍过板的样子。
