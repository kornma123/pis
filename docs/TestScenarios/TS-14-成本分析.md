## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| COST-01 | Given admin 用户已登录，When 调用 GET /api/v1/reports/cost-by-project，Then 返回 200，包含 summary 和 projects 数组 |
| COST-02 | Given 按项目成本报表，When 检查 projects 数组，Then 每条记录包含 projectId、projectName、totalCost、sampleCount、unitCost、ratio |
| COST-03 | Given 项目成本 ratio 计算，When 检查返回值，Then ratio = (project.totalCost / summary.totalCost × 100).toFixed(1)，保留 1 位小数 |
| COST-04 | Given 按物料成本报表，When 调用 GET /api/v1/reports/cost-by-material，Then 返回 200，materials 数组包含 consumption、totalCost、ratio |
| COST-05 | Given 按供应商成本报表，When 调用 GET /api/v1/reports/cost-by-supplier，Then 返回 200，suppliers 数组包含 amount、orderCount、ratio、status |
| COST-06 | Given 供应商成本中 status 字段，When 检查返回值，Then 固定返回 "long-term" |
| COST-07 | Given 按日期范围筛选 startDate="2026-01-01"&endDate="2026-12-31"，When 调用成本报表接口，Then 返回该日期范围内的数据 |
| COST-08 | Given 按物料分类筛选 categoryId="试剂类ID"，When 调用 GET /api/v1/reports/cost-by-material?categoryId=xxx，Then 仅返回该分类下的物料成本 |
| COST-09 | Given pathologist 用户，When 调用 GET /api/v1/reports/cost-by-project，Then 返回 200，正常读取 |
| COST-10 | Given finance 用户，When 调用 GET /api/v1/reports/cost-by-material，Then 返回 200，正常读取 |
| COST-11 | Given warehouse_manager 用户，When 调用 GET /api/v1/reports/cost-by-project，Then 返回 403 Forbidden |
| COST-12 | Given technician 用户，When 调用 GET /api/v1/reports/cost-by-supplier，Then 返回 403 Forbidden |
| COST-13 | Given procurement 用户，When 调用 GET /api/v1/reports/cost-by-project，Then 返回 403 Forbidden |
| COST-14 | Given 无数据时调用成本报表，When 调用 GET /api/v1/reports/cost-by-project，Then 返回空数组，summary.totalCost=0 |
| COST-15 | Given 无 Token 请求，When 调用 GET /api/v1/reports/cost-by-project，Then 返回 401 Unauthorized |
| COST-16 | Given 过期 Token，When 调用 GET /api/v1/reports/cost-by-material，Then 返回 401 Unauthorized |
| COST-17 | Given 伪造 Token，When 调用 GET /api/v1/reports/cost-by-supplier，Then 返回 401 Unauthorized |
| COST-18 | Given 项目成本 changeRate 字段，When 检查返回值，Then 固定返回 0 |
| COST-19 | Given 项目成本 changeDirection 字段，When 检查返回值，Then 固定返回 "down" |
| COST-20 | Given 成本数据来源验证，When 创建出库单后，Then 按物料成本报表中该物料的 totalCost 增加对应金额 |
| COST-21 | Given 成本数据来源验证，When 创建入库单后，Then 按供应商成本报表中该供应商的 amount 增加对应金额 |
| COST-22 | Given 比率总和验证，When 检查按项目成本报表，Then 所有项目 ratio 之和 ≈ 100（考虑四舍五入误差） |
| COST-23 | Given 响应结构检查，When 调用成本报表接口，Then 返回 success=true，data 包含 summary 和列表数组 |
| COST-24 | Given 日期筛选结束日期，When 调用 GET /api/v1/reports/cost-by-project?endDate=2026-05-12，Then 自动包含 2026-05-12 23:59:59 的数据 |
| COST-25 | Given SQL 注入 payload（startDate="' OR '1'='1"），When 调用 GET，Then 返回空结果或正常过滤，无 SQL 注入漏洞 |
| COST-26 | Given 超长日期范围（startDate="2020-01-01"&endDate="2030-12-31"），When 调用 GET，Then 返回该范围内的全部数据 |
| COST-27 | Given 按项目成本 unitCost 计算，When sampleCount=0，Then unitCost=0（避免除零） |
| COST-28 | Given 供应商成本 orderCount 统计，When 检查返回值，Then 等于该供应商的入库单数量 |
| COST-29 | Given 物料成本 consumption 统计，When 检查返回值，Then 等于该物料的出库数量之和 |
| COST-30 | Given 成本报表精度验证，When 检查金额字段，Then 使用 DECIMAL(18,4)，支持 4 位小数 |
