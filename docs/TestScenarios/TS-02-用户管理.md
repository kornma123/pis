## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| USER-01 | Given admin 用户已登录并携带有效 Token，When 调用 GET /api/v1/users，Then 返回 200，用户列表包含分页信息和用户数据 |
| USER-02 | Given admin 用户，When 调用 GET /api/v1/users?page=1&pageSize=10，Then 返回分页结果，pagination 字段包含 page、pageSize、total |
| USER-03 | Given admin 用户，When 调用 GET /api/v1/users?keyword=admin，Then 返回用户名或真实姓名包含 "admin" 的用户列表 |
| USER-04 | Given admin 用户携带正确的用户名/密码/真实姓名，When 调用 POST /api/v1/users 创建用户，Then 返回 201，新用户 ID 在响应中，密码已 bcrypt 哈希存储 |
| USER-05 | Given admin 用户创建新用户时传入 role="warehouse_manager"，When 调用创建接口，Then 返回 201，用户角色正确存储 |
| USER-06 | Given admin 用户编辑用户时仅传入 realName 字段，When 调用 PUT /api/v1/users/:id，Then 返回 200，仅 realName 更新，其他字段保持原值 |
| USER-07 | Given admin 用户编辑用户时传入 password="newpass123"，When 调用 PUT 接口，Then 返回 200，数据库中密码已重新 bcrypt 哈希 |
| USER-08 | Given admin 用户将用户 status 从 "active" 改为 "inactive"，When 调用 PUT 接口，Then 返回 200，该用户后续登录返回 401 "User not found or disabled" |
| USER-09 | Given admin 用户删除一个无操作记录的用户，When 调用 DELETE /api/v1/users/:id，Then 返回 200，该用户不再出现在列表中 |
| USER-10 | Given admin 用户，When 调用 GET /api/v1/users 获取列表，Then 响应中绝不包含 password 字段 |
| USER-11 | Given 创建用户时 username 已存在，When 调用 POST /api/v1/users，Then 返回 409 "Username exists" |
| USER-12 | Given 创建用户时未传 username 字段，When 调用 POST /api/v1/users，Then 返回 400 "Username, password and realName required" |
| USER-13 | Given 创建用户时未传 password 字段，When 调用 POST /api/v1/users，Then 返回 400 "Username, password and realName required" |
| USER-14 | Given 创建用户时未传 realName 字段，When 调用 POST /api/v1/users，Then 返回 400 "Username, password and realName required" |
| USER-15 | Given 编辑不存在的用户 ID，When 调用 PUT /api/v1/users/:id，Then 返回 404 "Not found" |
| USER-16 | Given 删除不存在的用户 ID，When 调用 DELETE /api/v1/users/:id，Then 返回 404 "Not found" |
| USER-17 | Given warehouse_manager 角色用户已登录，When 调用 GET /api/v1/users，Then 返回 403 Forbidden |
| USER-18 | Given technician 角色用户已登录，When 调用 GET /api/v1/users，Then 返回 403 Forbidden |
| USER-19 | Given pathologist 角色用户已登录，When 调用 POST /api/v1/users 创建用户，Then 返回 403 Forbidden |
| USER-20 | Given procurement 角色用户已登录，When 调用 DELETE /api/v1/users/:id，Then 返回 403 Forbidden |
| USER-21 | Given finance 角色用户已登录，When 调用 GET /api/v1/users，Then 返回 403 Forbidden |
| USER-22 | Given 创建用户后查询用户列表，When 调用 GET /api/v1/users，Then 新用户出现在列表中，status="active"，不包含 password 字段 |
| USER-23 | Given 禁用用户后，When 使用该用户凭据调用 POST /api/v1/auth/login，Then 返回 401 "User not found or disabled" |
| USER-24 | Given 编辑用户密码后，When 使用旧密码登录，Then 返回 401 "Invalid password"；使用新密码登录，Then 返回 200 |
| USER-25 | Given 删除用户后，When 查询该用户历史操作日志，Then 日志记录仍存在，user_id 成为悬空引用 |
| USER-26 | Given admin 用户，When 调用 GET /api/v1/users?page=0，Then page=0 被后端修正为 1，正常返回分页数据 |
| USER-27 | Given admin 用户，When 调用 GET /api/v1/users?pageSize=1000，Then 返回最多 1000 条记录 |
| USER-28 | Given admin 用户，When 调用 GET /api/v1/users?keyword=""（空字符串），Then 返回全部用户列表 |
| USER-29 | Given admin 用户创建用户时 username 传入超长字符串（>50 字符），When 调用 POST，Then 后端截断或返回 400 |
| USER-30 | Given admin 用户创建用户时传入 role="nonexistent_role"，When 调用 POST，Then 创建成功（后端不校验角色存在性），但角色无实际权限定义 |
| USER-31 | Given 无 Token 请求，When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| USER-32 | Given 过期 Token，When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| USER-33 | Given 伪造 Token（错误签名），When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| USER-34 | Given SQL 注入 payload（username="' OR '1'='1"）作为创建参数，When 调用 POST /api/v1/users，Then 用户被创建（username 为注入字符串），无 SQL 注入漏洞 |
| USER-35 | Given XSS payload（realName="<script>alert(1)</script>"）作为创建参数，When 调用 POST /api/v1/users，Then 创建成功，realName 原样存储（前端负责转义） |
| USER-36 | Given 已删除用户（is_deleted=1），When admin 调用 GET /api/v1/users，Then 该用户不出现在列表中 |
| USER-37 | Given 创建用户后响应，When 检查响应结构，Then 包含 success、data.id、message 字段，success=true |
| USER-38 | Given 用户列表响应，When 检查分页结构，Then pagination 包含 page、pageSize、total 三个字段 |
| USER-39 | Given 编辑用户时传入 status="invalid_status"，When 调用 PUT，Then status 不被更新（或转为默认值） |
| USER-40 | Given 用户已被禁用，When admin 再次调用 PUT 将其 status 改为 "active"，Then 返回 200，用户可重新登录 |
