## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| LOC-01 | Given admin 用户已登录，When 调用 GET /api/v1/locations，Then 返回 200，库位列表包含 code、name、type、zone、capacity、used 字段 |
| LOC-02 | Given admin 用户，When 调用 POST /api/v1/locations 传入 name="A区-新货架"、zone="A区"，Then 返回 201，code 自动生成（如 LOC-00027），used=0，status="active" |
| LOC-03 | Given admin 用户创建库位时传入 type="refrigerator"，When 调用 POST，Then 返回 201，type 正确存储 |
| LOC-04 | Given admin 用户编辑库位时传入 capacity=500，When 调用 PUT /api/v1/locations/:id，Then 返回 200，capacity 更新成功 |
| LOC-05 | Given admin 用户删除一个无库存关联的库位，When 调用 DELETE /api/v1/locations/:id，Then 返回 200，逻辑删除 |
| LOC-06 | Given warehouse_manager 用户，When 调用 GET /api/v1/locations，Then 返回 200，正常读取 |
| LOC-07 | Given warehouse_manager 用户，When 调用 POST /api/v1/locations，Then 返回 403 Forbidden |
| LOC-08 | Given technician 用户，When 调用 GET /api/v1/locations，Then 返回 403 Forbidden |
| LOC-09 | Given 创建库位时未传 name 字段，When 调用 POST，Then 返回 400 "Name and zone required" |
| LOC-10 | Given 创建库位时未传 zone 字段，When 调用 POST，Then 返回 400 "Name and zone required" |
| LOC-11 | Given 按类型筛选 type="refrigerator"，When 调用 GET /api/v1/locations?type=refrigerator，Then 仅返回冷藏柜类型库位 |
| LOC-12 | Given 按区域筛选 zone="A区"，When 调用 GET /api/v1/locations?zone=A区，Then 仅返回 A 区库位 |
| LOC-13 | Given 库位树查询，When 调用 GET /api/v1/locations/tree，Then 返回树形结构，包含 children 嵌套 |
| LOC-14 | Given 删除库位后，When 查询关联的 inventory 记录，Then inventory.location_id 成为悬空引用 |
| LOC-15 | Given 创建子库位时传入 parentId="父库位ID"，When 调用 POST，Then 返回 201，子库位挂载在父库位下 |
| LOC-16 | Given 无 Token 请求，When 调用 GET /api/v1/locations，Then 返回 401 Unauthorized |
| LOC-17 | Given 过期 Token，When 调用 POST /api/v1/locations，Then 返回 401 Unauthorized |
| LOC-18 | Given 伪造 Token，When 调用 PUT /api/v1/locations/:id，Then 返回 401 Unauthorized |
| LOC-19 | Given SQL 注入 payload（name="' OR '1'='1"），When 调用 POST，Then 创建成功，无 SQL 注入漏洞 |
| LOC-20 | Given XSS payload（name="<script>alert(1)</script>"），When 调用 POST，Then 创建成功，name 原样存储 |
| LOC-21 | Given 分页 page=0，When 调用 GET /api/v1/locations?page=0，Then page=0 被修正为 1 |
| LOC-22 | Given 搜索空字符串 keyword=""，When 调用 GET /api/v1/locations?keyword=""，Then 返回全部库位 |
| LOC-23 | Given 编辑库位时传入 used=100，When 调用 PUT，Then used 字段不被更新（接口不支持直接修改 used） |
| LOC-24 | Given 创建库位时 capacity=0，When 调用 POST，Then 创建成功，capacity=0 |
| LOC-25 | Given 库位响应结构，When 调用 GET /api/v1/locations，Then 每条记录包含 id、code、name、type、parentId、zone、shelf、position、capacity、used、status |
| LOC-26 | Given 删除库位后再次创建，When 调用 POST，Then 新库位 code 可能复用已删除库位的编号 |
| LOC-27 | Given 创建库位时 type="invalid_type"，When 调用 POST，Then 后端可能不校验 type 合法性，创建成功 |
| LOC-28 | Given 库位树中父库位删除，When 查询树形结构，Then 子库位仍可能显示（取决于树构建逻辑） |
| LOC-29 | Given 编辑库位 status="inactive"，When 调用 PUT，Then 返回 200，status 变为 "inactive" |
| LOC-30 | Given 按状态筛选 status="active"，When 调用 GET /api/v1/locations?status=active，Then 仅返回 active 状态库位 |
