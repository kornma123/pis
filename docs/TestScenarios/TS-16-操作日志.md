## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| LOG-01 | Given admin 用户已登录，When 调用 GET /api/v1/logs/operation，Then 返回 200，操作日志列表包含 id、userId、username、operation、description、ip、userAgent、createdAt |
| LOG-02 | Given 按日期范围筛选 startDate="2026-01-01"&endDate="2026-12-31"，When 调用 GET /api/v1/logs/operation，Then 返回该日期范围内的日志 |
| LOG-03 | Given 按用户筛选 userId="用户ID"，When 调用 GET /api/v1/logs/operation?userId=xxx，Then 仅返回该用户的操作日志 |
| LOG-04 | Given finance 用户，When 调用 GET /api/v1/logs/operation，Then 返回 200，正常读取 |
| LOG-05 | Given warehouse_manager 用户，When 调用 GET /api/v1/logs/operation，Then 返回 403 Forbidden |
| LOG-06 | Given technician 用户，When 调用 GET /api/v1/logs/operation，Then 返回 403 Forbidden |
| LOG-07 | Given pathologist 用户，When 调用 GET /api/v1/logs/operation，Then 返回 403 Forbidden |
| LOG-08 | Given procurement 用户，When 调用 GET /api/v1/logs/operation，Then 返回 403 Forbidden |
| LOG-09 | Given 无 Token 请求，When 调用 GET /api/v1/logs/operation，Then 返回 401 Unauthorized |
| LOG-10 | Given 过期 Token，When 调用 GET /api/v1/logs/operation，Then 返回 401 Unauthorized |
| LOG-11 | Given 伪造 Token，When 调用 GET /api/v1/logs/operation，Then 返回 401 Unauthorized |
| LOG-12 | Given 分页 page=0，When 调用 GET /api/v1/logs/operation?page=0，Then page=0 被修正为 1 |
| LOG-13 | Given 分页 pageSize=1000，When 调用 GET /api/v1/logs/operation?pageSize=1000，Then 返回最多 1000 条记录 |
| LOG-14 | Given 搜索空字符串 keyword=""（如支持），When 调用 GET /api/v1/logs/operation?keyword=""，Then 返回全部日志 |
| LOG-15 | Given 日志响应结构，When 检查每条记录，Then 包含 id、userId、username、operation、description、ip、userAgent、createdAt 字段 |
| LOG-16 | Given 日志 createdAt 字段，When 检查格式，Then 为 ISO 8601 格式或 DATETIME 格式 |
| LOG-17 | Given 日志中的 userId 对应已删除用户，When 查询日志，Then 日志记录仍存在，username 为删除时的用户名 |
| LOG-18 | Given 日志数量验证，When 查询全部日志，Then 系统初始化后有 13 条操作日志（seed 数据） |
| LOG-19 | Given 创建用户操作，When 查询日志，Then 存在一条 operation="create_user" 或类似描述的日志记录 |
| LOG-20 | Given 入库操作，When 查询日志，Then 存在一条 operation="inbound" 或类似描述的日志记录 |
| LOG-21 | Given 出库操作，When 查询日志，Then 存在一条 operation="outbound" 或类似描述的日志记录 |
| LOG-22 | Given 日志中的 IP 字段，When 检查格式，Then 为有效的 IPv4 地址格式（如 127.0.0.1） |
| LOG-23 | Given 日志中的 userAgent 字段，When 检查内容，Then 包含浏览器和操作系统信息 |
| LOG-24 | Given 无日志数据时，When 调用 GET /api/v1/logs/operation，Then 返回空列表或 200 无数据 |
| LOG-25 | Given 按不存在的 userId 筛选，When 调用 GET /api/v1/logs/operation?userId=不存在的ID，Then 返回空列表 |
| LOG-26 | Given 日期范围无数据时，When 调用 GET /api/v1/logs/operation?startDate=2099-01-01，Then 返回空列表 |
| LOG-27 | Given 日志不支持删除，When 尝试调用 DELETE /api/v1/logs/operation/:id，Then 返回 404（接口不存在） |
| LOG-28 | Given SQL 注入 payload（userId="' OR '1'='1"），When 调用 GET /api/v1/logs/operation?userId=xxx，Then 返回空结果或正常过滤，无 SQL 注入漏洞 |
| LOG-29 | Given 响应结构检查，When 调用 GET /api/v1/logs/operation，Then 返回 success=true，data.list 为日志数组，data.pagination 包含分页信息 |
| LOG-30 | Given 日志排序，When 查询日志列表，Then 默认按 createdAt DESC 排列（最新操作在前） |
| LOG-31 | Given 超大数据量分页，When 调用 GET /api/v1/logs/operation?page=999999，Then 返回空列表，不报错 |
| LOG-32 | Given 无效日期格式 startDate="invalid"，When 调用 GET，Then 返回 400 或忽略该参数返回全部数据 |
| LOG-33 | Given admin 用户查询日志，When 检查 username 字段，Then 与 users 表中的 username 一致 |
| LOG-34 | Given 日志 operation 字段枚举，When 检查所有日志，Then operation 值为预定义的操作类型（如 login、create、update、delete、inbound、outbound） |
| LOG-35 | Given 系统重启后，When 查询日志，Then 历史日志数据仍然保留（SQLite 持久化） |
