## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| OUT-01 | Given 有充足库存的物料，When 调用 POST /api/v1/outbound 创建出库单，Then 返回 200，outboundNo 格式为 OB-yyyymmdd-xxx，status="completed" |
| OUT-02 | Given 出库单包含多个物料，When 调用 POST /api/v1/outbound，Then 返回 200，所有物料按 FIFO 分配批次，items 包含各物料的 batchNo 和 unitCost |
| OUT-03 | Given 出库后，When 查询库存，Then inventory.stock 减少出库数量 |
| OUT-04 | Given 出库后，When 查询批次，Then 最早过期批次的 remaining 减少出库数量；若 remaining<=0，则 status=0 |
| OUT-05 | Given 出库后，When 查询 stock_logs，Then 存在 type="outbound" 的流水记录 |
| OUT-06 | Given 出库成本计算，When 检查返回的 items，Then unitCost = 分配批次的入库价，totalCost = quantity × unitCost |
| OUT-07 | Given 关联 projectId 出库，When 调用 POST /api/v1/outbound，Then 出库记录关联 projectId，用于成本归集 |
| OUT-08 | Given 库存不足的物料，When 调用 POST /api/v1/outbound，Then 返回 422 "Insufficient stock"，整单拒绝 |
| OUT-09 | Given 多物料出库时其中一个物料库存不足，When 调用 POST，Then 返回 422，整单拒绝，所有物料均未出库 |
| OUT-10 | Given FIFO 批次分配，When 物料有多个活跃批次，Then 分配最早过期（expiry_date ASC）的批次 |
| OUT-11 | Given 物料无活跃批次，When 调用 POST 出库，Then batchNo 可能为 null，unitCost=0 |
| OUT-12 | Given 出库单列表查询，When 调用 GET /api/v1/outbound?projectId=xxx，Then 仅返回该项目的出库记录 |
| OUT-13 | Given warehouse_manager 用户，When 调用 POST /api/v1/outbound，Then 返回 200，创建成功 |
| OUT-14 | Given technician 用户，When 调用 POST /api/v1/outbound，Then 返回 200，创建成功 |
| OUT-15 | Given pathologist 用户，When 调用 POST /api/v1/outbound，Then 返回 200，创建成功 |
| OUT-16 | Given procurement 用户，When 调用 POST /api/v1/outbound，Then 返回 403 Forbidden |
| OUT-17 | Given finance 用户，When 调用 GET /api/v1/outbound，Then 返回 403 Forbidden |
| OUT-18 | Given 创建出库单时未传 type 字段，When 调用 POST，Then 返回 400 "Missing required fields" |
| OUT-19 | Given 创建出库单时 items=[]（空数组），When 调用 POST，Then 返回 400 "Missing required fields" |
| OUT-20 | Given 出库单中 item.usage="return"，When 调用 POST，Then 返回 200，usage 字段正确存储 |
| OUT-21 | Given 出库单中 item.receiver="张三"，When 调用 POST，Then 返回 200，receiver 字段正确存储 |
| OUT-22 | Given 无 Token 请求，When 调用 POST /api/v1/outbound，Then 返回 401 Unauthorized |
| OUT-23 | Given 过期 Token，When 调用 GET /api/v1/outbound，Then 返回 401 Unauthorized |
| OUT-24 | Given SQL 注入 payload（remark="' OR '1'='1"），When 调用 POST，Then 创建成功，无 SQL 注入漏洞 |
| OUT-25 | Given XSS payload（remark="<script>alert(1)</script>"），When 调用 POST，Then 创建成功，remark 原样存储 |
| OUT-26 | Given 分页 page=0，When 调用 GET /api/v1/outbound?page=0，Then page=0 被修正为 1 |
| OUT-27 | Given 库存恰好等于出库数量，When 调用 POST，Then 返回 200，出库后该物料 stock=0 |
| OUT-28 | Given 出库后批次 remaining 恰好为 0，When 检查批次，Then batch.status=0，该批次不再参与后续 FIFO 分配 |
| OUT-29 | Given 项目成本归集验证，When 创建 project 类型出库单后查询项目详情，Then costStats.totalCost 包含该出库单成本 |
| OUT-30 | Given 出库单响应结构，When 检查返回数据，Then 包含 id、outboundNo、type、projectId、items、totalCost、operator、status |
