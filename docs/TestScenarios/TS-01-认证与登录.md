## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| AUTH-01 | Given 提供正确的用户名 admin 和密码 admin123，When 调用 POST /api/v1/auth/login，Then 返回 200，响应包含 token（有效期8h）、refreshToken（有效期7d）、expiresIn=28800、用户信息 |
| AUTH-02 | Given 提供正确的用户名 warehouse_manager 和密码 whm123，When 调用登录接口，Then 返回 200，用户 role="warehouse_manager" |
| AUTH-03 | Given 提供正确的用户名 technician 和密码 tech123，When 调用登录接口，Then 返回 200，用户 role="technician" |
| AUTH-04 | Given 提供正确的用户名 pathologist 和密码 path123，When 调用登录接口，Then 返回 200，用户 role="pathologist" |
| AUTH-05 | Given 提供正确的用户名 procurement 和密码 pro123，When 调用登录接口，Then 返回 200，用户 role="procurement" |
| AUTH-06 | Given 提供正确的用户名 finance 和密码 fin123，When 调用登录接口，Then 返回 200，用户 role="finance" |
| AUTH-07 | Given 密码错误，When 调用 POST /api/v1/auth/login，Then 返回 401，错误信息 "Invalid password" |
| AUTH-08 | Given 不存在的用户名，When 调用登录接口，Then 返回 401，错误信息 "User not found or disabled" |
| AUTH-09 | Given 已禁用用户的用户名和密码，When 调用登录接口，Then 返回 401，错误信息 "User not found or disabled" |
| AUTH-10 | Given 用户名空字符串，When 调用登录接口，Then 返回 400 "Username and password required" |
| AUTH-11 | Given 密码空字符串，When 调用登录接口，Then 返回 400 "Username and password required" |
| AUTH-12 | Given 用户名和密码同时为空，When 调用登录接口，Then 返回 400 "Username and password required" |
| AUTH-13 | Given 有效的 refreshToken，When 调用 POST /api/v1/auth/refresh，Then 返回 200，包含新的 access token 和 expiresIn=28800 |
| AUTH-14 | Given 使用 access token 调用刷新接口，When 调用 POST /api/v1/auth/refresh，Then 返回 401 "Invalid refresh token" |
| AUTH-15 | Given refreshToken 过期，When 调用刷新接口，Then 返回 401，需重新登录 |
| AUTH-16 | Given refreshToken 中用户已被禁用，When 调用刷新接口，Then 返回 401 "User not found or disabled" |
| AUTH-17 | Given 空 refreshToken，When 调用刷新接口，Then 返回 400 "Refresh token required" |
| AUTH-18 | Given 已登录用户调用登出接口，When 调用 POST /api/v1/auth/logout，Then 返回 200，服务端无状态变化 |
| AUTH-19 | Given 登出后使用原 token 访问受保护接口，When 调用 GET /api/v1/users，Then 返回 200（token 在8h内仍然有效，无黑名单机制） |
| AUTH-20 | Given 并发登录场景，When 用户 admin 从两个客户端同时登录，Then 各自获得独立的 token，均可正常访问 |
| AUTH-21 | Given 无 Token 请求，When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| AUTH-22 | Given 过期 access token，When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| AUTH-23 | Given 伪造 access token（错误签名），When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| AUTH-24 | Given 篡改 payload 的 token，When 调用 GET /api/v1/users，Then 返回 401 Unauthorized |
| AUTH-25 | Given SQL 注入 payload（username="' OR '1'='1"），When 调用登录接口，Then 返回 401 "User not found or disabled"（参数化查询防御） |
| AUTH-26 | Given XSS payload（username="<script>alert(1)</script>"），When 调用登录接口，Then 返回 401 "User not found or disabled" |
| AUTH-27 | Given 超长用户名（>100字符），When 调用登录接口，Then 返回 401 "User not found or disabled" |
| AUTH-28 | Given 登录成功响应，When 检查响应结构，Then 包含 success=true、data（token、refreshToken、expiresIn、user）、message |
| AUTH-29 | Given 登录成功响应中的 user 对象，When 检查字段，Then 包含 id、username、realName、role、department、permissions（固定数组） |
| AUTH-30 | Given 登录成功响应中的 permissions 字段，When 检查值，Then 固定为 ['inventory:view', 'inventory:edit', 'report:view', 'system:view']（不反映真实角色权限） |
| AUTH-31 | Given JWT payload 结构，When 解码 token，Then payload 包含 userId、username、role、iat、exp |
| AUTH-32 | Given JWT payload 结构，When 解码 refreshToken，Then payload 包含 userId、type="refresh"、iat、exp |
| AUTH-33 | Given Token 有效期验证，When 检查 access token 的 exp-iat，Then 差值为 28800 秒（8小时） |
| AUTH-34 | Given RefreshToken 有效期验证，When 检查 refreshToken 的 exp-iat，Then 差值为 604800 秒（7天） |
| AUTH-35 | Given 使用不同 Secret 签名的 token，When 调用受保护接口，Then 返回 401 Unauthorized |
