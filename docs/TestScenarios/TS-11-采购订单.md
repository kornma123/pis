## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| PO-01 | Given admin 用户已登录，When 调用 POST /api/v1/purchase-orders 创建采购单，Then 返回 201，orderNo 格式为 POyyyymmdd-xxxx，status="pending"，receivedQty=0 |
| PO-02 | Given 创建采购单时传入 materialId="物料ID"、orderedQty=100，When 调用 POST，Then 返回 201，totalAmount = orderedQty × unitPrice |
| PO-03 | Given 采购单状态为 pending，When 调用 PUT /api/v1/purchase-orders/:id/receive 传入 quantity=30，Then 返回 200，receivedQty=30，status="partial" |
| PO-04 | Given 采购单状态为 partial，When 再次收货 quantity=70（orderedQty=100），Then 返回 200，receivedQty=100，status="completed" |
| PO-05 | Given 采购单状态为 pending，When 调用 PUT /api/v1/purchase-orders/:id/cancel，Then 返回 200，status="cancelled" |
| PO-06 | Given 采购单状态为 completed，When 调用 PUT /api/v1/purchase-orders/:id/cancel，Then 返回 400 "已完成的订单不能取消" |
| PO-07 | Given 收货时 quantity > 剩余数量，When 调用 PUT /api/v1/purchase-orders/:id/receive，Then 返回 400 "入库数量超过订单数量" |
| PO-08 | Given 采购单列表查询，When 调用 GET /api/v1/purchase-orders?status=pending，Then 仅返回 pending 状态的采购单 |
| PO-09 | Given 按供应商筛选 supplierId="供应商ID"，When 调用 GET /api/v1/purchase-orders?supplierId=xxx，Then 仅返回该供应商的采购单 |
| PO-10 | Given 搜索采购单 keyword="PO20260512"，When 调用 GET /api/v1/purchase-orders?keyword=PO20260512，Then 返回匹配的采购单 |
| PO-11 | Given procurement 用户，When 调用 POST /api/v1/purchase-orders，Then 返回 201，创建成功 |
| PO-12 | Given procurement 用户，When 调用 PUT /api/v1/purchase-orders/:id/receive，Then 返回 200，收货成功 |
| PO-13 | Given warehouse_manager 用户，When 调用 GET /api/v1/purchase-orders，Then 返回 200，正常读取 |
| PO-14 | Given warehouse_manager 用户，When 调用 POST /api/v1/purchase-orders，Then 返回 403 Forbidden |
| PO-15 | Given technician 用户，When 调用 GET /api/v1/purchase-orders，Then 返回 403 Forbidden |
| PO-16 | Given 创建采购单时未传 materialId 字段，When 调用 POST，Then 返回 400 "物料和采购数量必填" |
| PO-17 | Given 创建采购单时未传 orderedQty 字段，When 调用 POST，Then 返回 400 "物料和采购数量必填" |
| PO-18 | Given 采购单详情查询，When 调用 GET /api/v1/purchase-orders/:id，Then 返回 200，包含 remainingQty（=orderedQty - receivedQty）计算字段 |
| PO-19 | Given 关联入库单创建后，When 查询采购单，Then receivedQty 自动增加，status 可能变为 partial 或 completed |
| PO-20 | Given 删除入库单后，When 查询采购单，Then receivedQty 回退，status 回退为 pending 或 partial |
| PO-21 | Given 取消的采购单，When 查询列表，Then status="cancelled" 的采购单仍可见 |
| PO-22 | Given 无 Token 请求，When 调用 GET /api/v1/purchase-orders，Then 返回 401 Unauthorized |
| PO-23 | Given 过期 Token，When 调用 POST /api/v1/purchase-orders，Then 返回 401 Unauthorized |
| PO-24 | Given SQL 注入 payload（remark="' OR '1'='1"），When 调用 POST，Then 创建成功，无 SQL 注入漏洞 |
| PO-25 | Given XSS payload（remark="<script>alert(1)</script>"），When 调用 POST，Then 创建成功，remark 原样存储 |
| PO-26 | Given 分页 page=0，When 调用 GET /api/v1/purchase-orders?page=0，Then page=0 被修正为 1 |
| PO-27 | Given 搜索空字符串 keyword=""，When 调用 GET /api/v1/purchase-orders?keyword=""，Then 返回全部采购单 |
| PO-28 | Given 收货 quantity=0，When 调用 PUT /api/v1/purchase-orders/:id/receive，Then 后端可能未拦截，receivedQty 不变 |
| PO-29 | Given 单号连续性验证，When 同一天创建多个采购单，Then orderNo 序号连续递增（如 PO20260512-0001、PO20260512-0002） |
| PO-30 | Given 响应结构检查，When 调用 GET /api/v1/purchase-orders，Then 每条记录包含 id、orderNo、materialId、materialName、orderedQty、receivedQty、remainingQty、status |
