## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| SUP-01 | Given admin 用户已登录，When 调用 GET /api/v1/suppliers，Then 返回 200，供应商列表包含 code、name、rating、cooperationCount 字段 |
| SUP-02 | Given admin 用户，When 调用 POST /api/v1/suppliers 传入 name="新供应商"，Then 返回 201，code 自动生成（如 SUP-00011），cooperationCount=0，totalAmount=0，rating=5 |
| SUP-03 | Given admin 用户创建供应商时传入 contact="张三"、phone="13800138000"、email="test@example.com"，When 调用 POST，Then 返回 201，所有字段正确存储 |
| SUP-04 | Given admin 用户编辑供应商时传入 status="inactive"，When 调用 PUT /api/v1/suppliers/:id，Then 返回 200，status 变为 "inactive" |
| SUP-05 | Given admin 用户删除一个无物料关联的供应商，When 调用 DELETE /api/v1/suppliers/:id，Then 返回 200，逻辑删除 |
| SUP-06 | Given warehouse_manager 用户，When 调用 GET /api/v1/suppliers，Then 返回 200，正常读取 |
| SUP-07 | Given procurement 用户，When 调用 GET /api/v1/suppliers，Then 返回 200，正常读取 |
| SUP-08 | Given warehouse_manager 用户，When 调用 POST /api/v1/suppliers，Then 返回 403 Forbidden |
| SUP-09 | Given technician 用户，When 调用 GET /api/v1/suppliers，Then 返回 403 Forbidden |
| SUP-10 | Given 创建供应商时未传 name 字段，When 调用 POST /api/v1/suppliers，Then 返回 400 "Name required" |
| SUP-11 | Given 搜索供应商 keyword="SUP-00001"，When 调用 GET /api/v1/suppliers?keyword=SUP-00001，Then 返回匹配 code 的供应商 |
| SUP-12 | Given 按状态筛选 status="active"，When 调用 GET /api/v1/suppliers?status=active，Then 仅返回 active 状态的供应商 |
| SUP-13 | Given 连续创建 3 个供应商，When 查询列表，Then code 连续递增（如 SUP-00011、SUP-00012、SUP-00013） |
| SUP-14 | Given 删除供应商后，When 查询关联的物料详情，Then 物料中 supplier_id 成为悬空引用，supplierName 可能为空 |
| SUP-15 | Given 删除供应商后再次创建新供应商，When 调用 POST，Then 新供应商 code 可能复用已删除供应商的编号（因为生成时仅查 is_deleted=0） |
| SUP-16 | Given 编辑供应商时仅传入 name 字段，When 调用 PUT，Then 仅 name 更新，其他字段保持原值 |
| SUP-17 | Given 无 Token 请求，When 调用 GET /api/v1/suppliers，Then 返回 401 Unauthorized |
| SUP-18 | Given 过期 Token，When 调用 POST /api/v1/suppliers，Then 返回 401 Unauthorized |
| SUP-19 | Given 伪造 Token，When 调用 PUT /api/v1/suppliers/:id，Then 返回 401 Unauthorized |
| SUP-20 | Given SQL 注入 payload（name="' OR '1'='1"），When 调用 POST /api/v1/suppliers，Then 创建成功，无 SQL 注入漏洞 |
| SUP-21 | Given XSS payload（name="<script>alert(1)</script>"），When 调用 POST /api/v1/suppliers，Then 创建成功，name 原样存储 |
| SUP-22 | Given 分页请求 page=0，When 调用 GET /api/v1/suppliers?page=0，Then page=0 被修正为 1 |
| SUP-23 | Given 分页请求 pageSize=1000，When 调用 GET /api/v1/suppliers?pageSize=1000，Then 返回最多 1000 条记录 |
| SUP-24 | Given 搜索空字符串 keyword=""，When 调用 GET /api/v1/suppliers?keyword=""，Then 返回全部供应商列表 |
| SUP-25 | Given 响应结构检查，When 调用 GET /api/v1/suppliers，Then 每条记录包含 id、code、name、contact、phone、email、address、status、cooperationCount、totalAmount、rating |
| SUP-26 | Given 创建供应商后，When 检查响应，Then 不包含 cooperationCount、totalAmount、rating 的默认值（返回 0、0、5） |
| SUP-27 | Given 编辑供应商 code 字段，When 调用 PUT，Then code 更新成功，但历史入库记录中的 supplier_id 不会同步更新 |
| SUP-28 | Given 供应商状态为 inactive，When 仍可在物料中选择该供应商，Then 物料创建成功，supplier_id 指向 inactive 供应商 |
| SUP-29 | Given 创建供应商时传入 email="invalid-email"，When 调用 POST，Then 后端不校验邮箱格式，创建成功 |
| SUP-30 | Given 创建供应商时传入 phone="invalid-phone"，When 调用 POST，Then 后端不校验电话格式，创建成功 |
