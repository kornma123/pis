## 测试维度说明

每个模块的测试场景均覆盖以下 7 个维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **正常流程** | 合法请求获得预期成功响应 | 正确用户名密码登录返回 200 |
| **异常/校验** | 非法参数、缺失字段、业务冲突 | 重复编码返回 409，库存不足返回 422 |
| **RBAC权限** | 6 角色权限矩阵验证 | admin 可创建，technician 禁止创建 |
| **数据一致性** | 操作后数据正确联动 | 入库后库存增加，删除入库后回滚 |
| **边界条件** | 分页极限、空字符串、特殊值 | page=0、pageSize=1000、price=0 |
| **安全测试** | Token 安全、注入攻击防御 | 无 Token 返回 401，SQL 注入被防御 |
| **交互认知** | 响应结构完整性、字段正确性 | 分页包含 total，错误信息人性化 |

## 使用方式

1. **按模块执行**: 每个模块独立成文，可分配给不同测试人员并行执行
2. **按维度筛选**: 可根据当前测试阶段筛选特定维度的场景（如先执行正常流程，再执行安全测试）
3. **自动化映射**: 场景 ID 可直接映射到自动化测试用例名称
