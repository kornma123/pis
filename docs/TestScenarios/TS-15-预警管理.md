## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| ALERT-01 | Given 任意角色用户已登录，When 调用 GET /api/v1/alerts/rules，Then 返回 200，包含低库存和临期两类预警规则 |
| ALERT-02 | Given admin 用户，When 调用 PUT /api/v1/alerts/rules/:id 修改 threshold=20，Then 返回 200，规则阈值更新成功 |
| ALERT-03 | Given admin 用户，When 调用 PUT /api/v1/alerts/rules/:id 修改 thresholdDays=60，Then 返回 200，临期阈值天数更新成功 |
| ALERT-04 | Given admin 用户，When 调用 PUT /api/v1/alerts/rules/:id 修改 enabled=false，Then 返回 200，规则禁用成功 |
| ALERT-05 | Given 任意角色用户，When 调用 GET /api/v1/alerts，Then 返回 200，预警记录列表包含 type、level、materialName、status 字段 |
| ALERT-06 | Given 按状态筛选 status="pending"，When 调用 GET /api/v1/alerts?status=pending，Then 仅返回待处理的预警 |
| ALERT-07 | Given 按类型筛选 type="low-stock"，When 调用 GET /api/v1/alerts?type=low-stock，Then 仅返回低库存预警 |
| ALERT-08 | Given admin 用户手动生成预警，When 调用 POST /api/v1/alerts/generate，Then 返回 200，系统扫描库存生成新的预警记录 |
| ALERT-09 | Given 某物料 stock <= safety_stock 且 safety_stock > 0，When 调用 POST /api/v1/alerts/generate，Then 生成低库存预警，message 包含当前库存和安全库存值 |
| ALERT-10 | Given 某批次 expiry_date <= today + thresholdDays，When 调用 POST /api/v1/alerts/generate，Then 生成临期预警，message 包含批号和过期日期 |
| ALERT-11 | Given 同一物料已存在 pending 的低库存预警，When 再次调用 POST /api/v1/alerts/generate，Then 不重复生成（去重规则） |
| ALERT-12 | Given 处理预警，When 调用 POST /api/v1/alerts/:id/handle 传入 action="handled"、remark="已采购"，Then 返回 200，status="handled"，handled_at 有值 |
| ALERT-13 | Given technician 用户，When 调用 GET /api/v1/alerts，Then 返回 200，正常读取 |
| ALERT-14 | Given pathologist 用户，When 调用 GET /api/v1/alerts，Then 返回 200，正常读取 |
| ALERT-15 | Given procurement 用户，When 调用 GET /api/v1/alerts，Then 返回 200，正常读取 |
| ALERT-16 | Given warehouse_manager 用户编辑预警规则，When 调用 PUT /api/v1/alerts/rules/:id，Then 返回 403 Forbidden |
| ALERT-17 | Given technician 用户编辑预警规则，When 调用 PUT /api/v1/alerts/rules/:id，Then 返回 403 Forbidden |
| ALERT-18 | Given 无 Token 请求，When 调用 GET /api/v1/alerts，Then 返回 401 Unauthorized |
| ALERT-19 | Given 过期 Token，When 调用 GET /api/v1/alerts/rules，Then 返回 401 Unauthorized |
| ALERT-20 | Given 伪造 Token，When 调用 POST /api/v1/alerts/generate，Then 返回 401 Unauthorized |
| ALERT-21 | Given 分页 page=0，When 调用 GET /api/v1/alerts?page=0，Then page=0 被修正为 1 |
| ALERT-22 | Given 搜索空字符串 keyword=""（如支持），When 调用 GET /api/v1/alerts?keyword=""，Then 返回全部预警记录 |
| ALERT-23 | Given 响应结构检查，When 调用 GET /api/v1/alerts，Then 每条记录包含 id、type、level、materialId、materialName、currentStock、threshold、message、status、createdAt |
| ALERT-24 | Given 预警规则响应，When 调用 GET /api/v1/alerts/rules，Then 每条规则包含 id、type、name、threshold、thresholdDays、enabled |
| ALERT-25 | Given 禁用规则后生成预警，When 调用 POST /api/v1/alerts/generate，Then 禁用的规则类型不触发预警生成 |
| ALERT-26 | Given 处理不存在的预警，When 调用 POST /api/v1/alerts/:id/handle，Then 返回 404 Not Found |
| ALERT-27 | Given 已 handled 的预警再次处理，When 调用 POST /api/v1/alerts/:id/handle，Then 返回 200，status 更新为新的 action |
| ALERT-28 | Given 无 pending 预警时生成，When 调用 POST /api/v1/alerts/generate，Then 可能无新预警生成，返回空或成功 |
| ALERT-29 | Given 预警消息格式验证，When 检查低库存预警 message，Then 格式为 "Low stock: current {stock}, safety {safety_stock}" |
| ALERT-30 | Given 预警消息格式验证，When 检查临期预警 message，Then 格式为 "Batch {batch_no} expires at {expiry_date}" |
| ALERT-31 | Given SQL 注入 payload（remark="' OR '1'='1"），When 调用 POST /api/v1/alerts/:id/handle，Then 处理成功，无 SQL 注入漏洞 |
| ALERT-32 | Given XSS payload（remark="<script>alert(1)</script>"），When 调用 POST /api/v1/alerts/:id/handle，Then 处理成功，remark 原样存储 |
| ALERT-33 | Given 分页 pageSize=1000，When 调用 GET /api/v1/alerts?pageSize=1000，Then 返回最多 1000 条记录 |
| ALERT-34 | Given 预警 level 字段，When 检查返回数据，Then level 为 "warning" 或 "danger" |
| ALERT-35 | Given 生成预警后查询库存，When 检查预警中的 currentStock，Then 与 inventory.stock 一致 |
