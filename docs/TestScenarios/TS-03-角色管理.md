## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| ROLE-01 | Given admin 用户已登录，When 调用 GET /api/v1/roles，Then 返回 200，角色列表包含 code、name、permissions 字段 |
| ROLE-02 | Given admin 用户，When 调用 POST /api/v1/roles 传入 code="test_role"、name="测试角色"、permissions=["inventory","alerts"]，Then 返回 201，permissions 被序列化为 JSON 字符串存储 |
| ROLE-03 | Given admin 用户创建角色时传入 status="active"，When 调用 POST，Then 返回 201，status 存储为 1 |
| ROLE-04 | Given admin 用户编辑角色时传入 permissions=["reports","system"]，When 调用 PUT /api/v1/roles/:id，Then 返回 200，permissions 被 JSON.stringify 后存储 |
| ROLE-05 | Given admin 用户删除一个无用户绑定的角色，When 调用 DELETE /api/v1/roles/:id，Then 返回 200，该角色不再出现在列表中 |
| ROLE-06 | Given admin 用户，When 调用 GET /api/v1/roles 获取角色列表，Then 响应中 permissions 字段为 JSON 字符串（如 '["inventory","alerts"]'），前端需 JSON.parse |
| ROLE-07 | Given 创建角色时 code 已存在，When 调用 POST /api/v1/roles，Then 返回 409 "Role code already exists" |
| ROLE-08 | Given 创建角色时未传 code 字段，When 调用 POST /api/v1/roles，Then 返回 400 "Code and name required" |
| ROLE-09 | Given 创建角色时未传 name 字段，When 调用 POST /api/v1/roles，Then 返回 400 "Code and name required" |
| ROLE-10 | Given 编辑不存在的角色 ID，When 调用 PUT /api/v1/roles/:id，Then 返回 404 "Role not found" |
| ROLE-11 | Given warehouse_manager 角色用户，When 调用 GET /api/v1/roles，Then 返回 403 Forbidden |
| ROLE-12 | Given technician 角色用户，When 调用 POST /api/v1/roles，Then 返回 403 Forbidden |
| ROLE-13 | Given pathologist 角色用户，When 调用 PUT /api/v1/roles/:id，Then 返回 403 Forbidden |
| ROLE-14 | Given procurement 角色用户，When 调用 DELETE /api/v1/roles/:id，Then 返回 403 Forbidden |
| ROLE-15 | Given finance 角色用户，When 调用 GET /api/v1/roles，Then 返回 403 Forbidden |
| ROLE-16 | Given 删除角色后，已绑定该角色的用户登录，When 调用 POST /api/v1/auth/login，Then 登录成功，RBAC 仍基于 role 字符串匹配，不校验 roles 表存在性 |
| ROLE-17 | Given 编辑角色 code 后，已绑定该角色的用户，When 查询用户列表，Then 用户 role 字段未同步更新，成为悬空引用 |
| ROLE-18 | Given 创建角色时传入 code="ADMIN"（大写），When POST，Then 与已有的 "admin"（小写）不冲突，code 区分大小写 |
| ROLE-19 | Given 创建角色时 permissions=[]（空数组），When POST，Then 创建成功，该角色无任何权限 |
| ROLE-20 | Given 无 Token 请求，When 调用 GET /api/v1/roles，Then 返回 401 Unauthorized |
| ROLE-21 | Given 过期 Token，When 调用 POST /api/v1/roles，Then 返回 401 Unauthorized |
| ROLE-22 | Given 伪造 Token，When 调用 PUT /api/v1/roles/:id，Then 返回 401 Unauthorized |
| ROLE-23 | Given SQL 注入 payload（code="' OR '1'='1"），When 调用 POST /api/v1/roles，Then 返回 201（code 被原样存储），无 SQL 注入漏洞 |
| ROLE-24 | Given XSS payload（name="<script>alert(1)</script>"），When 调用 POST /api/v1/roles，Then 创建成功，name 原样存储 |
| ROLE-25 | Given 创建角色后响应，When 检查响应结构，Then 包含 success=true、data.id、message 字段 |
| ROLE-26 | Given 角色列表响应，When 检查 permissions 字段，Then 为字符串类型（非数组），如 '"[\\"inventory\\"]"' |
| ROLE-27 | Given 编辑角色时仅传入 name 字段，When 调用 PUT，Then name 更新，其他字段保持原值（PUT 全量语义下可能覆盖为 null） |
| ROLE-28 | Given 预置角色 admin 已存在，When 尝试创建 code="admin"，Then 返回 409 "Role code already exists" |
| ROLE-29 | Given 预置角色 warehouse_manager 已存在，When 尝试创建 code="warehouse_manager"，Then 返回 409 |
| ROLE-30 | Given 删除角色后查询角色列表，When 调用 GET /api/v1/roles，Then 已删除角色不显示（is_deleted=1 被过滤） |
