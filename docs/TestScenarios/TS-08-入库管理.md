## 测试场景

| 场景ID | 场景描述 |
|--------|----------|
| INB-01 | Given admin 用户已登录，When 调用 POST /api/v1/inbound 创建采购入库单，Then 返回 200，inboundNo 格式为 IB-yyyymmdd-xxx，status="completed" |
| INB-02 | Given 创建入库单时关联 purchaseOrderId，When 调用 POST，Then 采购订单的 received_qty 增加，status 更新为 partial 或 completed |
| INB-03 | Given 创建入库单时传入 batchNo="BATCH001"，When 调用 POST，Then 若批次不存在则创建新批次，若存在则累加 quantity 和 remaining |
| INB-04 | Given 创建入库单后，When 查询库存，Then inventory.stock 增加入库数量 |
| INB-05 | Given 创建入库单后，When 查询 stock_logs，Then 存在一条 type="inbound" 的流水记录 |
| INB-06 | Given 编辑入库单，When 调用 PUT /api/v1/inbound/:id，Then 返回 200，更新成功 |
| INB-07 | Given 删除一个无出库记录、非使用中的入库单，When 调用 DELETE /api/v1/inbound/:id，Then 返回 200，inventory.stock 回退，PO.received_qty 回退，batches 扣减 |
| INB-08 | Given 删除入库单后 PO 回退，When 检查采购订单，Then 若回退后 received_qty=0，PO.status="pending"；若>0 且<ordered_qty，status="partial" |
| INB-09 | Given 删除有出库记录的入库单，When 调用 DELETE /api/v1/inbound/:id，Then 返回 400 "已有出库记录，不可删除" |
| INB-10 | Given 删除状态为使用中的入库单，When 调用 DELETE，Then 返回 400 "正在使用中" |
| INB-11 | Given 取消入库单，When 调用 POST /api/v1/inbound/:id/cancel，Then 返回 200，status="cancelled"，库存不变 |
| INB-12 | Given 取消后再次取消同一入库单，When 调用 POST /api/v1/inbound/:id/cancel，Then 返回 200，不会报错 |
| INB-13 | Given 创建入库单时未传 type 字段，When 调用 POST，Then 返回 400 "Missing required fields" |
| INB-14 | Given 创建入库单时未传 materialId 字段，When 调用 POST，Then 返回 400 "Missing required fields" |
| INB-15 | Given 创建入库单时未传 quantity 字段，When 调用 POST，Then 返回 400 "Missing required fields" |
| INB-16 | Given 创建入库单时未传 locationId 字段，When 调用 POST，Then 返回 400 "Missing required fields" |
| INB-17 | Given 入库单列表查询，When 调用 GET /api/v1/inbound?status=completed，Then 仅返回 completed 状态的入库单 |
| INB-18 | Given 按日期范围筛选 startDate="2026-01-01"&endDate="2026-12-31"，When 调用 GET /api/v1/inbound，Then 返回该日期范围内的入库单（endDate 自动附加 23:59:59） |
| INB-19 | Given warehouse_manager 用户，When 调用 POST /api/v1/inbound，Then 返回 200，创建成功 |
| INB-20 | Given procurement 用户，When 调用 POST /api/v1/inbound，Then 返回 403 Forbidden |
| INB-21 | Given technician 用户，When 调用 POST /api/v1/inbound，Then 返回 403 Forbidden |
| INB-22 | Given 创建入库单时 quantity=0，When 调用 POST，Then 后端未显式拦截，可能创建成功（边界行为） |
| INB-23 | Given 创建入库单时 quantity=-5（负数），When 调用 POST，Then 后端未显式拦截，可能创建成功 |
| INB-24 | Given 入库前检查 check-deletable，When 调用 GET /api/v1/inbound/:id/check-deletable，Then 返回 canDelete、reasons、record |
| INB-25 | Given 入库数量=0 后删除，When 调用 DELETE，Then 库存回退后为原值（无变化） |
| INB-26 | Given 无 Token 请求，When 调用 GET /api/v1/inbound，Then 返回 401 Unauthorized |
| INB-27 | Given 过期 Token，When 调用 POST /api/v1/inbound，Then 返回 401 Unauthorized |
| INB-28 | Given SQL 注入 payload（remark="' OR '1'='1"），When 调用 POST，Then 创建成功，无 SQL 注入漏洞 |
| INB-29 | Given XSS payload（remark="<script>alert(1)</script>"），When 调用 POST，Then 创建成功，remark 原样存储 |
| INB-30 | Given 分页 page=0，When 调用 GET /api/v1/inbound?page=0，Then page=0 被修正为 1 |
| INB-31 | Given 入库单金额计算，When 检查返回数据，Then amount = quantity × price |
| INB-32 | Given 单位自动获取，When 创建入库单时不传 unit，Then 自动从物料表获取 unit |
| INB-33 | Given 关联 PO 入库超量，When 入库 quantity > PO.remainingQty，Then PO.status 变为 "completed"，received_qty 可能超过 ordered_qty |
| INB-34 | Given 入库后批次 expiry_date，When 检查批次记录，Then expiry_date 与入库单传入值一致 |
| INB-35 | Given 删除入库单后批次 remaining<=0，When 检查批次，Then batch.status=0（耗尽） |
