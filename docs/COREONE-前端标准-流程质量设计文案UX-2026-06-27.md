# COREONE 前端标准 —— 流程 · 质量 · 设计系统 · 文案 · UX（2026-06-27 定稿）

> **本文件是全项目前端改造的唯一标准。** 适用范围＝**整个项目所有前端页面**（不止医院盈利）。
> 本会话只**定标准、不改代码**；前端实际修改从**下个会话**起，逐页按本标准执行。
> 标准之间冲突时优先级：**安全/数据诚实 > UX > 文案 > 视觉**。配套：视觉令牌见 `V1.1设计稿/v1.1/DESIGN.md`，本文件为其超集。

## 0. 范围与原则

- **范围**：全项目前端逐页重设计（库存/出入库/采购/盘点/BOM/对账/ABC 各页/医院盈利/导入向导/系统管理…）。
- **设计方向**：**Stripe 风格**（浅色金融 · 海军蓝标题 · **蓝色强调**[用项目主蓝 `#3b82f6`，非 Stripe 紫] · 蓝调阴影 · 等宽数字），用户已选定。
- **最高原则**：**不准不如不算**——不确定/未校正/缺数据一律显式标注，绝不假装精确（呼应后端「增量纠错架构」记忆 `coreone-incremental-correction-architecture`）。

## 1. 协作流程（标准前端流程，强制）

把"判断对不对"挪到**最便宜**的阶段，避免在最贵的真实现后才返工。

| 步 | 动作 | 产物 | 谁判断 |
|---|------|------|--------|
| ① | **一次性锁设计系统**（token/组件/状态/文案/UX 规范） | 本文件 + 可视组件画廊 | 已定 |
| ② | **每页先出内联 mockup / 可点击原型**（show_widget，假数据；**含关键交互与状态**，见 §2.4） | 内联可点原型 | 用户在便宜阶段定稿（看 + 点） |
| ③ | **参考驱动**：用户给视觉/交互参考（截图/产品名/"像某页"） | 参考锚点 | 用户 |
| ④ | **mockup 定稿后**才接真 API + 写真代码 | 真页面 | —— |
| ⑤ | **我自验**（preview 跑真页+真数据，截图+console+network），过验收清单**才交付** | 带截图的成品 | 用户只审成品 |
| ⑥ | **一次一页**纵向切完到用户点头，**每页一小 PR**，决策入 DESIGN/记忆不重决 | 小 PR | 用户 |

**红线**：mockup 未定稿，不写真代码；成品未自验过清单，不交付；用户审的永远是"已能跑且我认为对"的东西，不是让用户去跑去 debug。

### 1.5 反馈协议（让验收高效 —— 把"翻译成文字"换成"点/圈/选"）
**问题**：纯文字反馈有损耗、指认不精确、来回多。**对策＝两边都用对工具**：

| 通道 | 谁主导 | 怎么用 | 适合 |
|---|---|---|---|
| **A. 结构化点选**（mockup 自带反馈器） | 我建，你点 | 每个 mockup 带①区域编号 + 反馈面板（**快捷 chips + 每区一个自由输入框**[预设不准就直接说] + 备注 → **一键合并成一条提交**；或「✓ 这版定稿」一键过）。`sendPrompt` 把点选发回 | 常规调整、快速定稿 |
| **A+. 框选生成反馈**（内置） | 你框，我读 | 开"框选"模式 → 在 mockup 上**拖一个框** → 我拿到你框中的**区域 + 实际内容** → 你写一句就精确定位，**不写我也能据此给 2-3 个改法选项**。等于把"截图画圈"做进 mockup，连截图都省 | 精确指认、"这块不对说不清" |
| **B. 批注截图** | 你 | 截我的输出 → 系统标注画圈/箭头/叉/写字 → 粘回；我读批注 | 复杂布局、跨多处 |
| **C. 变体选择** | 我给 | 争议处我并排 A/B/C，你选字母（选 > 描述） | 密度/配色/布局取舍 |
| **D. 参考图** | 你 | 粘你喜欢的界面截图当视觉锚点（比"做好看点"准十倍） | 定方向 |

**原则**：① 一次过（我每版附一句 rubric，你一轮给全）；② **点选/框选 > 描述、图 > 字**；③ 满意一键定稿，不满意点/框具体处——都不必长篇打字。预设选项**永远配自由输入兜底**。
**桌面端提醒**：框选/点选内置控件 + 截图标注 + 粘参考图 + 语音口述，是你这端最省力的组合。

## 2. 质量保证机制（怎么保证产出质量 —— 流程之外单独定）

质量 = **一把标尺 + 三道关 + 小步交付**。

### 2.1 标尺：每页验收清单（Definition of Done）
一页"做完"必须逐项过，缺一不算完：
- [ ] **用设计系统**：只用 §3 的 token/组件，无一次性魔法样式/魔法数字。
- [ ] **五态齐全且正确**：空（给下一步引导，非死胡同）/ 加载（骨架，非闪烁 spinner）/ 错误（可重试 + 说人话）/ 无权限（说清 + 指路）/ **数据未校正（显式标注）**。
- [ ] **文案过 §4**：术语表一致、自然中文、无直译感。
- [ ] **交互过 §5**：反馈即时、防错优于纠错、同类同模式、键盘可达。
- [ ] **权限正确**：按 `capabilities` 能力门控（nav + 页内）。
- [ ] **响应式**：桌面优先，窄屏（≥768）不破（表格横向滚动，不挤压）。
- [ ] **真实数据自验**：用真 API + 真数据渲染验证，非仅 mock。
- [ ] **无回归**：`vite build` 绿 + 相关单测/E2E 绿。

### 2.2 三道关
1. **便宜关（方向）**：②mockup / ③参考 定稿 → 用户点头。**防方向错**（最贵的返工）。
2. **自验关（交付前，我做）**：preview 跑真页 → 截图 + `console 0 error` + `network 接口/字段对` + 五态演示 + 窄屏 → 验收清单逐项过 → 才交付。**防交付破东西/不一致**。
3. **复核关（高风险/批量页）**：对抗式多代理 或 Codex 独立复核（bug/一致性/UX/文案/可达性），同后端那两轮深审。**防隐藏缺陷**。

### 2.3 交付单元
**一页一小 PR**，PR 带：真实截图（含五态）+ 验收清单勾选 + 自验证据（console/network）。决策写进本文件/记忆，不重决。

### 2.4 交互验收（静态图验不了交互 → 两层验）
视觉用 mockup 验；**交互（点行下钻 / 导入向导多步 / 空→加载→错→重试 / 表单即时校验 / hover）必须"点着走一遍"才验得了**。
1. **便宜阶段 —— 可点击原型**：mockup 升级成 `show_widget` JS 状态机 + 假数据，用户**实际点、走流程、触发每个状态**，在交互层定稿。
2. **契约 —— 交互清单**：每页附 `每个交互 → 预期行为 + 各态` 清单；原型与真代码都必须满足。
3. **真代码后 —— 我自验**：用 preview 工具驱动真页，把每个关键交互截成**动作序列**给用户 + 跑 **E2E** 关键流，证明真页＝原型。
4. **交互正式进 DoD**（§2.1「交互过 §5」做实）：缺交互验证不算完。

## 3. 设计系统（Stripe · 组件/样式/UI）

> 已在组件画廊可视化定稿。下为令牌与组件清单（实现时落成 Tailwind class 常量 + 复用组件）。

### 3.1 色彩令牌（role-based）
| 角色 | 值 | 用途 |
|---|---|---|
| 强调/主操作 | `#3b82f6`（项目主蓝，用户定；hover `#2563eb`） | 主按钮、链接、选中态、关键数据线 |
| 标题 ink | `#0a2540` | 页/卡标题、医院名、关键数值 |
| 正文/次要 | `#64748b` | 正文、表体 |
| 弱化/标签 | `#94a3b8` | 标签、占位、元信息 |
| 盈利/正 | `#059669` | 正毛利、增长 |
| 亏损/负 | `#e11d48` | 负毛利、错误、危险 |
| 未校正/警示 | `#d97706` | 数据未校正、警告 |
| 表面/分隔 | `#ffffff` / `#f8fafc` / `#e2e8f0` | 卡面 / 表面 / 边框 |

### 3.2 字阶 · 形状
- 字体 `Inter`；**金额/百分比/计数一律 `tabular-nums`**。
- 页标题 24/600（-0.02em）· KPI 数值 21–22/600 · 卡标题 13/600 · 正文 13/400 · 标签 11–12/400。**只用 400/500/600 三档**。
- 圆角：卡片 `12px`、控件 `6–8px`；控件高 `38px`。
- 阴影（Stripe 蓝调）：`0 2px 5px -1px rgba(50,50,93,.07), 0 1px 3px -1px rgba(0,0,0,.05)`；focus ring `0 0 0 3px rgba(59,130,246,.1)`（主蓝 10%）。

### 3.3 组件清单（复用，不重造）
按钮（主/次/幽灵）· 输入&日期&下拉 · KPI 统计卡 · 卡片 · 表格（表头/普通行/负值行红底/选中行紫环/hover）· 徽章（已核实/未校正 N/成本未归集）· 状态胶囊（亏损计数/同业对比标注）· 分页 · Tab · 空态/加载骨架/错误态/无权限态 · 导入向导骨架（解析→预览→导入三步）。

## 4. 文案 / 中文措辞规范（自然中文，不直译）

> 现有文案部分像英→中直译（"未校正"、"成本未接通"、"benchmark 未病种校正"）。统一按下表。

### 4.1 语气原则
- **动词开头、简短**：导出报表、刷新数据、导入病例、重新计算。
- **提示有温度但专业**：「数据没能加载，点击重试」＞「加载失败」。
- **空态给方向**：「本期还没有医院盈亏数据，先导入病例和对账单」＞「暂无数据」。
- **不向用户暴露内部字段名/英文黑话**（P&L、benchmark、partial_quantities…）。
- **数字格式**：`¥` + 千分位；百分比一位小数；负值用真减号 `−` + 红色 + 文字三重表意（不只靠颜色）。
- **全角标点**；中英文/数字间留一个空格（统一）。
- **同一概念全站同一词**（术语表为准）。

### 4.2 术语表（用左、弃右）
| 用 | 弃（直译/黑话感） | 说明 |
|---|---|---|
| 损益 / 盈亏 | P&L | 「各医院盈亏」「医院损益看板」 |
| 实收 / 实收金额 | —— | 财务实收可保留 |
| 实验室收入 | lab revenue | |
| 成本 / 作业成本 | ABC 成本（技术语境可留） | 面向用户用「成本」 |
| 毛利 / 毛利率 | gross margin | |
| 亏损（医院/case） | 负毛利（内部可） | UI 面向用户用「亏损」 |
| 同业对比 / 横向对比 | benchmark | |
| **估算 / 口径待核** | **未校正** | 见 4.3 |
| 暂无成本 / 成本未归集 | 成本未接通 | |
| 按技术占比折算 | 实收 × 技术占比 | |
| 亏损优先 / 亏损置顶 | 负毛利置顶 | |
| 汇总（往上一层） | 上卷 | OLAP 黑话；面向用户用「汇总」 |
| 下钻 / 看明细 | 下卷 | 面向用户用「下钻」 |
| 入库 | IN | 库存方向用中文（`/inbound`=入库记录） |
| 出库 | OUT | 库存方向用中文（`/outbound`=出库记录） |
| 实收合计 / 结算金额合计 | 服务项结算之和 | 病理对账口径黑话 |

### 4.3 重点改写（before → after）
- 「院级 P&L · 负毛利置顶」→ **「各医院盈亏 · 亏损优先」**
- 「未校正 N」→ **「估算 N 例」**（hover 解释：该院 N 例缺切片数据，技术/诊断拆分为估算）
- 「成本未接通」→ **「暂无成本」**
- 「benchmark 未病种校正」→ **「同业对比 · 未按病种校正」**
- 「实验室收入 · 实收 × 技术占比」→ **「实验室收入 · 按技术占比折算」**
- 「院级毛利 / 毛利率 27.0%」→ **「整体毛利 / 毛利率 27.0%」**
- 空态「本期暂无院级 P&L 数据」→ **「本期还没有医院盈亏数据，先导入病例和对账单」**
- 错误态「加载失败」→ **「数据没能加载，点击重试」**

## 5. UX / 交互规范（不只是 UI —— 单独定）

视觉一致只是底线；交互与流程同样要标准化。每页交互必须满足：

1. **反馈即时**：每个操作有 loading/disabled/toast 反馈；提交类**防重复**（disabled+loading）；可逆操作给"已保存/已撤销"。
2. **五态是"行为"不只是"有"**：空态给下一步引导；加载用骨架（不闪 spinner）；错误可重试且说人话；无权限说清并指路；**数据不全/估算显式标注**（产品级原则）。
3. **信息架构/导航**：能力驱动 nav；当前位置高亮；深层页有返回/面包屑；高频操作不藏二级。
4. **表格**：默认按有意义维度排序（如亏损优先）；可筛选/分页；行可下钻；空结果≠空白页；列对齐（金额/数字右对齐+等宽，文字左对齐）。
5. **表单**：失焦/即时校验 + 错误定位到字段；**危险操作二次确认**；不可用项**禁用并说明**（防错优于纠错）。
6. **一致性（最重要）**：同类流程同一骨架——**所有"导入"走同一向导**（解析→预览→导入）；所有"删除"同一确认框；所有列表页同一布局（页头+筛选+表+分页）。
7. **性能感知**：首屏骨架；并行加载（`Promise.all`）；大列表分页/虚拟化；不阻塞主线程。
8. **可达性**：键盘可达；focus 可见；对比度 ≥ AA；图标配 `aria-label`；**不靠颜色单独传意**（负毛利＝红＋`−`＋文字）。
9. **响应式**：桌面优先，窄屏（≥768）不破，表格横向滚动而非挤压。
10. **动效克制**：交互态 `transition-colors` 150ms；无炫技；尊重 `prefers-reduced-motion`。

> UX 质量由 §2 的「自验关」（我演示五态+交互）与「复核关」（专审 UX/一致性）共同保证。
