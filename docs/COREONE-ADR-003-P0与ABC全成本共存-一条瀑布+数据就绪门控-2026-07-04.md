# ADR-003 · P0 贡献毛利与现有 ABC 全成本毛利的共存

> **日期**：2026-07-04 | **状态**：已采纳（PM 拷问 Q5 + Claude 三视角对抗面板 `wf_c8309493` 补强）| **性质**：hard-to-reverse（决定 `/hospital-pnl` 的头条去留信号 + 两个"利润数"的关系），**动到已上线的 `/hospital-pnl`**。
> **关联**：`partner-pnl-service.ts:190`（ABC grossMargin）；`HospitalPnLDashboard.tsx`（消费方）；P0 spec；ADR-001/002；方法论固化 §3（贡献毛利判去留·死亡螺旋）+ codex HIGH1（分 lane）。

## 背景（Q5·已核实）

- `/hospital-pnl` 现在展示 `partner-pnl-service.ts:190` 的 `grossMargin = 实验室收入 − ABC 全成本`；`costTotal` 取 `outbound_abc_details.total_cost`，`cost-calculator.ts:379` = material+labor+equipment+qc+indirect（**含整块固定成本**）。
- P0 = `实验室收入 − 可避免材料`（不含工时/设备/间接）。**分子同、分母差一整块固定成本 → P0 ≥ ABC 恒成立。**
- 边际院必然出现 **ABC<0（全成本亏）但 P0>0（边际赚）** → 同一家院 ABC 说"砍"、P0 说"留" → 财务两个都不信 → **adoption 红线**。且这正是方法论警告的**死亡螺旋**（按全成本砍边际院 → 固定成本转嫁 → 更多院翻红 → 继续砍）。
- **现状代码把去留信号全长在 ABC 上**：worst-first 排序 `:41`、"负毛利 N 家"KPI `:112/:67`、负数染红 `:167`、负毛利筛查栏 `:207`、页脚"决策以院级毛利为准"`:230`。

## 决策

**不是"哪个数对"——两个都对，各答不同问题（内圈贡献毛利=去留/谈价；中圈全成本=长期够本）。问题是"两个利润数并列打架"毁信任。解法 = 一条瀑布 + 贡献毛利独占去留信号 + 全成本降级 + 数据就绪门控渐进切换。**

**1｜一条瀑布，不是两个并列数**：`/hospital-pnl` 单卡显示一条钱的瀑布：
```
实验室收入 → −可避免材料 = 贡献毛利（主·带去留状态）
            [默认收起，一键展开] → −工时/设备/分摊 = 全成本毛利（长期趋势·不作去留依据）
```

**2｜贡献毛利独占全部去留信号（把 ABC 从下游信号夺走·面板 trust#1/deathspiral#1）**：`lossCount`/"负毛利 N 家"KPI（`:112/:67`）、列表排序键（`:41`）、红色语义（`marginColor` `:167`）、负毛利筛查栏（`:207`）、页脚口径（`:230`）——**全部从 `grossMargin` 切到贡献毛利**。ABC 永不参与排名、永不用"亏损红"。

**3｜全成本降级为"看得见但不驱动"（面板 trust#4·答 Q2）**：同卡**默认收起、一键展开即完整**（**不移到另一个视图**——移走=坐实"藏亏损"）。展开态：中性灰/琥珀色（非 rose 红）、顶部硬门禁横幅"工时/设备/间接为分摊估值·不得用于去留/砍院/谈判·仅供成本结构长期观察"。标签用"**不作去留依据·仅长期趋势**"，**不用"估/不准"**（面板 trust#3：'估'触发不信并传染整表，'不作依据'只触发不用）。

**4｜因果锚 + 反事实，把负数重框（面板 trust#2/deathspiral#1）**：贡献毛利"可留"旁印一句因果（印在卡上，非 tooltip）：**"边际可留 = 多做省下 ¥X 变动成本；固定成本（工时/设备/房租）不管做不做都付，砍这家省不掉它。"** 并给"砍这家会怎样"的反事实提示：移除该院 → 其固定成本转嫁其余院、其余院全成本毛利变差——**在同屏戳破"砍=螺旋"**。

**5｜产能闸：贡献毛利>0 是保底不是背书（面板 deathspiral#2）**：`贡献毛利>0` = 淘汰绝对亏损院的**下限**，**不等于无条件"该留"**。产能受限时，"边际赚但占满瓶颈工位"的院会拖死——去留看**单位瓶颈资源贡献毛利**。状态表 `需限量` 从"仅人工置位"升为可判：`贡献毛利>0 且 每例CM<院群中位数 且 capacity_constrained`。文案层必须写死"贡献毛利>0=保底，是否该留还看它占多少稀缺产能"，不让 PM 把"可留"读成"该留"。

**6｜数据就绪门控·逐院逐月渐进切换（面板 trust#5/deathspiral#3·最关键，别一步到位翻口径）**：
- P0 贡献毛利 lane **现在没真数**（spec §7/§10.F：`case_revenue`/`lis_case_markers`/`antibodies` 空）。上线若"贡献毛利空 + 全成本藏" = 用空口径顶掉能用口径 → 财务一次崩。
- 规则：某院某月**三件套未齐** → 主瀑布显式显示"贡献毛利：需导入对账单+抗体清单后生成"（给补数动作，非空白），此期间**全成本不折叠、正常展示但标"过渡口径·数据补齐后由贡献毛利替代"**；**只有三件套齐、贡献毛利真出数** → 才切"贡献毛利主 + 全成本折叠"目标态。财务永不撞见"新的空 + 旧的被藏"。
- 加真实性门禁：`outbound_abc_details` 真实归集率低于阈值 → 全成本行只出"数据未就绪"占位、不出假数字（防假负数锚定）。

## 后果

- ✅ 财务在主视图只有**一个**驱动去留的数（贡献毛利），无"两个利润数打架"、无假负数警报。
- ✅ 死亡螺旋在信号层被切断（ABC 不喂红/排名/砍），且反事实提示在决策屏戳破。
- ✅ 全成本一键可达、完整 → 挡"藏亏损"指控；渐进切换 → 挡"空窗期"信任崩。
- ⚠️ **迁移工作**：改 `HospitalPnLDashboard.tsx` 的 `marginColor`/排序 `:41`/KPI `:112`/筛查栏 `:207`/页脚 `:230` 从 grossMargin 切贡献毛利；新增 P0 贡献毛利读侧 lane（spec §10.A，不经 outbound）。`grossMargin` 数本身不改（golden ¥13,152/¥27,870 是收入侧，不受影响；ABC 数值不动、只降级其呈现与信号角色）。
- ⚠️ 产能闸依赖 `capacity_constrained` 人工置位（产能数据不在三件套）——文案层先兜，数据层后补。

## 备选（未采纳）

- **退役 ABC grossMargin / `partner-pnl`**：否——全成本有长期定价/撤线的正当用途（中圈），且退役已上线消费方（`/hospital-pnl`）风险大（见新旧重叠处置清单 #4：partner-pnl 不可退役）。
- **全成本移到另一个视图**：否——坐实"藏亏损"，比并列更伤信任。
- **一步到位把头条从 ABC 翻成 P0**：否——P0 无真数时 = 空窗期信任崩。
