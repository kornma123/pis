# COREONE 写码质量 Loop（薄入口 · 实现阶段）

> **状态**：v1.0（2026-07-12）· 待 PM 定稿拍板
> **从属**：本文是《`docs/COREONE-质量Loop契约-2026-07-12.md`》在**实现阶段**的薄入口——只写本阶段增量。**六步骨架、五件套（及其「交付面让位 handoff 契约」边界，见契约 §2）、假设两级、异步节奏、证据分级、复核堆栈 L0/L1、通用定稿退出模板、护栏，全部在契约、本文不复制。** 冲突以契约与其上游（共用契约 / CLAUDE.md / pr-governance）为准。
> **输入**：功能实现 = 定稿 PRD + 定稿 mockup（缺一样回上游）；bug 修复 = 可复现路径 + 期望行为。**适用**：功能实现、bug 修复；纯文档改动不走本 loop。

## 0. 位置 + 分档

mockup 定稿 →【写码（本文）】→ PR + 独立复核 → PM 批准合并 → 真跑验收。
**先分档**：微改（纯文案/样式，不碰逻辑/钱/权限）= 轻量道（只走 ③机器关 + diff 自查）/ 常规 = 全走 / 碰钱·口径·权限 = 全走 + 第 3 节加固。拿不准按高的算。

## 1. 本阶段增量（契约之外，写码特有）

**① 开工核实**：preflight develop、fetch 现场 master、`gh` 现查分支/PR（别信记忆）。现状核实：要修的 bug 还在吗？要建的已有现成吗？（清单常失真）bug 修复铁律：**先复现再动手**（在提交态 dev 库上复现，别先 seed 洗掉现场）；复现不了就把「复现不了」作为发现交出去，不硬修。
**② 写码**：一轮一个可运行可验证小目标，禁「一把梭到最后才第一次跑」。发现定稿 PRD/mockup 不合理 → 停，记进**偏离清单**回上游拍，不码上自由发挥。
**③ 机器关**：相关测试 + 黄金锚必绿（红了如实说红，不「应该没影响」）、构建纪律闸/lint 过、`git diff` 自查（无越界文件/没碰别人 worktree/dev 库已复原/无临时日志或调试代码）。
**④ 变异证有牙（写码特有的牙）**：新写的每个关键测试，先人为改坏实现 → 测试必须变红 → 改回 → 绿。**抓不住坏实现的测试等于没写。** bug 修复：修完用**同一条复现路径**证明不再复现。
**⑤ 交付走 handoff 契约**：PR body 唯一字段源 = TEMPLATE + `check-pr-body.cjs`（契约 §2），**不自造字段名**；**偏离清单、变异证据作内容要求并入既有字段**（变更摘要 / 验证），不并列一套。
**⑥ 复核堆栈（契约 §6）**：L0 = `ai-review-gate` 的评论**逐条处置**（修 / 讨论项 / 误报附反证），L0 红 = 机器关未过与 vitest 同级；碰钱档触发 **L1 定向对抗**（留定式产物：镜头清单/结论/CONFIRMED-REFUTED 计数/run 指针，**无产物 = 未做**）。

## 2. 写码特有退出追加项（叠加契约「通用定稿退出模板」，不重写那四条）

- 机器关全绿（相关测试/黄金锚/纪律闸）；
- 关键测试做过变异验证；bug 修复有「同路径不再复现」证据；
- 偏离清单零未拍残留；
- L0 评论逐条处置完、碰钱档 L1 有定式产物；
- **合并 = PM 批准后执行，本 loop 不含自动合并。**

## 3. 碰钱/口径/权限档加固（叠加契约）

① 黄金锚零漂移是硬门（逐位一致，漂了先解释清楚再谈合并）；② 每个关键数字带等级 + 源（契约 §5），不许拿测试自洽把 C 悄悄标 A；③ 数值边界专项（Infinity / 超界 / 空串 写库前单独过一遍，这类坑单测常漏）；④ 权限对照运行库真实权限（非源码种子矩阵）；⑤ L1 升格为多镜头对抗面板。

## 4. 会话注入块（开工贴给 AI）

```
【写码质量 Loop 已激活】按《质量Loop契约》+ docs/COREONE-写码质量Loop-2026-07-12.md 执行：
1. 功能实现输入=定稿PRD+定稿mockup；发现定稿不合理→记偏离清单回上游拍，不码上自由发挥。
2. 先分档(微改/常规/碰钱)，拿不准按高档走。
3. 每轮:小目标写码→机器关→变异证有牙→摊偏离→(碰钱)L1对抗；红了如实说红。
4. bug先复现再动手，修完同路径证明不再复现。
5. 交付走handoff契约字段(不自造名)，偏离清单/变异证据并入既有字段；L0评论逐条处置、L1留定式产物。
6. 合并等PM批准，不自动合并。
```

## 5. 给 PM 的大白话

真正写代码这段：AI 每写一小块就自测一轮，还故意把代码改坏一次、证明测试真抓得住错（抓不住的测试白写）。发现你拍过板的需求/草稿有问题不许自作主张，记下来回头找你重拍。提交给你批准前要过机器的 AI 复核门（它的意见逐条处置、它红了当机器关没过）、涉及钱的还要第二个 AI 独立挑刺并留下挑刺记录（没记录当没做）——它自己不许合。改错别字这类小事走简化流程不兴师动众；碰钱碰权限层层加码。
