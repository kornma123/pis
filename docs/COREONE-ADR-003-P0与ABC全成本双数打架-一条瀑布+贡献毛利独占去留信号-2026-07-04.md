# ADR-003 · P0 贡献毛利 与 现有 ABC 全成本毛利"两个利润数打架" —— 一条瀑布 + 贡献毛利独占去留信号

> **日期**：2026-07-04 | **状态**：已采纳（PM 拷问 Q5）| **性质**：hard-to-reverse（决定 /hospital-pnl 头条去留信号 + P0 与既有 ABC P&L 的关系），且**动到已上线的 /hospital-pnl**。
> **关联**：`partner-pnl-service.ts:107` grossMargin；`HospitalPnLDashboard.tsx`；方法论固化 §6 codex HIGH1（现 grossMargin=全成本）；ADR-001/002。
> ⚠️ 本 ADR 的对抗面板（`wf_c8309493`）中途被停、未出完整结论；决策基于本地已核事实 + 前几轮 codex/红队结论，**迁移实现前需补一次 file-level 复核**。

## 背景（PM Q5 已核）

- `/hospital-pnl` 现展示 `partner-pnl-service.ts` 的 `grossMargin = 实验室收入 − ABC 全成本`（含工时/设备/间接分摊，来自 `outbound_abc_details`）。
- P0 的数 = 实验室收入 − 可避免材料（不含工时/设备/间接）。
- 两者**分子同（实验室收入）、分母差一整块固定成本** → **P0 永远 ≥ ABC**。边际院必现：ABC<0（全成本亏）但 P0>0（边际赚）→ **同一家院 ABC 说砍、P0 说留**。
- 顶层风险**不是算错，是两个"利润数"打架、互相摧毁可信度**：财务看到"ABC 说亏、P0 说留"→ 两个都不信 → 踩 adoption-first 红线（有用但没人信→没人用→失败）。
- 补充事实：ABC 全成本这条现**跑在 seed/未认真统计的出库数据上**（数据真空），今天本就不是真数。

## 决策

**根不是"哪个数对"——两个数各自对不同问题**（贡献毛利=短期去留/谈价下限·内圈；ABC 全成本=长期够不够本·中圈）。**错在把它们当两个并列的"利润"数展示。** 故：

1. **/hospital-pnl 改成一条【瀑布】，不再并列两个"利润"数**：
   ```
   实验室收入
     − 可避免材料(一抗/二抗显色/特染/组织处理)
     = 贡献毛利            ← 【主】去留/谈价看这行
     − 工时/设备/房租/间接分摊
     = 全成本毛利          ← 长期趋势看这行 · ⚠️不作砍院依据(死亡螺旋) · 标"估·数据未就绪"
   ```
   财务看到的是**同一家院的钱逐层往下走**，不是两个互相矛盾的断言——"矛盾"因可见的一条分解而消解。
2. **去留/砍院的状态推荐，只由贡献毛利（+产能+缺价+数据完整度）驱动**；全成本毛利那行**不喂任何红色/砍院状态**——死亡螺旋在**决策层**被堵死，不只是标签层。
3. **头条去留信号从 ABC grossMargin 迁到 P0 贡献毛利**；ABC 全成本**降级为瀑布里一行**、显式标"长期参考·不砍院·估(数据未就绪)"。
4. **永不并列两个"利润/毛利"标签**；每行按**决策**命名（"该不该继续接单"=贡献毛利；"长期够不够本"=全成本）。

## 后果

- ✅ 解掉"两个数打架、财务都不信"——一条瀑布 + 按决策贴标签 + 状态只由贡献毛利驱动。
- ✅ 死亡螺旋在决策层堵死（砍院状态不可能来自全成本行）。
- ✅ 与方法论一致（去留用贡献毛利、固定成本不下沉）。
- ⚠️ **迁移动到已上线 /hospital-pnl**：头条换数 + ABC 降级为行。需：核 `partner-pnl-service.ts`/`HospitalPnLDashboard.tsx` 消费点、确认 golden(¥13,152/¥27,870=收入侧，不受成本展示影响)零回归、codex 早前的"价源契约测试"。**列为迁移实现项，PM 拍板方向后另立。**
- ⚠️ 全成本行现无真数（seed）→ 上线初期该行标"数据未就绪"，甚至可默认折叠，避免"你在掩盖亏损"的反向不信任（面板未验，迁移时定）。
