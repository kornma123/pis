> **依据**：`~/Desktop/方案-后续四项-实施计划(内部·真实标识符).md` §四 ①止血 + §三 导出口径声明列。

## 一、为什么要这个

政策分摊常量 `SPLIT_DIAG_FEE = 105`（`后端代码/server/src/utils/statement-revenue.ts`）单方面决定「范围内(实验室)收入 vs 范围外(诊断桶)」的份额分界。它派生出的**实验室收入 / 院级毛利**结论，对外**可能显著高估约 2 倍**，且**业务方尚未认账**。

「止损」不能只靠口头通知，要有**执法机制**——让任何消费该结论的**对外输出自带「口径未认账」水印**、让**导出携带口径声明列**：防旧材料被当现状转发（传播算新增使用、生成时间不豁免），防导出把免责声明剥离（导出是免责声明被剥离的头号通道）。

## 二、交付了什么（三件·全部只加展示/元数据，不动钱的计算）

### 1. 后端响应水印（机器执法层）
新增纯口径工具 `后端代码/server/src/utils/caliber-ratification.ts`：`splitCaliberRatification()` 返回 `{ ratified:false, state:'UNRATIFIED', sourceTag:'derived', basisVersion, label:'口径未经业务认账', note, ratifiedAt:null }`。

挂到**每一个消费拆分结论的对外输出响应**上（`caliberRatification` 附加字段，不改原有数字字段）：

| 路由 | 端点 | 携带的拆分派生数字 |
|---|---|---|
| statement-import | `POST /preview`、`POST /commit` | `labRevenue` / `diagnosisSettle` |
| hospital-pnl | `GET /health`、`GET /` | `totalCm` / `coverageMultiple` / `cm` / `cmRate` |
| partner-pnl | `GET /`、`GET /cases` | `labRevenueTotal` / `grossMargin` / `marginRate` |
| account-reconcile | `GET /overview`、`GET /workbench`、`POST /hospital-months/:id/complete` | `confirmedLabRevenue` |

> **精度**：裸时序数组端点（`/trend`）形状不改（防破坏消费者，水印在同页 overview 响应上已带、与趋势图同视线）；非拆分消费端点（如 `/cross-partner-audit`）**不误挂**水印。

### 2. 前端渲染（与数字同视线·LEG-2）
`前端代码/src/pages/hospital-pnl/HospitalPnLDashboard.tsx`（当前线上仍在用的旧盈利视图，正是承载「高估约 2 倍」数字的对外输出）：在实验室收入/毛利数字**正上方**渲染醒目水印横幅 + 在院级盈亏表头挂「口径未认账」徽标。
- **fail-closed**：只有后端**明确回 `ratified===true`** 才免水印；字段缺席（旧缓存/丢字段）→ **仍显水印**（宁可多提示，不可漏提示）。
- 认账落地后（后端把占位接到真实状态、回 `ratified:true`），前端**自动摘牌**，无需改前端。

### 3. 导出口径声明列（机制就绪 + 诚实天花板）
工具提供 `buildExportDeclaration({ exportedAt, periodRange })` + `decorateExportRows(rows, decl)`：给每一行导出**逐行**追加 `_sourceTag / _basisNote / _basisVersion / _exportedAt / _periodRange / _ratified` 六列——旧导出永生在邮件里时**自己声明自己「未认账 + 过时」**，声明随每行走、裁到单行也剥不掉。

## 三、诚实天花板 / 覆盖边界（§四.4·必读）

- **当前没有任何导出通道消费拆分结论**：全仓核实（见 PR 复核记录），所有已接线的导出（前端 `pages/cost/*`、`pages/report/*`、后端 `abc-cost-export`）都是 **ABC 全成本口径**，**不含**拆分派生的实验室收入/院级毛利；账实核对/对账页的「导出」按钮是**未接线占位**。故本次**没有可挂声明列的现存拆分导出**——`decorateExportRows` 作为**执法就绪机制**交付，第一个真去导出这些数字的通道**按构造即带声明**（配套单测证明装配正确）。
- **启用本机制之前产生的导出无法枚举**：此前已流出的导出文件（邮件/表格里的旧副本）无法追溯加签、无法穷举。存量导出通道的覆盖边界须在**业务侧取证结论**里写明——工程侧管不了历史副本。
