# 院级贡献毛利真实解锁闭环 mockup

## 目的

本原型只验证 `/hospital-cm` 既有两层结构中的三种状态，不增加第三层或新仪表盘：

1. **校准态**：真实证据不全时，完整体检组件不进入 DOM；未知金额不显示为 `¥0` 或 `0%`。
2. **真实证据齐全**：四道门均由当前有效证据满足后，完整体检出现，并同屏展示数据截止、固定池版本、历史周期与独立复核证据。
3. **值变化后失效**：固定池值版本、口径或输入证据变化后自动退回校准态；旧完整数值被收回，组件从 DOM 移除。

打开 `index.html` 后可点击顶部三个状态按钮。页面中的全部金额、月份、账户数、百分比、版本与证据标签都只是交互示例，不对应真实医院、周期或既有 golden，也不能成为 readiness 证据。

## 与捕获基线的关键差异

设计核对基于 `origin/master@c06166331f37`（2026-07-12）完成；该基线与原型的渲染截图属于 PR 评审附件，不进入 master，也不代表打开文档时的实时 master。`index.html` 是本目录唯一权威、可交互的 mockup。

- 捕获基线的空数据会显示 `¥0` 和未测覆盖 `0%`；原型改为“暂无可计算数据 / 金额覆盖未知”。只有财务月度名册完整、可测金额完整、未知金额均属于已识别未测账户、金额非负且分母大于 0、期间/币种/口径一致、集合互斥无重复时，才展示公式化下限；否则保持 `null`。
- 当前清单只有结果；原型让每道门可展开查看来源、检查时间、版本和失效条件。
- 历史成本失真从脚注升级为逐周期状态；不可核周期断线且不计入三个有效周期。
- 纯代送/会诊院缺可靠金额时保持“不可测”；先显示已识别账户、已知金额与缺失原因，不把未知折成零。满足完整名册与分母前提后才可给公式下限。
- ready 后仍不自动点名、排名或生成砍院/谈价清单；默认继续按绝对贡献展示。
- 拆分口径是否升级为完整态硬门仍待 PM 拍板；原型不把它画成 A 已实现的地基门。

## PM 验收问题

- 没有可算数据时，是否同意主数显示“暂无可计算数据”，而不是 `¥0`？
- 医院全集或可测收入分母不完整时，是否同意未测占比显示“不可计算”，只给已识别账户、已知金额和缺失原因？
- 是否同意每道门必须同屏可追溯到来源、版本、最后检查时间和失效条件？
- 是否同意固定池值、公式口径或证据变化后，系统立即收回完整态及旧数值？

## 交互验收目标

- 校准态：`full-physical-exam` 数量为 `0`。
- 就绪态：`full-physical-exam` 数量为 `1`。
- 值变化后失效：`full-physical-exam` 数量恢复为 `0`。
- 三态切换无浏览器控制台错误。

实时浏览器验收结果与渲染截图放对应 PR；它们只证明交互与视觉核对过程，不能成为 readiness 证据。
