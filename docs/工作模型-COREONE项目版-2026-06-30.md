## 1. 本项目的硬实证（模型有效性，可引用说服）

**前置会（讨论 + 摊假设 + 真数据）在写一行代码前就抓出：**
1. **功能形态搞反**：财务导入是"批量多文件按医院名分发"，AI 一度理解成"一张张导" → 讨论当场纠正，避免按错靶子盖楼。
2. **收入口径系统性错**：原"实验室收入"把医生诊断/报告/现场服务都算进去了 → 真·纯实验室 全月 ¥55,541→¥27,870，**高估约 2 倍**。产品级纠正，全靠讨论 + 读真代码/收费代码。

**"手核答案要靠独立现实、不能自证"的实证**：
- 假黄金 → 真黄金：自证的 ¥13,152（手填 net 求和）→ **LIS 双核验的 ¥27,870**（免疫组化 LIS80≈账单79、特染2=2、蜡块260>部位150，165病例100%匹配）。
- 精度靠真系统数据：数量-忽略 22,835 → 数量估 25,772 → **LIS 真蜡块 27,870**（估算比真数据低 22%）。

**独立复核抓出**：G2 模型 3 处算术错、preflight 对"黄金全自证"的误判——都在被当真前拦下。

**结论**：本项目最大的价值在**前置会**——把 UU-2（AI 误读意图）在最便宜的阶段（无代码）解决掉。

## 2. 本项目专项决策 / 口径（模型产出，供实现遵循）

- **纯实验室收入口径**：见 `COREONE-G1收入口径-…-2026-06-30.md`（IN=制片/染色；诊断/报告/现场=诊断桶；外送/共建=外送桶；老码按 `36×LIS蜡块/(36×LIS蜡块+105)` 逐病例拆）。golden：全月 **¥27,870** / W4 ¥7,118。
- **成本口径**：见 `COREONE-G2成本基准-…-2026-06-30.md`（单位技术成本；耗材硬、工时间接估；工作量从 LIS 来）。
- **架构含义**：收入计算依赖 **对账单 × LIS 按病理号 join**（制片按真蜡块），不只是分类器。
- **说人话黑名单**（本项目 lint）：上卷/下卷/rollup/IN·OUT/ABC/P&L/结算之和/技术占比… 见 `coreone-ui-copy-plain-chinese`。
- **前端红线**：mockup 先行、亮色控件显式白底深字、Stripe 设计系统。见 `coreone-frontend-standards`。
- **codex 操作铁律**：串行单跑勿并发、pkill 清僵尸、token 注入不填表单、快速失败。见 `coreone-codex-deep-review`。**异机中继预案**：本机被公司 VPN 阻断 codex 时，在另一台设备跑 codex + 结果经 GitHub（push/PR/findings 文件）中继回来——已在用（findings/09、/10 即此产出），不是"独立轴离线就零预案"。
- **上线后黄金撤回仪式**（碰钱纠错，接增量纠错架构 `coreone-incremental-correction-architecture`）：一个已登记/已进 CI 的黄金事后被发现口径错时——① 改 golden-registry 那一行（标"已撤回/重算"+ 原因 + 日期，不删历史）；② 明确**下游重算范围**（哪些派生数字/报表依赖它）并重跑；③ 留痕（原始不可变、派生可重算、推断可覆盖但留痕）。先例：G-REV-1 ¥13,152 被 codex 09 判"非实验室收入"时只做了 ad-hoc 更名、没走这套——补齐。**前置依赖**：撤回仪式要真生效得先有 CI 锁（backend-tests.yml 黄金门禁已于 2026-07-02 落 master + master 分支保护 required=vitest 已设）。
- **历史报表口径演进处理**（决策④，2026-06-29 拍板）：口径变更（如 ¥55,541→¥27,870）后——**已结账的月份出调整单**（不改历史账），**未结账的重算**。避免同一月出现两份自相矛盾的数。

## 3. 本项目真实数据源（做手核答案用）

`~/Downloads`：`单据2026062607544902号收费单据.xls`（W4含收费代码）、`2026年对账单.7z`（结算表，py7zr解压）、`病例导出文档20260701 (1).xls`（LIS，含蜡块/免疫组化/特染数）、`病理类项目收费代码-YZ.xlsx`（国标81码）、成本三件（结转成本/免疫组化耗材/常规耗材）。复现：`docs/analysis/hemujia-golden-lis-join.cjs` + `docs/analysis/handcheck/`（T1 手核脚本：收入拆分/毛利 band/一抗成本）。

> ⚠️ **文件名编码坑**：这些文件在磁盘上是**浏览器下载的百分号编码**形式（如「对账单」存为 `%E5%AF%B9%E8%B4%A6%E5%8D%95`）。用字面中文 `find -name "*对账单*"` 会**扑空**、误判"数据源蒸发"——文件其实都在。搜索改用编码无关模式（`*.7z`/`*.xls`）或先 `python3 -c "import urllib.parse; ..."` 解码文件名。（2026-07-01 曾有一轮据此误报"数据源蒸发"，实为假警报。）

## 4. 演进机制（怎么让下个会话接住 —— 本项目侧）

1. **入口**：`.claude/session-log.md` + 本文档互指；会话启动读 session-log → 读本文档 + 通用版。
2. **改哪份**：普适机制 → 改**通用版**；本项目实证/口径 → 改**本项目版**。每次修正在下方「变更记录」追一条。
3. **交接**：会话结束在 `session-log` 留指针（当前状态 + 下一步 + 对模型的修正）。
4. **待办承接**：codex 复核✅ → Phase 2 实现✅（红→绿 ¥27,870，零回归 507）。下一步 = 接线 `/preview`·`/commit` 喂 LIS 蜡块（月结向导按病理号取 LIS）；对账单 × LIS 按月份归属（结算表月，非登记日期）。
