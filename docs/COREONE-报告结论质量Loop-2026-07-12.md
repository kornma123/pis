# COREONE 报告/结论质量 Loop（调研 · 审计 · 评估 · 复盘类产出）

> **状态**：v1.0 草稿（2026-07-12）· 按「产出必过质疑关」规矩，PM 质疑/拍板后生效
> **从属关系**：「产出必过质疑关」在**结论类文档**上的执行细化。横切工具，不挂在主链某一段：只要产出物是「结论 / 判断 / 清单」（调研报告、功能审计、系统评估、事故复盘、去留盘点），就走本 loop。
> **为什么需要**：这类产出在本项目翻过真车——评估报告带硬伤、行业调研被对抗复核剔掉十几条失真、把从未上线的功能当在产功能做了完整审计、复核视角拿过期事实过度断言。病根一致：**断言没分级、取证没现场、结论没被反攻过**。

## 0. 位置与使命

```
定靶 → 【报告/结论质量 Loop（本文）】→ 结论交付（供 PM 决策）
```

使命：交到 PM 手上的每个关键断言，要么是现场核实过的，要么明确标着「这是推测」。**不许让 PM 拿着失真结论做决策。**

## 1. Loop 全景

```
┌─> ① 定靶（回答什么问题、给谁看、要支撑什么决定）
│   ② 取证（现场实测 > 文档引用 > 记忆/推测；可用性以实测运行为准）
│   ③ 成稿（每个关键断言标证据等级 A/B/C）
│   ④ 反向核查（把每条结论当假设，主动找反证）
│   ⑤ 对抗复核（第二视角专挑失真）
│   ⑥ 交付（结论 + 证据分级 + 不确定清单 + 大白话）
└── 复核挑出失真 → 回 ② 重新取证
    高危断言全达标 → 定稿
```

## 2. 核心规矩

### 断言分级（本 loop 的地基）

- **A · 现场实测**：亲自跑过 / 查过活代码 / 查过运行库 / 用真数据算过。
- **B · 出处引用**：来自文档、日志、记录，附出处；并且明知「文档 ≠ 现状」。
- **C · 推测**：显式标注「推测」。

**要支撑 PM 决策的断言必须是 A 级**；达不到就写明「未核实」，不许用肯定语气盖过去。

### 失真高发形态自查清单（每条都来自本项目真实翻车）

- [ ] 把「文档 / 清单 / 记忆说」当「现状是」——清单常失真，核实先行
- [ ] 把旧快照当实时——分支、PR、CI 状态一律现查
- [ ] 把「名字出现过」当「可用 / 在用」——工具、能力、功能是否可用，以实测运行为准
- [ ] 审计对象没先确认是否真在产——从未上线的功能不值得按在产功能做完整审计
- [ ] 比对基准没验明——diff / 对照类结论，先确认比的是对的两个东西
- [ ] 没拿到结论先写「已完成 / 已兜底」——措辞纪律，等真结论落地再写
- [ ] 数字不可复算——每个关键数字给出算法和输入，别人要能重算出来

### 检测与处方分离

报告只做**诊断**（是什么、证据是什么）；要不要修、怎么修单独列为**建议**，决策权在 PM。诊断和处方混写 = 用结论夹带主张。

### 取证纪律

- 报告类任务只读取证，不顺手改生产文件。
- 每查一处记出处（文件:行 / 命令 + 输出 / 数据来源），复核方要能沿着出处走一遍。

## 3. 退出条件（全部满足才算「定稿」）

1. 支撑决策的断言全部 A 级，或显式降级标注「未核实」
2. 反向核查跑过：每条高危结论找过反证并记录结果（找到了什么 / 没找到）
3. 对抗复核意见全处理（采纳，或给出证据反驳）
4. 不确定项显式成节——没有被删掉，也没有藏进模糊措辞里
5. 关键数字全部可复算
6. 结尾大白话齐

## 4. Loop 护栏

| 护栏 | 规则 |
|---|---|
| 防断言通胀 | 复核/对抗视角同样守证据分级——挑刺也不许拿过期事实过度断言 |
| 防体量崇拜 | 报告价值 = 断言质量，不是页数和视角数量；长报告开头必须有一页结论摘要 |
| 防静默截断 | 只审了一部分就说清边界（「只看了 X 没看 Y」），别让局部结论冒充全景 |
| 防结论漂移 | 修订轮不许悄悄改已交付结论；要改就显式写「此前结论 X 有误，证据是 Y」 |

## 7. 给 PM 的大白话

这份文档管所有「给结论」的活：调研、审计、评估、复盘。核心规矩是 AI 说的每句要紧的话都得标明来路——是它亲自跑过验过的，还是从文档里抄的，还是它猜的；给你做决策用的结论必须是亲自验过的那种。写完之后它还要自己当反方攻一遍自己的结论、再让第二个 AI 专门挑失真，不确定的事必须白纸黑字列出来，不许用「应该没问题」糊弄过去。以前吃过的亏（报告里的硬伤、把没上线的功能当在用的来审）就是这套规矩要堵的洞。
