# COREONE 报告/结论质量 Loop（薄入口 · 调研·审计·评估·复盘）

> **状态**：v1.0（2026-07-12）· 待 PM 定稿拍板
> **从属**：本文是《`docs/COREONE-质量Loop契约-2026-07-12.md`》的**横切**薄入口——只要产出物是「结论/判断/清单」（调研/审计/评估/事故复盘/去留盘点）就走。**证据分级 A/B/C 的完整定义在契约 §5、六步骨架/异步节奏/复核堆栈/退出模板全在契约，本文不复制。** 冲突以契约与其上游为准。
> **为什么需要**：这类产出在本项目翻过真车（评估报告带硬伤 / 行业调研被剔 14 条失真 / 把从未上线功能当在产审计 / 复核拿过期事实过度断言）。病根一致：**断言没分级、取证没现场、结论没被反攻过。**

## 0. 位置与使命

定靶 →【报告/结论（本文）】→ 结论交付（供 PM 决策）。使命：交到 PM 手上的每个关键断言，要么现场核实过、要么明标「推测」。**不许让 PM 拿失真结论做决策。**

## 1. 本阶段增量（契约之外，报告特有）

**① 断言分级 = 契约 §5 的 A/B/C**（定义以 §5 为准、本文不复述）；本阶段义务 = **支撑 PM 决策的断言必须 A 级**，达不到写明「未核实」、不许用肯定语气盖过去。
**② 失真高发自查清单**（每条都来自本项目真实翻车）：□ 把「文档/清单/记忆说」当「现状是」（清单常失真、核实先行）□ 把旧快照当实时（分支/PR/CI 现查）□ 把「名字出现过」当「可用/在用」（以实测运行为准）□ 审计对象没先确认是否真在产 □ 比对基准没验明（diff 先确认比的是对的两个东西）□ 没拿到结论先写「已完成/已兜底」□ 数字不可复算（给算法和输入）。
**③ 检测与处方分离**：报告只做**诊断**（是什么/证据是什么）；要不要修/怎么修单独列**建议**，决策权在 PM。混写 = 用结论夹带主张。
**④ 取证纪律**：只读取证、不顺手改生产文件；每查一处记出处（文件:行 / 命令 + 输出 / 数据来源），复核方能沿出处走一遍。

## 2. 报告特有退出追加项（叠加契约「通用定稿退出模板」，不重写那四条）

- 支撑决策的断言全 A 级，或显式降级「未核实」；
- 反向核查跑过（每条高危结论找过反证并记录 找到了什么/没找到）；
- 第二视角对抗复核意见全处理（采纳，或给证据反驳）；碰钱/口径/权限档按契约 §6 升级为 L1；**单点小核查可否只走自查清单、免对抗面板 = 待 PM 拍**；
- 不确定项显式成节（没被删、没藏进模糊措辞）；关键数字全可复算；大白话齐。

## 3.「口径校准任务」——报告 Loop 的具名标准靶型

完整定义（定靶 = 口径升 A / 退出 = 升到 A 或「升不动 + 缺什么数据」/ 不新增文档）**见契约 §5，本文不复述**。用法：想把某个 B/C 级口径逼到 A，就用这个靶型开一份报告任务。

## 4. 护栏（叠加契约 §7）

防断言通胀（复核/对抗视角同守证据分级，挑刺也不许拿过期事实过度断言）/ 防体量崇拜（报告价值 = 断言质量非页数，长报告开头必有一页结论摘要）/ 防静默截断（只审一部分就说清边界「只看了 X 没看 Y」）/ 防结论漂移（修订轮不许悄悄改已交付结论，要改显式写「此前结论 X 有误，证据是 Y」）。

## 6. 给 PM 的大白话

管所有「给结论」的活（调研/审计/评估/复盘）：AI 说的每句要紧的话都得标明来路——亲自跑过验过的、从文档抄的、还是猜的；给你做决策用的必须是亲自验过的那种。写完自己当反方攻一遍、再让第二个 AI 专挑失真，不确定的白纸黑字列出、不许「应该没问题」糊弄。以前吃的亏（报告硬伤、把没上线功能当在用来审）就是这套规矩要堵的洞。
