# ADR-005 · P0 去留判定的稳健层 —— 样本量 + 结算 + 持续性 + 置信度 + 乐观上界

> **日期**：2026-07-04 | **状态**：已采纳（PM 拷问 Q7）| **性质**：hard-to-reverse（决定"停止候选/可留"这类经营裁决何时可发、带什么置信）。
> **关联**：P0 spec §5 状态真值表；ADR-001（关账/暂态）、ADR-004（率覆盖）；§3 可变人力未建模。

## 背景（PM Q7）

§5 原真值表从"数据质量闸（优先级1）"直接跳到"每例贡献毛利<0 → 停止候选（优先级3）"，中间缺三道闸 + 一层置信：
1. **无样本量下限**：某院当月仅 2 个 case、恰好高材料低收入 → 每例 CM 负 → 直接"停止候选"。**n=2 是噪声不是信号**。
2. **无结算门**：Q2/ADR-001 已定未关账月是暂态（补收/纠错会重算）。在会被重算的月上喊"砍" → 下月一笔补收进来它翻正 → **裁决自己会抖**。
3. **无持续性**：单月转负=停止候选，但真实去留该看**趋势**（连续/多数已结算月为负），非单月快照。
4. **置信度没挂到裁决**：覆盖率/缺价率/①\*占比/P0率覆盖占比现只 gate"需补数据"，没给"可留/停止候选"标置信 → `①*60%+覆盖85%` 勉强过闸的"可留"和 `①*5%+覆盖99%` 的"可留"**长得一样**。且 §3 可变人力未建模 → "可留"本身**偏乐观**（真实可避免成本 > P0 材料成本）。

## 决策

**在点估计 CM 之上加一层"稳健裁决"：强判定（停止候选/可留）须过全部闸 + 带置信 + 认 P0 为乐观上界。**

1. **样本量闸** `MIN_CASES_FOR_VERDICT`（默认 20）：判定窗口有效 case 数不足 → **观察中·样本不足**，出数不出去留判定。
2. **结算闸**：判定所依据的月未关账 → **暂态·待结算**，出临时贡献毛利、**不出强判定**（ADR-001 暂态）。
3. **持续性闸** `PERSIST_MONTHS`（默认 3）：**停止候选**须**连续/多数已结算月**每例 CM `< −CM_MARGIN_FOR_VARLABOR`；单月转负 → 只到"**需谈价/需关注**"。
4. **置信度挂裁决**：每个经营判定挂 `置信=f(覆盖率,缺价率,①*占比,P0率覆盖占比,样本量)`；看板显式"可留·高置信" vs "可留·低置信"，**同判定不同可信度必须可见**。
5. **P0 = 乐观上界**：可变人力（外包/加班/计件）是可避免成本但 P0 未建模 → **真实贡献毛利 ≤ P0 数**。故"可留"只发给 CM **明确 >0 且留出 `CM_MARGIN_FOR_VARLABOR` 余量**的院；**近零正的 CM = 符号不确定 → 需谈价，不判可留**。所有 P0 CM 标"乐观上界·未含可变人力"。

## 后果

- ✅ 不会因一个薄月/未结算月/单月快照喊"砍"——去留裁决稳、不抖。
- ✅ "可留/停止候选"带可信度，勉强过闸的乐观"可留"不再冒充高置信。
- ✅ 认清 P0 是上界 → 近零正不误发"可留"，与"可变人力未建模"诚实对齐。
- ⚠️ 保守代价：小院/新院/薄月常停在"观察中/需谈价"，强判定发得少——但去留是 hard-to-reverse 动作，宁慢勿错。
- ⚠️ `CM_MARGIN_FOR_VARLABOR`（可变人力余量）需一个保守估或校准值——在有可变人力数据前，取行业可变人力占比保守设，标"待校准"。

## 备选（未采纳）
- 单月 CM<0 直接停止候选：否（Q7 病根，噪声/暂态/单月/无置信四重脆弱）。
- 用统计显著性检验替代样本量闸：过度工程；P0 阶段用简单 case 数下限 + 持续月数即可。
