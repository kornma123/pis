## 开发顺序建议

1. 先做 `PRD-0` 红测试和迁移设计，第一步先跑 ABC/LIS `case_no` 跨院审计。
2. 并行细化 `PRD-1A` schema 与验收测试，本 v1.1 已固定赣州行级归属月、东安 seed 分类、P&L 并表 TODO。
3. `PRD-0` 绿后，进入 `PRD-1A` 的 schema 和 parser normalization。
4. 不再为路线图/PRD 做多轮纯文档互评；后续评审集中在实现 PR、测试红绿和迁移报告。

## 阶段闸门

- Phase 0 不依赖 Phase 1A，必须先保证现有数据链路不再污染。
- Phase 1A 只证明最小对账闭环，不承诺完整院级 P&L 关账。
- Phase 1A 若三样本任一无法无损落 `statement_normalized_lines`，先修 schema，不扩模板范围。

## v1.1 吸收项

- 赣州行级归属月默认使用 `report_date`，并在规范行记录 `settlement_month_basis`。
- 东安 `category_summary` 写入固定 seed 规则：HPV/基因/FISH 归 OUT，常规/IHC/EBER/特殊染色/冰冻/P16 归 IN。
- EBER、特殊染色归 IN 作为本版默认领域判断，开发前由业务确认；如被否定，仅调整规则 seed 和验收锚点，不改 schema。
- `partner_month_revenue_ledger` / `out_settlement_ledger` 与现有 `case_revenue` / `partner-pnl` 看板的并表放在 Phase 1A 之后，不在本阶段制造双口径假闭环。
- Schema 补入 normalized line 幂等唯一键、派生账本 `UNIQUE(source_line_id)` 守卫、`declared_total_scope`、`quality_flags(partner_id, settlement_month)` 工作台索引。
- 验收测试补入 `duplicate_file` 去重、同 batch 重派生幂等、东安 detail 合计闭合和 FISH=0 行口径。
