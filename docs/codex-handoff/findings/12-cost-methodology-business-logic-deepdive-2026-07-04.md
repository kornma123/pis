# Codex 深审 12：成本口径业务逻辑结论

> 日期：2026-07-04
> 分支：`claude/cost-methodology-deepreview`
> 目的：在 11 号技术复核之外，继续从经营决策逻辑深挖“标准成本 + 贡献毛利”到底能不能指导医院去留、谈价、限量、运营纠偏。

## 一句话结论

COREONE 当前真正需要的不是一个“每张切片到底真实花了多少钱”的数，而是一套能支撑经营动作的成本决策系统：

```text
已确认实收
- 与本 case 工序集匹配的可避免标准变动成本
= 贡献毛利下限

再叠加：产能瓶颈、合作形态、缺价暴露、数据完整度、战略约束
=> 决定续、谈、限、停、纠偏
```

所以，方法论大方向成立：单片实际成本追不到，固定成本不能摊下去砍院，应用标准成本和贡献毛利。但这套方法只有在“业务动作分层”后才安全。一个统一的“医院利润”数字会误导，因为不同动作需要扣不同成本。

## 1. 先定业务动作，再定成本口径

同一个医院毛利数，放在不同决策里含义完全不同。建议把看板上的动作分成 6 类，每类只允许使用对应口径。

| 业务动作 | 要回答的问题 | 能用的成本 | 不能用的成本 | 判断逻辑 |
|---|---|---|---|---|
| 接不接这类 case | 多做这一例是不是比不做好 | 可避免变动成本：一抗、二抗、显色、特染盒、外购服务、按件/加班等 | 在编人力、折旧、房租、历史沉没成本 | 贡献毛利 > 0 是下限，通过后再看产能 |
| 医院续约/谈价 | 当前合同价能不能覆盖这类服务范围 | 合作形态裁剪后的标准变动成本 + 必要的阶梯/批次成本 | 实验室整体固定成本逐院硬摊 | 低 CM 或高缺价暴露，优先谈价或改服务范围 |
| 限量/排队 | 产能满时谁优先做 | 每瓶颈分钟贡献毛利 | 单纯每例 CM 或总额 | CM/瓶颈分钟低者限量，不一定停院 |
| 运营纠偏 | 浪费、返工、深切、复染是否异常 | 标准成本 vs 期末实际差异 | 把差异直接归罪给医院 | 差异是异常信号，先查 SOP、BOM、操作和盘点 |
| 关账/财务核对 | 本月总账能否闭合 | 期末盘点/财务实付/实际差异回摊 | 单片“真实耗用”叙事 | 可做总额勾稽，不证明某片真实用量 |
| 投资/撤线 | 设备、人力、房租是否值得保留 | 长周期全成本、产能利用、市场战略 | 单 case 贡献毛利 | 这是季度/年度管理会问题，不是 case 毛利问题 |

最危险的误用是：把“标准全成本毛利”或 “ABC 毛利”拿去做接单/砍院决策。固定成本在短期停做后并不会消失，这会把高量高贡献医院误判成亏损源。

## 2. 成本可避免性随时间尺度变化

同一项成本在不同时间尺度下性质不同。文档里“固定成本不下沉”是对短期去留决策的保护，但不是说固定成本永远不用管理。

| 时间尺度 | 典型动作 | 成本行为 |
|---|---|---|
| 当天/本周 | 接单、加做、排班 | 只看材料、外购、加班、可取消耗材等真正增量 |
| 月度 | 院级运营监测、补收、差异排查 | 可看标准成本 + 期末实际差异，但差异是监管信号 |
| 合同周期 | 谈价、调服务范围、最低量承诺 | 可纳入阶梯人力、设备占用、质控批次、服务成本 |
| 年度 | 关线、扩线、买设备、裁撤人员 | 才能看全成本、折旧、房租、固定人力和战略收益 |

因此建议字段不要只有 `costTotal`。至少拆成：

- `avoidableVariableCost`：贡献毛利用。
- `batchOrStepCost`：批次/工序阶梯成本，需按场景判断是否可避免。
- `allocatedFixedCost`：固定/间接分摊，只做全成本经营分析。
- `actualVarianceAdjustment`：期末实际与标准差异，只做监管和总账勾稽。

## 3. 合作形态是成本口径的第一前置

业务逻辑上，医院不是一种客户，而是多种合作形态。不同形态下，同一个“免疫组化片”该扣的成本集合不同。

| 合作形态 | 收入可能覆盖 | 成本应扣 | 最大误判 |
|---|---|---|---|
| 全流程技术 | 取材/处理/包埋/切片/HE/染色 | 全流程技术耗材 + 染色耗材 | 若漏掉组织处理成本，会虚高毛利 |
| 只代送加做染色 | 染色技术费，通常不含前处理 | 一抗、二抗、显色、载玻片/必要切片，通常不扣本院组织处理 | 套全流程 BOM 会冤枉代送院 |
| 技术 + 诊断 | 技术收入 + 诊断/报告收入 | 实验室技术成本；医生/报告另桶 | 把诊断收入计入实验室毛利或把医生成本混入 |
| 现场服务 | 现场人工/服务费 | 服务成本桶，不是实验室制片/染色成本 | 误放进实验室工序毛利 |
| 外送/转销/NGS | 转销收入或分成 | 外包协议成本，已核才进毛利 | 按 0 成本算高毛利，或误认为分子已建模 |
| 共建/分成 | 净额或分成结算 | 需按合同定义责任边界 | 把对方承担的工序成本算到本中心 |

业务结论：**case 工序集比医院主数据更重要**。一个医院可能同时有全流程、代送加做、外送转销。只在 partner 上设 `technical_only/with_diagnosis` 不够，必须能在 case 层覆盖。

最低可用规则：

```text
每个 case 必须有 service_step_scope。
service_step_scope 来源 = contract 默认 / bill 账单码 / LIS 工序 / manual 人工。
低置信或缺失时，不出“精确贡献毛利”，只出待确认。
```

## 4. 贡献毛利为正不等于应该保留

贡献毛利只是“做比不做强”的下限，不是最终经营结论。至少有 4 类反例：

1. **产能瓶颈**：染色机、技师、病理流程满负荷时，应比较 `贡献毛利 / 瓶颈分钟`，不是每例毛利。低价高耗时医院可能 CM 为正但应限量。
2. **服务范围错配**：代送院如果不承担前处理收入，也不应扣前处理成本。错配后会把应保留的轻服务院判成亏损。
3. **缺价集中**：分子、PD-1、TROP-2 等缺价如果集中在某院，CM 会虚高。缺价率高时不能做续约/砍院，只能做“待补价”。
4. **战略/渠道价值**：某院低 CM 但带来病例量、市场入口、专家关系或覆盖固定排班，不能只用单期 CM 判停。

建议看板动作不要只有“盈利/亏损”。应使用状态：

- `可保留`：CM 正、数据完整、无产能冲突。
- `需谈价`：CM 正但低于目标，或高价抗体/复杂工序占比高。
- `需限量`：CM 正但瓶颈分钟效率低。
- `需补数据`：缺价、工序集、收入匹配或 LIS 覆盖不足。
- `需运营排查`：标准与实际差异异常。
- `可考虑停止`：确认收入、成本、工序集、缺价、产能和战略约束后，CM 仍长期为负。

## 5. 标准成本的商业用途不是“证明真实”，而是“形成可谈判的下限”

标准成本最有价值的商业场景：

1. **报价底线**：知道每项服务按 SOP 最低要覆盖多少变动成本。
2. **医院谈价**：告诉医院“贵不是因为平均成本，而是你们 case mix 中高价抗体/复杂工序占比高”。
3. **服务范围重定**：如果代送只付染色费，就不要默认提供/承担前处理。
4. **异常发现**：实际耗用长期高于标准，说明 BOM、收得率、复染、浪费或盘点有问题。
5. **跨院比较**：比较同工序同 case mix 下的贡献差异，而不是拿总毛利排名。

标准成本最不该做的事：

- 不该拿去和财务实付逐 case 对账。
- 不该声称“这张片真实花了这么多”。
- 不该在缺价/工序集未知时给出无条件排名。
- 不该把固定成本均摊后当砍院依据。

## 6. 期末实际差异回摊可以做，但要降级为监管口径

历史文档里有“期末实际 = 期初 + 入库 - 期末盘点，差异按标准份额回摊到 case/医院”。这个思路能解决一个财务问题：让 `Σ医院 = 实验室实际总额`。

但从业务逻辑上，它不是“每个 case 的真实消耗”。它是期间总差异的会计分配。正确定位：

- 可用于月度总账勾稽。
- 可用于看哪类物料/工序差异异常。
- 可用于提醒 BOM/收得率/复染率需要校准。
- 不应用于逐 case 接单决策。
- 不应把差异直接归责给医院，除非能证明该院特定行为导致差异。

建议展示为：

```text
标准贡献毛利：用于日常经营决策
期末调整后毛利：用于月度经营复盘和总账勾稽
差异率：用于运营异常排查
```

## 7. 分子线和外购转销要拆开说

“分子线未建模”这句话需要更精确：

- **院内 FISH/NGS 标准成本未建模**：探针、反应、文库、flowcell、pooling 决定成本，不能用 IHC 的“配方 × 染色片数”。
- **外购转销 NGS 可以算毛利**：如果有售价和外包成本，就可以按订单算 `sell_price - outsource_cost`，但未核成本单必须单列，不进毛利。

业务上建议：

- 院级看板拆出 `inHouseLabMargin` 和 `outsourcedNgsMargin`。
- 院内分子/FISH 在未建模期间显示 `未建模收入/数量`，不要静默当 0 成本或混入 IHC。
- 做分子的大院不能因为“成本未建模”被默认看成高毛利，也不能因为 IHC 主线成本低就误判。

## 8. 业务规则清单：实现前必须拍板

这些不是代码细节，是经营口径。如果不先定，实现会反复返工。

1. **贡献毛利目标线**：不同业务线/合作形态最低 CM 率或每例 CM 是多少。
2. **瓶颈资源**：IHC 染色机、技师、切片、报告哪个是真瓶颈，是否需要 `CM / bottleneck minute`。
3. **服务范围默认**：每家医院默认做哪些步骤，case 层如何覆盖。
4. **缺价阈值**：缺价金额或缺价片数超过多少时，不允许出精确排名。
5. **分子/FISH 标注**：未建模收入/数量如何展示，是否影响院级总毛利。
6. **期末差异回摊用途**：是否只做复盘和勾稽，禁止用于日常接单。
7. **关账冻结策略**：抗体价、收得率、BOM、工序集修正后是否重算历史，还是记次月调整。
8. **战略豁免**：哪些医院可因渠道、专家、区域战略保留，即使短期 CM 低。

## 9. 推荐的业务看板结构

不要只给“收入、成本、利润”。建议每家医院每月展示 5 层：

### A. 收入事实

- 已确认实收。
- 技术收入、诊断桶、外送/转销。
- 实收来源：对账单 / 估算 / 人工修正。

### B. 服务范围

- 全流程、代送加做、外购转销、现场服务。
- 工序集来源和置信度。
- 未确认工序集 case 数。

### C. 标准贡献毛利

- 可避免标准变动成本。
- 贡献毛利、CM 率、每例 CM。
- 高价抗体/特染/外购成本贡献。

### D. 经营约束

- 瓶颈分钟或估计产能占用。
- `CM / 瓶颈分钟`。
- 复染、深切、返工、补收、超期免费。

### E. 可信度和动作

- 缺价率、缺价金额暴露。
- LIS 匹配率、对账覆盖率。
- 标准 vs 期末实际差异率。
- 推荐动作：保留 / 谈价 / 限量 / 补数据 / 运营排查 / 停止候选。

## 10. 最终裁决

我对当前方法论的业务裁决：

1. **成立**：用标准成本替代单片实际成本，用贡献毛利替代全成本分摊，方向正确。
2. **必须加护栏**：贡献毛利、标准全成本、ABC 毛利、财务实际成本要分口径，不允许一个“利润”数通吃。
3. **第一落地点不是抗体换价源，而是 case 工序集**：没有工序集，成本与收入不可能同源。
4. **先经营决策后会计闭环**：接单/谈价用标准贡献毛利，月度复盘再看实际差异回摊。
5. **看板要输出动作和置信度**：成本数只是输入，真正的产品价值是告诉 PM 下一步该谈价、限量、补价、排查还是停止。

如果只允许做一个下一步，我建议先做：

```text
成本指标 ADR + case 工序集 resolver + 贡献毛利/标准全成本双 lane 输出
```

这三件事能防止方法论在实现层跑偏，比立即改 `MAT-IHC-*` 价源更优先。
