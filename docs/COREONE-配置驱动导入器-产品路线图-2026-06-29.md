## 0. 一句话判断
**已证明「规整逐项明细」能端到端跑通（和睦家黄金 ¥13,152），但离财务每月真正用起来，差一个「月结产品」闭环。** 产品本质 = **病理实验室月结 sub-ledger**（收入·外送·成本·调整·关账闭环），导入只是入口。

## 1. 北极星 + 成功标准
**北极星**：财务能在一个月结周期内完成所有医院结算单导入、分类确认、异常处理、院级盈亏生成与审计留痕，**不再依赖线下 Excel 当主账**。
**成功标准**：① 覆盖真实 7 模板家族，每类有明确「入库/登记/转外送台账」路径；② 每月每院每文件有状态；③ 黄金 ¥13,152 不回退；④ 纯外送/远程/共建/类别汇总不被「无法逐 case 落库」阻断；⑤ 所有金额可解释为 IN 劳务 / OUT 外送·合作 / 调整 / 未匹配 / 成本未就绪；⑥ 改规则能展示影响范围并重算或出调整单，不静默改写历史。

## 2. 产品原则
- 不强行把所有结算单塞进病例级模型（逐病例/科目汇总/远程/共建/纯外送 并存）。
- 原始文件不可变、解析可重算、人工判断可覆盖但留痕。
- **OUT 不是实验室劳务收入，但仍是月结事实，必须进财务闭环。**
- **🌟「0」只能代表明确业务值；缺失（缺成本/缺规则/缺 LIS/未归类）必须以质量标记暴露，不能按 0 处理。** 外推：未匹配 LIS ≠ 没有病例；缺成本 ≠ 毛利=收入；纯 OUT ≠ 金额消失；未归类 ≠ 应收=0。
- 配置驱动 = 用样表/推荐规则/一键确认/影响预估降低配置成本，不是让财务填配置表。

### 2.1 已评估并搁置的方向（记录，避免重复讨论）
- **「LIS 生产驱动 / 双向对账」翻转**（以 LIS×收费目录算实验室收入、账单降级为对账）：经核实**真实 LIS 较薄（多半只有病例登记、生产数量不全）**，生产驱动收入算不准 → **对账单仍为主源，暂不采纳双引擎**。LIS 的角色保持为「病例匹配 + 完整度校验 + 成本归属」，不作收入主脊。后续若 LIS 数量列变厚，可重新评估。

## 3. 用户角色
- **财务月结负责人**：汇总各院、确认应收/扣率/外送分摊/院级盈亏。诉求：少返工、可对账、可解释。
- **运营/实施人员**：新院上线、模板识别、规则维护、异常处理。诉求：样本训练规则、知道规则是否影响历史。
- **管理层/经营分析**：看院级收入/成本/毛利/外送占比。诉求：数据可信、可追溯到来源。

## 4. ✅ 已锁定决策
1. **病例身份 = 全链路复合键 `(partner_id, case_no)`**（医院各自编号可撞号；LIS 自然键/导入/人工覆盖/ABC 回填/P&L join/成本 rollup 全升级）。
2. **04 修复时机 = 先修「正确性子集」**（跨院串账全链路 + 配置归一读/回滚 + NGS 缺成本质量标记）；v0 死锁随 Epic F、NGS 读 RBAC 随 Epic C。
3. **联合运营/共建 = 逐院配置 `special.joint` 决定**是否进院级 P&L 主表（默认 OUT 单列，逐院可覆盖）。
4. **重算 = 已结账只出调整单 / 未结账可重算**（新快照，不改已上报数）。
5. **数据主源 = 对账单**（LIS 薄，不作生产驱动主脊；见 §2.1）。

## 5. 数据模型（两层，采自 codex v2 优化）
**不要一上来把业务事实分裂成多张最终表**；先建统一规范行承接中间态，再派生账本。

**第一层 — 不可变导入 + 规范行**
- `statement_import_batches`：批次（医院/月份/模板/原始文件哈希/解析器版本/配置版本）= 追溯·重算·审计骨干。
- `statement_raw_rows`：原始网格行（不可变留痕）。
- **`statement_normalized_lines`（核心）**：系统理解后的对账单金额行，承接宽表/科目汇总/纯外送的全部中间态。字段：`batch_id / partner_id / service_month / case_no / source_sheet / source_row / source_column / source_label / template_family / line_grain(case|aggregate|out|joint|adjustment) / business_line(IN|OUT|UNKNOWN) / service_code / amount_role(gross|settlement|discount|cost|adjustment) / amount / classification_status / rule_version / quality_state / **raw_payload**(原始上下文长尾——避免一次把字段设计满，长尾信息塞这里，控制 schema 过度设计风险)`。

**第二层 — 派生账本与快照（从规范行派生，可重算）**
- `case_revenue_ledger`（自然键 `(partner_id, case_no, service_month)`，决策①）/ `partner_month_revenue_ledger`（聚合）/ `out_settlement_ledger` / `joint_operation_ledger` / `quality_flags` / `pnl_snapshots` / `adjustment_entries`。
- 好处：宽表/科目/纯外送同层承载、不丢原始上下文；规则重算 = **重新派生账本**而非改原始理解层；新模板先落规范行、再决定能否进各账本。

## 6. 月结状态机（采自 codex；绑定两维度，工作台围绕"还差什么不能关账"）
**批次态**（一份文件走到哪）：
| 状态 | 含义 | 下一步 |
|---|---|---|
| `uploaded` | 文件已上传 | 解析 |
| `parsed` | 已解析为 raw_rows / normalized_lines | 分类 |
| `classification_required` | 有待归类行 | 人工确认或建规则 |
| `review_required` | 可入账但需复核 | 财务复核 |
| `posted` | 已派生到账本 | 等成本或关账 |
| `cost_pending` | 已入账但成本未就绪 | 补成本或标记待补 |
| `ready_to_close` | 无阻断项 | 关账 |
| `closed` | 已关账 | 只能调整 |
| `adjusted` | 已生成调整单 | 追踪调整 |

**医院月份态**（这院这月能不能关账）：`not_started / in_progress / blocked / reviewing / ready_to_close / closed / needs_adjustment`。
验收：财务能在月份视图看到每家医院**为什么不能关账**；每个阻断态能跳到具体批次/行/处理动作；状态变更有操作人/时间/原因。

## 7. 质量标记模型（采自 codex；升级为可行动工作项，不是告警文案）
每条 `quality_flags` 至少含：`flag_type`（缺LIS/缺成本/缺价/缺规则/未归类/疑似重复/对账不平/人工覆盖/重算影响）、`severity`（blocking/warning/info）、`owner_role`（财务/实施/成本/管理员）、`resolution_action`（补LIS/补成本/建规则/人工确认/忽略并说明）、`blocks_posting`、`blocks_closing`、`related_line_id`、`related_batch_id`、`resolved_by/at/note`。工作台据此回答：是否影响入账？是否影响关账？谁该处理？

### 7.1 默认阻断矩阵（采自 codex v3.1；落地按此，逐院可调）
| flag_type | 阻断入账 | 阻断关账 | 说明 |
|---|---|---|---|
| 缺规则（未归类） | ✅ | ✅ | 不知道算 IN/OUT，不能入账 |
| 缺价格 | ✅ | ✅ | 算不出金额 |
| 缺成本 | ❌ | ⚠️ | 收入可入账；毛利关账阻断或 P&L 单列「成本未就绪」 |
| 缺 LIS | ❌ | ⚠️ | 收入可暂入账；关账前需补或财务带原因放行 |
| 对账不平 | ❌(需 confirm) | ⚠️ | 差异即信号，确认带原因后可继续 |
| 纯 OUT 无病例号 | ❌ | ❌ | 正常，进 `out_settlement_ledger` |
| 疑似重复 | ✅ | ✅ | 防重复入账 |

## 10. ⚠️ 异常与边界全清单（v3 新增重点 —— 每条都要在设计/验收里有明确处置）
> 目的：少漏细节。按层组织，标注【处置原则】。落地时每条对应一个 `quality_flag` 或一条解析/校验规则。

### 10.1 文件 / 批次层
- 重复上传同一文件（按文件哈希去重，提示「已导入，是否覆盖/忽略」）。
- 同院同月多文件（分批/补单）→ 同 `service_month` 多 batch，金额累加且各自留痕，不互相覆盖。
- 损坏/空/加密 xlsx、多 sheet（选错 sheet）、非 Excel（pdf 扫描件）→ 明确报错 + 引导。
- 模板识别置信度低 / 误判 → 显示「匹配模板 X，置信度 N%」+ 允许人工改模板重解析。
- 列错位 / 合并单元格 / 跨页重复表头 / 中间小计行混入 / 表头多行换行。
- 红冲行 / 负数金额 / 退费行 / 手工调整行 → 识别为 `adjustment`，不当正常收入。
- 单位/币种异常、千分位、全角数字、空单元格 defval。

### 10.2 病例层
- 跨院同号（决策①复合键）；全角/大小写（NFKC）；病理号格式变体（前缀/分隔符差异）。
- 一个病理号多行（多服务项）→ 同 case 聚合；无病理号行（纯外送/汇总）→ 走聚合/台账，不报「缺病理号失败」。
- LIS 未到 / 晚到 / 无登记月 / 错院（医院名别名）→ 病例匹配可恢复态（H），不阻断收入入库（标记成本/病例待补）。
- 退检 / 作废 / 重做 / 补做 case；跨月 case（本月做、上月或下月结算）→ 归属规则需显式（按结算月 or 登记月）。

### 10.3 金额 / 口径层
- 扣率缺失 / 异常（"90%" 坑——保存与**读/回滚路径**都归一，§11.2）；结算列优先 > 开单×行扣率 > 开单×配置三级扣率。
- `declaredTotal` 缺失（无独立合计行→需 confirm）/ 被逐行 Σ 污染（必须取独立声明合计）。
- 含税 / 未税口径（神州含税 6%）；小计/合计/签名行不计入逐行。
- 分配率（温州）；月度固定保底费 retainer（如苍南 ¥13000，按月一次性、非逐 case）；共建分成净额（逐院配置）。
- 四舍五入累积误差（统一 round 口径）；0 金额行；对账不平（Σ结算 vs declaredTotal 差）→ 差异即信号，分级呈现。

### 10.4 分类层
- 未匹配 / 歧义（多命中→人工归类，不静默取首条）；宽表列语义（B）。
- 新院无配置 v0（建档向导首写，§11 / Epic F）；同前缀冲突（同长前缀→歧义）。
- 常见 OUT 漏配（HPV-E6E7/FISH/NGS/远程 预置目录，G）；IN/OUT 边界模糊（同服务有时自做有时外送）→ 需 case/协议级覆盖。
- 规则改了影响历史（E：影响预估 + 重算/调整单）。

### 10.5 成本层
- ABC 成本未接通 ≠ ¥0（显式「未接通」，不参与毛利率或显著标注）；成本晚于收入到达 → 回填触发重算（未关账）或调整单（已关账）。
- `cost_status`（pending_cost/cost_exception/voided）排除；case 级成本下钻按复合键不串院；NGS 外购成本缺失 → 质量标记，不按 0。

### 10.6 关账 / 重算层
- 已关账改规则（只出调整单）/ 未关账（可重算）；关账后补单 → 进 `adjustment_entries`。
- 调整单审计（原因/操作人/时间/前后差异/审批）；基线生效期（从哪个账期起生效）；追溯重算范围（哪些 batch/月受影响）。

### 10.7 权限 / 治理层
- 财务 vs 管理员 vs 运营实施权限边界；成本可见性白名单；NGS 利润/协议价可见性（C：读路由收窄）。
- 确认入库（confirm）必须带原因 + 影响摘要 + 入库后保留「待复核」态（审计）。
- maker-checker（大额规则变更 / 未识别确认入库 / OUT 改 IN 需复核，Phase 2）。

### 10.8 看板 / 诚实呈现层
- 估算 / 缺 LIS / 缺成本 / 未归类 各自显式标注 + **drill-down + 责任人**（点开 case 清单 / 跳成本回填 / 跳测试台批次 / 跳 NGS 补成本）。
- 毛利率分子分母口径一致（待核算病例不进分子稀释）；多院区两级聚合（和睦家：主院/新城/新睦妇儿）。

### 10.9 跨系统 / 数据治理层
- LIS 导入 ↔ 对账单 ↔ ABC 成本 三方时序错位（任一晚到都不阻断，完整度标注）。
- 医院名别名映射库（账单医院名 ≠ LIS partner → 以 LIS 为权威 + mismatch 预警）。
- 收费目录版本漂移（GOV-1：账单实收 vs 引擎理论金额差异报表；目录单一权威源 + 版本/生效日期/变更人）。

## 11. Phase 0 — 正确性修复（按决策②，先做，不依赖产品 Epic）
1. **跨院串账全链路（决策①重方案）**：`lis_cases` 自然键→`(partner_id, case_no)`；LIS 导入 ON CONFLICT、人工覆盖、ABC 回填、`partner-pnl-service` 的 LIS join、`getCaseCostRollup` 全带 partner 维度。回归覆盖 `buildPartnerPnl`/`loadCasePnlsWithCost`/`backfillAbcPartnerIds`，黄金不回退。
2. **配置归一覆盖读/回滚路径**：`row2config`/loadConfig/peekConfig/rollbackConfig best-effort normalize，治历史坏扣率。
3. **NGS 缺成本/售价质量标记**：缺值不再 confirm 后按 0 静默入账 → 写质量标记，P&L 默认排除/单列，响应返回 `missingPriceCount`。

## 12. 分阶段（采 codex 纵向切片）
- **Phase 0（1–2 周）**：§11 正确性止血 + 守黄金。
- **Phase 1A — 最小可关账月（核心）**：规范行 `statement_normalized_lines` + 月结状态机基础 + **3 样本端到端**（东安科目汇总 / 赣州纯外送 / 平泉宽表）+ 最小月结差异包。目标=用 3 样本**关一个小账**，不求 7 模板全覆盖、不做复杂 maker-checker。
- **Phase 1B — 模板与规则扩展**：温州/宁波/石门 + 默认目录（G）+ 模板化预览 + confirm 原因/影响/审计。
- **Phase 2 — 治理与关账制度**：规则影响预估 + 重算（决策④）+ 黄金值产品化（F）+ LIS 匹配恢复（H）+ maker-checker + 月结导出包。

## 13. 验收标准（功能/数据/体验 + 异常驱动）
- **功能**：同月可并行导入 5 类样本并在工作台见统一状态；赣州纯外送→labRevenue=0 不阻断；东安科目→IN/OUT 聚合层；平泉宽表金额可追溯来源列、列级规则下月复用；§10 每类异常有明确处置（去重/红冲/晚到/别名/未匹配…）；人工确认必填原因且审计可查。
- **数据**：黄金 ¥13,152 不变；每批次可追溯文件哈希/模板版本/规则版本/操作；月结金额可拆为 IN/OUT/调整/未匹配/成本未就绪；规则变更不静默改写已结账月。
- **体验**：「待人工归类」必给候选规则+影响金额+下一步；科目/纯外送/共建有结构化预览表；月结负责人无需多隐藏入口完成导入→异常→确认→导出。

### 13.1 关键验收场景（Given/When/Then，采自 codex v3.1）
- **东安科目汇总**：Given 上传东安科目汇总 → When 解析 → Then 每科目金额进 normalized_lines 并映射 IN/OUT/调整/待确认，已确认科目派生到 `partner_month_revenue_ledger`。
- **赣州纯外送**：Given 无病例号纯外送 → When 识别为纯 OUT → Then 不因缺 case_no 失败，派生到 `out_settlement_ledger`（labRevenue=0/outSettle=全额）。
- **平泉宽表**：Given 上传宽表 → When 解析金额列 → Then 每金额行保留 sourceColumn/sourceLabel，财务可按列确认「远程会诊结算→OUT」。
- **缺成本**：Given NGS/OUT 缺成本 → When 入账/出 P&L → Then 不按 0 成本算正常毛利，生成质量标记，按矩阵决定是否阻断关账。
- **规则变更**：Given 财务改 HPV-E6E7 OUT 规则 → When 保存前 → Then 展示受影响院/月/批次/金额，已关账月只生成调整单。
