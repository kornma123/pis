## 0. 一句话判断
**已证明「规整逐项明细」能端到端跑通（和睦家黄金 ¥13,152），但离财务每月真正用起来，差一个「月结产品」闭环。**
本版已并入数据模型骨架与 4 项决策，可据此排期。

## 1. 北极星 + 成功标准（采自 codex）
**北极星**：财务能在一个月结周期内完成所有医院结算单导入、分类确认、异常处理、院级盈亏生成与审计留痕，**不再依赖线下 Excel 当主账**。
**首版成功标准**：① 覆盖真实 7 模板家族，每类有明确「入库/登记/转外送台账」路径；② 每月每院每文件有状态（未导入/待分类/待复核/已入库/成本未就绪/已出盈亏）；③ 黄金 ¥13,152 不回退；④ 纯外送/远程/共建/类别汇总不再被「无法逐 case 落库」阻断；⑤ 所有差异可解释为 院内劳务收入 / 外送·合作结算 / 未匹配 LIS / 成本未就绪 / 人工调整 / 规则变更影响；⑥ 改规则能展示影响范围并重算或出调整单，不静默改写历史。

## 2. 产品原则
- 不强行把所有结算单塞进病例级模型（逐病例 / 科目汇总 / 远程 / 共建 / 纯外送 并存）。
- 原始文件不可变、解析可重算、人工判断可覆盖但留痕（增量纠错架构）。
- **OUT 不是实验室劳务收入，但仍是月结事实，必须进财务闭环。**
- **🌟「0」只能代表明确业务值；缺失（缺成本/缺规则/缺 LIS）必须以质量标记暴露，不能按 0 处理。**（采自 codex，统一了 NGS 缺成本 + 诚实信号）
- 配置驱动 = 用样表/推荐规则/一键确认/影响预估降低配置成本，不是让财务填配置表。

## 3. 用户角色
- **财务月结负责人**：每月汇总各院、确认应收/扣率/外送分摊/院级盈亏。诉求：少返工、可对账、可解释。
- **运营/实施人员**（采自 codex，原 v1 漏）：新医院上线、模板识别、规则维护、异常处理。诉求：用样本训练规则、知道规则是否影响历史。
- **管理层/经营分析**：看院级收入/成本/毛利/外送占比。诉求：数据可信、可追溯到结算单来源。

## 4. ✅ 已锁定决策（2026-06-29，本版据此设计）
1. **病例身份 = 全链路复合键 `(partner_id, case_no)`**。医院各自编号、可能撞号 → LIS 自然键 / LIS 导入 upsert / 人工覆盖 / ABC 回填 / P&L join / 成本 rollup **全部**升级为该口径（即架构「重方案」）。
2. **04 修复时机 = 先修「正确性子集」**（与产品方向无关、现在就污染数据的）：跨院串账全链路 + 配置归一读/回滚路径 + NGS 缺成本/售价质量标记。其余（v0 死锁、NGS 读 RBAC）随对应 Epic（F、C）一起做。
3. **联合运营/共建（石门类）= 逐院配置 `special.joint` 决定**是否进院级 P&L 主表（默认按「共建分成净额 OUT、不进实验室工序毛利、单列展示」，逐院可覆盖）。
4. **重算策略 = 已结账（关账）月份只出调整单；未结账月份可一键重算**。重算生成新快照，保留旧结果，不静默改写已上报数字。

## 5. 数据模型 / 事实层（采自 codex，按决策①调整）
把「导入文件」拆成多个事实层，避免所有模板都依赖 `case_no`：
- **`statement_import_batches`**：文件批次（医院 / 月份 / 模板 / 原始文件哈希 / 解析器版本 / 配置版本）= 追溯·重算·审计骨干。
- **`statement_case_revenue`**：逐病例收入事实，承接现有 `case_revenue`/`case_revenue_lines`。**自然键升级为 `(partner_id, case_no, service_month)`（决策①）**。
- **`statement_aggregate_revenue`**：非逐病例收入事实（科目汇总 / 纯外送 / 远程会诊 / 联合运营汇总），partner-month 粒度。
- **`statement_out_ledger`**：外送/合作结算台账（NGS / FISH / HPV-E6E7 / 远程 / 共建），不计入实验室劳务收入但进月结。
- **`statement_quality_flags`**：缺成本 / 缺规则 / 缺 LIS / 疑似重复 / 人工覆盖 / 历史重算 等质量标记（落实原则「缺失≠0」）。

## 6. 史诗（Epic）与优先级

| # | Epic | 解决什么 | 档位 | 影响采纳 | 工作量 |
|---|------|---------|------|---------|--------|
| A | 月结事实层（全模板可入库） | 类别汇总/纯外送/共建 进盈亏，院不消失 | 阻断采纳 | 🔴极高 | 中–大 |
| B | 宽表按列归类 | 远程/诊断服务费可操作（sourceColumn/Label） | 阻断采纳 | 🔴高 | 中 |
| C | OUT 一等公民 + 台账 | 转销/远程/共建闭环 + NGS 入口 + 成本完整度 | 阻断采纳 | 🔴高 | 中–大 |
| D | 月结工作台 | 月份×医院 文件状态总控 + 差异报表 | 重要缺口 | 🟠高 | 大 |
| E | 规则治理 + 历史重算 | 改规则敢用（影响预估+重算+生效期+调整单） | 重要缺口 | 🟠中–高 | 中–大 |
| F | 样表驱动建档向导 + 信任锚 | 首用顺 + 黄金值/体检卡进真实流程 | 重要缺口 | 🟠中–高 | 中 |
| G | 默认服务目录升级 | 预置 HPV/E6E7/FISH/NGS/远程，少手工补 | 重要缺口 | 🟡中 | 小 |
| H | 病例匹配可恢复 | LIS 晚到/错院/别名 的解决路径 | 重要缺口 | 🟡中 | 中 |

**Epic 要点**（详细问题/素材证据见 `findings/05-product.md`）：
- **A**：`category_summary` 类别→IN/OUT 映射写 `statement_aggregate_revenue`；纯 OUT 账单入库为 `labRevenue=0/outSettle=全额` 进 `statement_out_ledger`；`joint_venture` 按决策③逐院配置定是否进 P&L 主表。
- **B**：解析宽表展开每个金额来源列为带 `sourceColumn/sourceLabel` 的明细，测试台**按列归类**+候选规则（远程会诊结算→远程 OUT / 免组结算→IHC IN），财务确认写回配置。
- **C**：月度工作台分「实验室收入 / 外购转销·远程·共建（移出）/ 未归属」；NGS 前端导入入口 + 缺成本质量标记 + 完整度评分 + 公司级交接状态。
- **E**：保存配置前按历史 import_batch dry-run 列受影响月份/院/金额差；**已结账出调整单、未结账重算（决策④）**；基线绑定「从哪个账期起生效」。
- **F**：新医院建档向导（上传样表→识别→IN/OUT 候选→设扣率/特殊结算→跑黄金/体检卡→设首基线）；体检卡加「期望实收」录入（修 v0 死锁顺带在此重写首用流程）。

## 8. ⚡ Phase 0 — 正确性修复（按决策②，先做；不依赖产品 Epic）
1. **跨院串账全链路（决策①重方案）**：`lis_cases` 自然键 → `(partner_id, case_no)`；LIS 导入 `ON CONFLICT(partner_id, case_no)`；人工覆盖、ABC 回填（`abc-partner-link`）、`partner-pnl-service` 的 `JOIN lis_cases ON case_no`、`getCaseCostRollup` 全部带 partner 维度。回归测试覆盖 `buildPartnerPnl`/`loadCasePnlsWithCost`/`backfillAbcPartnerIds`，黄金 ¥13,152 不回退。
2. **配置归一覆盖读/回滚路径**：`row2config`（或 loadConfig/peekConfig/rollbackConfig）对读出的 `config_json` 做 best-effort normalize，治历史 `discount.def=90`；回滚 target 也校验。
3. **NGS 缺成本/售价质量标记**：缺成本/缺价不再 confirm 后按 0 静默入账 → 写 `statement_quality_flags`（或等价标记），P&L 默认排除/单列「未核 NGS 毛利」；响应同时返回 `missingPriceCount`。
> v0 首次归类死锁 → 随 Epic F 建档向导重写；NGS 读 RBAC → 随 Epic C 权限统一。

## 9. 分阶段实施
- **Phase 0（1–2 周）**：§8 正确性修复 + 守黄金。
- **Phase 1（2–4 周，真实月结 MVP）**：月结工作台基础版 + `statement_aggregate_revenue`/`statement_out_ledger`（A/C）+ 宽表列级语义（B）+ 默认目录补 HPV-E6E7/FISH/NGS/远程（G）+ 模板化预览 + confirm 带原因/影响/审计。**验收包 = 3 样本**：东安科目汇总 / 赣州纯外送 / 平泉宽表远程会诊，打通「预览→分类→入库/登记→月结汇总→导出」。
- **Phase 2（3–6 周，治理可审计）**：规则影响预估 + 重算（决策④）+ 黄金值产品化（F）+ LIS 匹配恢复（H）+ maker-checker + 月结导出包。
- **Phase 3（后续）**：批量文件自动识别/模板推荐 + 样本库 + NGS 前端化与公司级外送接口 + 完整度/风险评分 + 通知任务分派。

## 10. 验收标准（采自 codex）
- **功能**：同月可同时导入 5 类样本并在工作台看到统一状态；赣州纯外送形成外送事实 labRevenue=0 不阻断月结；东安科目汇总按科目映射 IN/OUT 进聚合层；平泉/宁波宽表金额可追溯来源列、列级规则下月复用；人工确认必填原因且审计可查；缺成本/价/LIS 以质量标记呈现不计正常完成。
- **数据**：和睦家黄金 ¥13,152 不变；每批次可追溯文件哈希/模板版本/规则版本/操作；月结金额可拆为 IN 劳务/OUT 外送·合作/调整/未匹配/成本未就绪；规则变更不静默改写已结账月。
- **体验**：「待人工归类」必给候选规则+影响金额+下一步；科目汇总/纯外送/共建有结构化预览表（非一行说明）；月结负责人无需多隐藏入口即可完成导入→异常→确认→导出。
