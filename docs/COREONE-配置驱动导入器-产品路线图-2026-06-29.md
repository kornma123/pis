## 0. 定位与本质
产品本质**不是 Excel 导入器，而是病理实验室月结 sub-ledger**（承接收入·外送·成本·调整·关账·审计）。导入只是入口。

## 1. ✅ 已锁定决策
1. **病例身份 = 全链路复合键 `(partner_id, case_no)`**（医院各自编号可撞号；LIS/导入/人工覆盖/ABC 回填/P&L join/成本 rollup 全升级）。
2. **04 修复时机 = 先修「正确性子集」**（跨院串账全链路 + 配置归一读/回滚 + NGS 缺成本质量标记）；v0 死锁随 Epic F、NGS 读 RBAC 随 Epic C。
3. **联合运营/共建 = 逐院 `special.joint` 决定**是否进院级 P&L 主表；默认 OUT 单列。
4. **重算 = 已结账只出调整单 / 未结账可一键重算**（新快照，不改已上报数）。
5. **对账单为收入主源；LIS 不作收入生产主脊**（LIS 较薄，只做病例匹配/完整度校验/成本归属——双引擎已评估搁置，见 §3.1）。
6. **Phase 1A = 最小对账闭环**（非完整院级盈亏关账）；3 OUT/汇总样本 + **和睦家病例级黄金锚**。

## 2. 北极星 + 两层成功标准
**北极星**：财务能在一个月结周期内完成所有医院结算单导入、分类确认、异常处理、院级盈亏生成与审计留痕，**不再依赖线下 Excel 当主账**。
- **Phase 1A 成功（最小对账闭环）**：东安科目汇总入聚合账本；赣州纯外送入 OUT 台账（labRevenue=0/outSettle=全额）；平泉宽表保留来源列、按列确认 OUT；3 样本可追溯文件/sheet/行/列/规则版本；质量标记说明是否阻断入账/关账/谁处理；可导出最小差异包；**和睦家黄金 ¥13,152 病例级回归**。
- **完整月结成功（Phase 1B/2）**：7 模板均有路径；病例级/聚合/OUT/共建/调整统一进月结视图；未结账可重算、已结账只调整单；P&L 快照可审计；所有金额可解释为 IN/OUT/调整/未匹配/成本待补/人工确认。

## 3. 产品原则
- 不强行把所有结算单塞进病例级模型（逐病例/科目汇总/宽表/纯外送/共建/retainer 并存）。
- 原始不可变、规范行可重算、派生账本可重建、人工覆盖留痕。
- OUT 不进实验室工序毛利，但必须进月结事实闭环。
- **「0」只能代表明确业务值；缺失（缺成本/价/LIS/规则/未归类）必须以质量标记暴露。**
- 配置驱动 = 样表/候选规则/影响预估/一键确认 降低配置成本，不是让财务填表。
- 工作台核心问题不是"导了几个文件"，而是"**这院这月还差什么不能关账**"。

### 3.1 已评估并搁置：LIS 生产驱动双引擎
以 LIS×收费目录算实验室收入、账单降级为对账 —— 经核实**真实 LIS 较薄（多半只有登记、数量不全）**，生产驱动收入算不准 → **对账单仍为主源**。LIS 数量列若将来变厚可重评。

## 4. 用户角色
- **财务月结负责人**：汇总各院、确认应收/扣率/外送分摊/调整、判断能否关账、出盈亏/差异包/审计。诉求：少返工、可对账、可解释。
- **运营/实施**：新院建档、模板/列识别、规则与默认目录维护、异常处理。诉求：样表驱动配置、改规则前知影响范围。
- **管理层/经营分析**：看院级收入/成本/毛利/OUT 占比/风险、追溯来源。诉求：数据可信、口径稳定。

## 5. 期间归属规则（v4.1 关键补项 —— 真实样本存在冲突）
真实冲突：赣州覆盖 2026-01~03（跨月）；平泉文件名/标题=202603 但 sheet=202510；宁波文件名=202602 但 header=2024-01；和睦家 2026-03 日期落在 2 月结算表内。
- **批次字段**（`statement_import_batches`）：`declared_period_start/end`、`settlement_month`、`period_source(filename|sheet|header|user_selected|row_dates)`、`period_conflict_flag`、`period_conflict_note`、`multi_month_batch`。
- **行字段**（`statement_normalized_lines`）：`service_date/send_date/receive_date/report_date`、`row_settlement_month`、`period_conflict_flag`。
- **默认优先级**：① 用户在工作台选的 `settlement_month` > ② 文件标题/表头声明期间 > ③ 文件名 > ④ sheet 名 > ⑤ 行级日期。
- **规则**：冲突时**不静默选月**，生成 `period_conflict` 质量标记（阻断入账+关账）；跨多月文件原始 batch 不拆，派生账本按 `row_settlement_month` 汇总；已结账补单只出调整单。

## 6. 数据模型（两层）
**第一层 — 不可变导入 + 规范行**：`statement_import_batches`（医院/月份/模板/文件哈希/解析器版本/配置版本/上传人）、`statement_raw_rows`（原始网格快照）、**`statement_normalized_lines`（核心）**。

### 6.1 `statement_normalized_lines` 字段（v4.1）
`line_id / batch_id / partner_id / settlement_month / row_settlement_month / case_no(可空) / external_subject_key(无号时稳定行身份) / patient_ref(脱敏) / department / item_name / service_code / source_sheet / source_row / source_column / source_label / section_label / template_family / row_kind(detail|subtotal|declared_total|header|note) / line_grain(case|aggregate|out|joint|adjustment|retainer) / business_line(IN|OUT|UNKNOWN|NEUTRAL|EXCLUDED) / entity_scope(case|partner|department|project|hospital) / financial_metric(revenue|cost|profit|profit_rate|share) / amount_role(gross|settlement|discount|cost|adjustment|surcharge|declared_total|subtotal) / amount / quantity / unit_price / rate / rate_type(discount|allocation|tax|manual) / formula_basis(声明|行公式|配置|人工) / service_date/send_date/receive_date/report_date / classification_status / rule_id / rule_version / parser_confidence / classifier_confidence / numeric_parse_status(正常|疑似日期格式污染|非数字) / quality_state / raw_payload(长尾上下文)`。

### 6.2 关键枚举
- **`row_kind`**：防小计/合计/签名/说明行当明细（避免重复入账）。
- **`line_grain`**：case 逐病例 / aggregate 科目·partner-month 聚合 / out 外送·远程 / joint 联合运营 / adjustment 红冲·补单·调账 / **retainer 月度固定保底费**。
- **`business_line`**：IN 工序收入 / OUT 移出 / UNKNOWN 待判 / NEUTRAL 说明·小计·非业务 / EXCLUDED 明确排除。

### 6.3 石门共建表达（利润表，非收入行）
用 `entity_scope` + `financial_metric` 表达：医院应分得收入(hospital,revenue/share)、医院承担成本(hospital,cost)、医院利润(hospital,profit)、科室/项目(department/project,...)、成本利润率(profit_rate)。**数字被 Excel 日期格式污染 → 生成 `numeric_format_anomaly`，阻断入账。**

### 6.4 声明合计/小计
独立合计行：`row_kind=declared_total` + `amount_role=declared_total` + `declared_total_scope(batch|section|business_line)`；小计：`row_kind=subtotal` + `section_label`。**两者只用于对账闭合/差异解释，不派生为收入明细。**

**第二层 — 派生账本与快照**：`case_revenue_ledger`（自然键 `(partner_id,case_no,service_month)`）/ `partner_month_revenue_ledger` / `out_settlement_ledger` / `joint_operation_ledger` / `quality_flags` / `pnl_snapshots` / `adjustment_entries`。
派生规则：只从 `row_kind=detail` 或明确可入账的 `adjustment/retainer` 派生；subtotal/declared_total 仅对账校验；UNKNOWN 不进正常收入(除非人工确认)；OUT 进台账默认不进工序毛利；已结账只新增调整单不重写。

## 7. 月结状态机
**批次态**：`uploaded / template_unrecognized / parse_failed / parsed / classification_required / review_required / posted / posted_with_flags / cost_pending / recompute_pending / recomputing / recompute_failed / ready_to_close / closed / adjusted / voided / superseded`。
**医院月份态**：`not_started / collecting / in_progress / blocked / posted_with_flags / ready_to_close / closed / needs_adjustment`。
原则：工作台首页围绕"还差什么不能关账"；每个 `blocked` 可下钻到质量标记/批次/规范行/责任人；未结账可从 `posted_with_flags` 回 `recompute_pending`；已结账只进 `needs_adjustment`，不直接重算覆盖。

## 15. 关键验收场景（Given/When/Then）
1. **期间冲突**：文件名/sheet/header/行日期冲突 → 生成 `period_conflict`，不静默入错月。
2. **声明合计/小计**：合计/小计行 → `row_kind=declared_total/subtotal` 保留，不当明细重复入账。
3. **赣州跨月纯OUT**：覆盖 1–3 月 → 原始 batch 不拆，OUT 台账按 `row_settlement_month` 汇总。
4. **平泉宽表**：远程会诊结算列 → 每金额保留来源列/标签，财务按列确认 OUT。
5. **石门数字异常**：金额列出现日期格式值 → `numeric_format_anomaly`，阻断入账+关账。
6. **缺成本**：NGS/OUT 缺成本 → 不按 0 算正常毛利，生成阻断毛利关账的标记。
7. **跨院同号（防护回归）**：A/B 同月同号 commit → 两院各自保留不串账，黄金 ¥13,152 不变。

## 16. 指标树（三层）
- **主指标**：月结完成时长（首文件→关账自然日）。1A=3 样本演示月闭环；1B=7 模板有路径、文件闭环率≥80%。
- **过程指标**：文件闭环率 / 金额解释率 / 自动分类率 / 人工触达率(每万元) / 成本就绪率 / 规则变更影响可解释率。
- **防护指标（红线=0/通过）**：黄金 ¥13,152 不回退；误入账率(缺成本/价/LIS 当正常)=0；已关账静默改写=0；跨院同号串账回归通过。

## 17. 风险与控制
| 风险 | 控制 |
|---|---|
| Phase 1A 膨胀 | 只做最小对账闭环，不做完整月结 |
| 错误月份入账 | period_conflict 默认阻断 |
| 小计重复入账 | row_kind 区分 detail/subtotal/declared_total |
| 石门共建算错 | entity_scope+financial_metric+数字异常标记 |
