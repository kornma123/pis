## 1 · 起点：抗体不就是一种物料吗？直接算库存不就行了？

**用户问**（大意）：账实核对里发现"逐抗体成本（antibody-cost）"这个后端功能没有前端 UI。抗体不就是一种物料吗？不应该直接计算库存就行吗？把抗体的字段重新设置一下不就行了？你说的"补 UI"是这个意思吗？

**我做的事**：先只读核实系统里抗体到底怎么建模的（Workflow 多 agent 核代码）。结论：
- 抗体在生产成本模型里**不是**按 `materials`（物料）表算的。它有一张**专属主数据表 `antibodies`**（`DatabaseManager.ts:1171`），含 `conv_rate`（换算率，一瓶做多少片）、`per_test_price`（每片一抗价，台账已换算好）、`form`（剂型：原液/即用，价差约 6 倍，`UNIQUE(name,form)`）。
- `materials` 表里也有抗体，但只是**演示 seed 数据**（整瓶采购价），成本引擎不读它。
- **为什么另建表**：抗体真价差达 344 倍（¥0.29~99.82/片），且"每瓶做多少片"这个换算是 `materials` 的字段（`price`=整瓶价、`spec_qty`=包装规格量）**装不下**的——`materials` 缺"换算率 / 每片用量 / 剂型"三个关键字段。

**收敛**：抗体不能简单当普通物料塞进 `materials`。"补 UI"这个念头背后其实是一个更深的问题——**我们到底想用库存/BOM 算出什么？** 这引出下一节。

**新问题**：如果目标是"单切片成本"，那 BOM + 库存这条链到底能不能算准？

## 2 · 真问题：「准确的单切片成本」里藏着两个打架的诉求

**用户说**（大意）：用 BOM 算切片成本，是的，跟现在"医院盈亏看板"的目标是一致的；我加入"消耗对账"这些功能，目的就是为了**更准确地**算单切片成本。我认可你提的挑战，但现在遇到一个问题，我该怎么办？先不从产品实现角度，**先从业务逻辑出发**。

**我怎么想 / 收敛**：我把用户要的"准确的单切片成本"拆开，发现它同时想要两个在数据本质上**打架**的属性：

| | 能不能摊到单片 | 准不准（是不是真实消耗） |
|---|---|---|
| BOM 标准成本 | ✅ 天生按片算 | ❌ 是"应该用多少"，不是"实际用了多少" |
| 实际消耗（出库/对账） | ❌ 是月度、按物料/批次汇总的一坨 | ✅ 真金白银 |

**核心洞察**：那"一坨"实际消耗里混着正常用量 + 抗体开瓶浪费 + 重做 + 共用试剂 + 盘点误差，而且**系统里没有"这一坨是哪几张片造成的"这个记录**——出库是按物料/批次记的、不是按切片记的。**物理上就摊不回单片。**

→ 所以用户加"消耗对账"想让单片成本更准，会卡在这里：**对账能告诉你"这个月总量对不对"，但告诉不了你"这一张片实际花了多少"。**

**我当时的初步方案**（后面被自己和多轮复核修正）：分两层——单片永远用"标准成本"算（只有标准可摊到片）；实际消耗不去改单片，而是**周期性地校准"标准的参数"**（如某料实际系统性比 BOM 高 20% → 回填损耗率）。即对账 = 校准标准，不是算单片。

**新问题**：这套要用到"实际消耗"来校准——可这个数据真的有吗？

## 3 · 诚实的墙：库存从来没有被认真统计过

**用户诚实交底**：我并没有办法给你"实际总消耗"，因为**库存从来没有被认真统计过**。所以我是不是可以这么理解——**整个进销存和 BOM 这套逻辑，只在逻辑上成立、业务上根本不成立？**

**这是全场第一个转折点。** 我的回答（劈两半，不一锅端）：

| 这一层 | 依赖什么 | 库存没统计还成立吗 |
|---|---|---|
| **单片标准成本**（BOM 用量 × 采购价 + 抗体瓶价÷换算率） | 配方对不对 + 采购价对不对 | ✅ **成立**——不需要出入库流水 |
| **消耗对账 / 用实际校准成本** | 真实出库/消耗数据 | ❌ **不成立**——实际侧是空的 |

**关键**：单片标准成本**不依赖"库存被认真统计"**——它只要配方（SOP/技术员经验就能定）+ 采购价（发票就有），两样都不需要出入库流水。真正塌掉的，是"用消耗对账去证明/校准成本准不准"那一层。

**更狠的一层（adoption）**：实验室这种地方，出入库流水**大概率永远不会被认真记**（费人工、没人考核）。把成本准确性寄托在"认真做出库对账"上 = 寄托在一个**业务上不会发生的行为**上。这不是逻辑错，是**准确性的来源选错了地方**。（呼应项目原则：有用但没人用 = 产品失败。）

**换锚点（初版）**：你缺"出库流水"，但你不缺两样天然存在的东西——**采购流水**（买要付钱）+ **LIS 真实片数**（做检测出报告）。所以：**每片真实材料成本（粗）= 一段时间采购总额 ÷ 同期真实片数**，完全绕开库存统计。

**新问题**：这个"采购÷片数"看着能绕开库存，但它经得起推敲吗？

## 4 · 三个致命追问：滞后 / 拆到医院 / 哪个价的抗体

**用户连发三问**（这是全场最硬的一轮）：
1. **采购到消耗是滞后的**：采购发生在 3 月，制片可能在 5 月。（→ "本月采购÷本月片数"时段错配）
2. **即使有这个数据，你怎么拆分到每家医院？**（一瓶抗体服务很多医院的片）
3. **核心：怎么确认"这批病例用的是哪个价格的抗体"？你没法确认。**（3 月的货 vs 5 月的货，价不同；某病例用了哪瓶，无从追溯）

**这是第二个、也是最深的转折点。** 我的回答：**你在试图解决一个成本会计一百年前就"主动放弃"解决的问题。**

- 追"哪张片用了哪瓶、哪个价"——**不是数据不够走不通，是它本来就不该走**。全世界工厂都做不到把某颗螺丝的采购价追到某台成品上。会计的解法叫**"约定价/标准成本"**：同一种料，每次消耗都记**同一个约定单价**，不管物理上用的哪批。加权平均法、标准成本法**存在的唯一意义，就是让"哪瓶对哪片"这个问题变得不需要回答**。

用这个视角，三问里两问直接消失、一问被修正：

- **问 3（哪个价）→ 消失**：同一抗体每张片记**一个约定价**。"用 3 月还是 5 月的货"这个问题不存在了，因为你**选择**让它们都用同一个价。你放弃"单片精确"，换来"口径统一、可汇总"。
- **问 2（拆到医院）→ 消失**：**不拆瓶，拆产出**。某院某抗体成本 = 该院该抗体**真实片数（LIS 有）× 约定每片价**。瓶子永远不用拆到医院——走"约定每片价"这个中间量。
- **问 1（采购滞后）→ 修正了我**：正因为滞后，"本月采购÷本月片数"是**错的**（用户抓死了我上一版的粗办法）。采购数据的正确用途**不是**逐月算每片，而是**拉长窗口（滚动 6~12 个月）周期性重定"约定价"**。滞后被长窗口 + 约定价平滑吸收。

**收敛（重要）**：用户系统里那张**抗体台账的 `per_test_price`，本质上就是一个"约定标准价"**。台账这个东西**方向从来没错**——错的只是它手工不更新、且是孤岛不喂盈亏。

**新问题（用户马上提出）**：可"约定标准价"要靠 BOM/配方来乘用量——**做 BOM 不就又回到"没人维护"那个问题了吗？**

## 5 · 自我质疑：BOM 没人维护？能不能跳出 BOM？

**用户要求**：做 BOM 不就遇到你刚才说的"没人维护"问题吗？你能跳出 BOM 这个思路去思考吗？**你质疑下自己的观点，看有没有遗漏或者不妥的**。这一版讨论的核心是**基于我们的实际情况先摸清业务逻辑**。

**我做的事**：启动一个对抗性自审 Workflow（4 个视角砸我当时倾向的"自上而下按加权产出分摊"方案）：会计严谨 / 数据现实（读码）/ 采纳维护 / 决策有用性。自审 + 我自己反思，逼出几个我原方案的硬伤（这些后来都进了固化 doc）：

- **没区分变动 vs 固定成本**（最大会计硬伤）：把"总支出"一锅端分摊，混淆了变动（试剂，可避免）和固定（折旧/房租/人力，不可避免）。判去留应看**贡献毛利**，固定成本别硬摊。
- **自上而下分摊"总额对但院际分配不可验证"**：权重错了没有 ground truth 纠。
- **固定成本分摊做去留决策的陷阱**：砍掉低毛利院会把固定成本转嫁给其他院 → 死亡螺旋。
- **我又一次乐观假设"总支出可得"**：该回收（用户刚教育过别假设他没有的数据）。

**收敛**：判去留改用**贡献毛利 = 实收 − 可避免变动成本**；固定成本不下沉、只在实验室整体看。

**新问题**：这些还是"嘴上论证"。用户要求——**拿数砸**。

## 6 · 拿数砸：端到端数值模拟（cm_sim.py）

**用户要求**：你先假设一些数据，跑一遍计算，**边跑边模拟真实数据输入**（采购等行为都要还原真实场景），看能不能发现你的想法漏洞。**先砸模型，砸完做一次优化，再砸。**

**我做的事**：写了 `docs/analysis/cm_sim-贡献毛利端到端模拟-2026-07-03.py`——4 家画像不同的医院（大三甲代送/县医院/专科特染/小客户分子）、一个季度、还原真实采购行为（大宗囤货 + 批次涨价 5~25% + 收得率 240~310 片/瓶的波动 + 缺价抗体 + LIS 认不出漏例）。跑出来暴露的失效模式（**注意：具体百分比是假设数据的产物、不代表真实量级；价值在暴露结构性问题**）：

1. **死亡螺旋（数值坐实）**：按抗体价加权把固定成本摊到院，把**贡献毛利第一名**的大三甲**翻成"亏最多"**。据此砍院 = 砍掉最赚钱的那家。→ 证明"固定成本不下沉、判去留用贡献毛利"是对的。
2. **约定价系统性偏乐观 ~30-40%**：标称换算率（如 300 片/瓶）> 真实收得率（240~310）+ 批次涨价 → 变动成本被低估、毛利虚高。**（其中一半是"囤货/期末库存没摊完"的时间错配 = 正好坐实采购滞后；另一半是真偏差。）**
3. **缺价精准打击贵抗体院**：真缺的几种价恰是贵的分子抗体，集中在做分子的院 → 那家院毛利虚高最狠。
4. **覆盖率漏例**：每例贡献毛利稳、院总额少算 → 院际对比要用"每例"、别用总额。
5. **收入/成本必须同粒度**（我模拟自己的漏）：我用了统一片价，导致贵抗体（成本 > 统一片价）算出负贡献——不真实，真实里贵抗体收费也高。**暴露硬约束：细成本必须配细收入，否则冤枉贵抗体院。**

**收敛**：贡献毛利判去留站得住；但它的"变动成本"这条腿本身有 ~30% 乐观偏差（收得率 + 批次价 + 缺价），不校准就是"看着精确的虚高毛利"。**好消息**：最狠的收得率偏差，用"采购瓶数 ÷ LIS 片数"就能校准（两个都是能数的事实）。

## 7 · 认识论转折：「数据只是数字，算得出就算得出」→ ①②③ 判断框架

**用户的关键一击**：所以你这个思路不成立对吗？因为在我的认知里，**数据只是数字，并不影响你基于假设做算数**，且你知道哪些数据可用，**算得出来就算得出来，算不出来就是算不出来**。

**这是第三个转折点，也是全场智力上的高点。** 用户点破了我"需要真数据来验证"这句话的含糊——**对一个定好的公式，真假数字不改变"算不算得出"，只改变"准不准"**。

我的回应：你是对的，而且我这几轮跑的**不是数字，是结构**。并给用户的"算得出/算不出"二分加了一个**中间档**（因为它最会骗人）：

| 档 | 是什么 | 例子 | 危险 |
|---|---|---|---|
| **① 算得出·有真值** | 有唯一正确答案，数据只是填进去 | 材料成本=配方×采购价、收入=分级×扣率、组织处理=蜡块×单价、收得率=瓶数÷片数 | 无。用户说的"数据只是数字"说的就是这档 |
| **② 算得出一个数·无真值** | 永远能选个权重除一下得个数，但那个量**本身是约定不是事实**、不可证伪 | 房租/折旧/共用批 **摊到某院或某例**、机会成本 | **最会骗人**——长得跟①一样，但没有对错可验证 |
| **③ 真·算不出** | 事实从未生成、无法归因 | 单张片**实际**消耗、哪瓶对哪片 | 真没有。但**方案本来就没用它** |

**收敛（本方法论的核心）**：**判一个成本口径成不成立，不看"有没有数据"，看"这个量属于①②③哪一档"。** ①的照算（缺数据也不降级③）；②的老实标"约定不可证伪、别当精算砍院"；③的绕开。
→ 直接回用户："思路不成立吗？" **不**。贡献毛利只用①的量 → 成立；不成立的是"院级全成本分摊"（②，是约定不是事实，所以我从头就不摊它）；③我压根没依赖。

## 8 · BOM 翻案：不估算要准确算 → BOM 是唯一入口

**用户定调**：我的诉求就是**不估算，准确计算**。所以基于这个思路，你还觉得 BOM 清单没用吗？

**这是第四个转折点——用户把我最初"BOM 没人维护、意义不大"的判断绕回来了，而且他是对的。**

我的回应：**在"不估算"这个诉求下，BOM 不但不是没用，它是唯一入口。**
- 准确计算每例材料成本 = **配方 × 单价**。配方就是 BOM。没有 BOM 就没有配方，就只能退回"约定价/分摊"= 估算。你要不估算，就**必须**有 BOM。
- **坦白**：我之前那个"约定价"（瓶价÷换算率）**其实是偷偷用了一份退化的、每抗体一行的 BOM**——我从没逃开配方，只是用了个粗版。逃不掉。
- 我那两条 BOM 批评在"不估算"下得重判：**"没人维护"不是 BOM 的罪，是准确的价钱**（你要准就得养配方）；**"理论≠实际"是伪命题**（你已认实际是③测不到，那 BOM-标准成本本身就是口径、有真值，不是对实际的近似）。

**用户拍板的维护原则**：BOM 清单可以维护，但**维护频次拉到年以上**。我认可并加了一个精度：**要维护的是"配方"（稳定量→年度/事件维护），不是"价格"（易变量→随采购快流）。两个时钟必须分开。** 这消解了"要么冻一年不准、要么天天维护"的假两难。

**两条诚实边界**：① BOM 给的是"准确的**标准成本**（该花多少）"，不是"准确的实际花多少"（③，做不到）；② BOM 能把材料成本做成①，但**每例固定成本分摊永远是②**——"不估算"这诉求，材料层可 100% 满足、固定成本那层永远满足不了（是共用成本的本性，范畴性的）。

## 10 · 最终固化了什么（结论速览，细节见固化 doc）

1. **①②③ 判断框架** = 判成本口径的总闸（看档位不看有没有数据）。
2. **成本口径 = 标准成本（该花多少）**，不是实际、不是分摊；标准≠实际是特性，别跟财务对账。
3. **判去留 = 贡献毛利**（实收 − 可避免变动成本）；**固定成本不下沉到院**（死亡螺旋）；院际比用每例、别用总额；贡献毛利≠利润、≠无脑保留（产能）。
4. **BOM 是准确材料成本的唯一入口**；配方慢（年度/事件）价格快（随采购）双时钟；约定价=退化一行 BOM（但含固定成本的打包价要拆）。
5. **两条地基**：成本单位"片"三口径（试剂乘染色片数）；配方按**合作形态**裁工序集（与收入侧同源）。
6. **抗体**：已进引擎但用扁平 seed 价 → 换台账真价（路径 a）；改前对齐 ABC 口径。
7. **收得率 = 瓶数÷片数**把系统性损耗做实，但管道有硬前置（命名映射 + 真实采购流水）。

## 12 · 仍然开放的问题 / 给 codex 的靶子（我们自己知道还没解决的）

按已知重要性排（**这些是最值得你深挖的**）：

1. **合作形态/工序集怎么落**：已拍板"现在纳入"，但"case 级协议默认 + 账单码/LIS 工序反推"怎么可靠地判"这个 case 我们做了哪几步"？判错就系统性高估/低估。与收入侧步骤集"强制同源"具体怎么保证？
2. **分子线（FISH/NGS）成本口径**：暂标"未建模"。但分子最贵、最集中在大院，长期挂着会让这些院的盈亏看板缺一大块。另立的口径（探针/反应 + case 级文库批量摊 flowcell）怎么设计才不重蹈"配方×片数"的错？
3. **对照片/质控片（②）**：每批固定、摊到单片随 batch 浮动。是"整体层面兜"还是"每 N 张染色片配 1 套对照的标准率（①\*）"？后者的分母（行业均值 or 本实验室实测）怎么定？
4. **收得率管道的硬前置**：三方命名（材料码 `MAT-IHC-*` / LIS marker 自由文本名 / 台账 `(name,form)`）不互通，只有一个有真撞键（TCR αβ/γδ）的模糊 resolver；采购/批次是 seed 演示数据非真实流水。这两个地基不建，收得率、"价格走采购快流"都只能产演示值（= 假精确风险）。
5. **抗体两套并存表示**：`antibodies` 台账（真价）vs `materials` 一抗物料（seed 价），零 JOIN。换价源（路径 a）前必须先定权威关系，否则造出第三套口径。改它波及 ABC 预览 + 出库快照的生产报表数——**要不要像收入侧账实核对一样走双轨？**
6. **深切/连切/rescue 系统性低估复杂院**：成本动因用"染色片/明细行数"抓不到深切连切的额外消耗（差一个量级），系统性偏向低估疑难/肿瘤大院。用"疑难标准附加率（①\*）"吸收够不够？
7. **产能/机会成本**：贡献毛利判去留只是"单看该 case 值不值得做"的下限判据。产能受限时（染色机时/人力有上限），低单价院挤占产能 = 负机会成本，可能该缩量而非留。机会成本是②（无真值不下沉），但产能受限时怎么"定性纳入"去留？
8. **标准成本长期漂移**：配方年度维护、价格随采购——如果没人真去年度复核配方、或收得率校准信号本身有偏（如漏例导致片数偏低 → 收得率算高 → 成本算低），标准会不会系统性漂移且无人发现？校准的校准是什么？
