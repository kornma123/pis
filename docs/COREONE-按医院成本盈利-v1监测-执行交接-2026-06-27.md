## 0. 产品定位（锁定）
- **面向中大型第三方医联体（多院）的 ABC 运营监测平台**；目标用户＝实验室管理者；核心＝**运营监管**（不只是算账）。
- **v1 = 监测（回看）**；**前瞻轴（报价/盈亏平衡/预算/销售赋能）= v2**（用户决策）。
- **LIS 统一一套**（跨院数据可信，数据治理风险已降）。

## 1. 锁定的核心模型（不要再推翻，除非用户改）
1. **收入 = 财务实收 × 引擎技术占比**；**收入独立计算，绝不改 `cost-calculator.ts`**（比原计划更安全）。
2. **成本 = 既有 ABC 引擎**（M1–M5 已验证）。⚠️ **黄金用例 ¥120 只证"算得对(数学)"，不证"算得真(客观)"**——真实性靠 **实测数据 + 总账勾稽 + 差异监测**，**不用普适常量**（呼应调研"每实验室自有口径"）。
3. **v1 P&L = L1 + L2**：L1 直接贡献毛利(收入−试剂耗材[+外送]) / L2 作业毛利(−技术人工−设备)。**不含房租/医生 L3**（避免假精确）。
4. **技术占比 = 收费引擎**按真实目录规则（基础量+增量+封顶 / 分段单价 / 多动因）算每 case 的 诊断/技术/取材 金额 → 占比。**适用旧码/私立**（只需 LIS 数量）。
5. **LIS = 批量导入；每 case 只给「检测项目 + 基础数量」** → **必须新建 `检测项目→收费码模板`**：把项目的基础数量(蜡块/切片/抗体…)展开成各收费码 + 数量，引擎才能算。
6. **执行起点 = 合成数据先把全链(导入→引擎→院级 P&L→异常)搭通+测试**，真实 LIS/财务样例到位再接。
7. **RBAC 数据驱动**：`partners`/`partner_pricing` 模块已注册(29)，定价归 finance/lab_director。
8. **期间默认按服务月**归集（实收回填到服务月）；消耗/运营异常=日级，真实毛利=月级(受财务结算)。
9. **PSI 低成本准确**：标准消耗 = BOM×LIS case 自动推算(零摩擦)；实际 = 周期盘点；差异 = 对账(已建)，既校验又是异常信号。**不要逐 case 手记**。
10. **财务 = 手动上传"对账单"（非实时、非标准编码）**：现实是财务给带编码明细的对账单、手动上传、编码新老混用+自费。**v1 做 code-agnostic 金额导入器**：只取「金额 + case 匹配」，**不解析/分类编码**（拆分靠引擎 LIS 占比）；原始编码明细行原样存(备未来)但不依赖其语义 → 穿越"编码 6–12 月标准化"不返工；自费照记金额；未来"基因检测管理系统"= 另一独立 feed(模块化预留, v2)。

## 1.5 成本归集机制 + 时间模型（关键，本轮新增 — 出库/财务都非实时）
**不物理追踪"哪瓶试剂用在哪个 case"——靠 BOM 标准 + 期末盘点对账 + 差异回摊（已定加回摊）：**
1. **归集(每 case·准实时)**：单 case 标准成本 = Σ(BOM 标准用量 × 物料价)，输入仅 LIS(项目+数量)。不依赖出库/财务，case 进来即有成本。
2. **实际(每期·非实时)**：期间实际消耗(每物料) = 期初 + 入库 − 期末盘点（总额，非 per-case）。⚠️ **出库记录批量+滞后 → 不作 per-case 成本依据**，只作 WIP/粗信号；权威实际 = 盘点。
3. **对账 + 回摊(期末)**：差异(每物料) = 期间实际 − Σ该期各 case 标准用量；**按标准份额回摊到 case** → 实际调整后单 case 成本，使 `Σ各case实际 = 期间实际盘点`（总账勾稽成立）。差异本身 = 异常信号(BOM 错 / 浪费)。
4. **双口径呈现**：实时"标准估"(连续·日级早警) + 期末"实际核实"(月度·勾稽)；**两者之差 = 运营监管核心信号**。
5. **多院(共享库存)**：实际消耗只有实验室级总额(共享瓶)，无法物理按院拆 → **按医院成本 = 该院各 case 标准 + 差异按标准份额回摊**，天然 `Σ各院 = 实验室实际总额`(勾稽)。
- **时间表**：日级 = 标准成本 + 消耗异常粗信号；月末（入库价 + 盘点 + 对账单实收）= 实际核实 + 真实毛利。
